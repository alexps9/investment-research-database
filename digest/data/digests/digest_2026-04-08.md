### 推理阶段 Scaling 规律：更大模型用更少 inference tokens 达到更高准确率

- **信号源**：Ross Taylor @ Anthropic（引用 Claude Mythos Preview system card）
- **链接**：[Ross Taylor on inference scaling](https://x.com/rosstaylor90/status/2041598448189735328)

**信号内容**

Ross Taylor 从 Claude Mythos Preview system card 中提取关键图表，展示更大模型在推理阶段以更少 inference tokens 即可达到更高准确率，并将此现象类比为 GPT-3 时代发现的训练 sample efficiency 规律在推理阶段的延伸。

**认知提取**

这一发现揭示了推理阶段的 scaling 规律：更大的模型在 inference 时更"token 高效"——用更少的推理 token 就能达到更好的准确率。这在某种程度上类似于 GPT-3 时代发现的"大模型用更少训练数据就能学得更好"，只是现在这一规律从训练阶段延伸到了推理阶段。

## 认知模型 (Cognitive Models)

### LLM Knowledge Editing 的"表面顺从"问题：模型并未真正修改内部记忆

- **信号源**：Kai Zhang 等 @ arXiv
- **链接**：[Surface Compliance in Knowledge Editing](http://arxiv.org/abs/2604.05995v1)

**认知提取**

这项研究揭示了 knowledge editing 领域的一个深层问题：现有评测框架容易被模型"表面顺从"所欺骗，模型只是在特定 prompt 下输出正确答案，并没有真正改写内部参数记忆——这有点像学生死记答案应付考试，但并没有真正理解和内化知识。递归编辑还会让模型记忆状态不可逆地退化。

**论文摘要**

研究发现 LLM knowledge editing 存在"Surface Compliance"现象：模型在 benchmark 上表现良好，但实际上并未真正修改内部记忆，仅是模仿目标输出。递归修改还会积累表示残留，导致认知不稳定。标准 benchmark 指标虚高的根源在于评测方式过于依赖特定 prompt 模板，无法穿透模型的"背答案"行为。

**核心方法**

- **方法框架**：提出一套诊断框架，将 knowledge editing 后的模型置于 in-context learning（ICL）的判别式自评估（discriminative self-assessment）设置下进行测试，通过多选题（MCQ）形式考察模型在更贴近真实应用的条件下是否真实修改了内部知识。
- **技术细节**：具体做法类似于出"判断题"而非"填空题"：让模型在 ICL 上下文中对多个候选答案进行判别打分（SA-MCQ），从而绕开 knowledge editing 后模型在特定 prompt 模板下"背答案"的行为，揭示其内部信念是否真正发生改变。

**实验成果**

- 发现 Surface Compliance 现象：knowledge editing 高 benchmark 成功率背后，模型仅模仿目标输出而非真正修改内部信念
- 提出基于 ICL 的判别式自评估诊断框架，更准确地检测 knowledge editing 是否真正生效
- 递归知识修改会积累表示残留，导致认知不稳定，并永久降低模型记忆状态的可逆性

**总结与反思**

- **结果总结**：实验表明当前主流 knowledge editing 方法普遍存在 Surface Compliance，标准 benchmark 指标虚高；递归编辑场景下模型表示空间出现不可逆退化，说明现有编辑范式在长期可持续性上存在严重隐患。

### In-Place TTT：无需重训即可为现有 LLM 添加 Test-Time Training 能力

- **信号源**：Tianle Cai 等 @ arXiv
- **链接**：[In-Place TTT](http://arxiv.org/abs/2604.06169v1)

**认知提取**

Test-Time Training 让模型在推理阶段也能更新部分参数以适应新信息，但现有方案与主流 LLM 架构不兼容、计算开销大。In-Place TTT 通过复用 MLP 中本就存在的投影矩阵作为"快速可调权重"，并将训练目标对齐 Next-Token-Prediction，使 TTT 能像插件一样嵌入已有模型——这在某种程度上类似于给已出厂的发动机加装一套自适应喷油控制器，而不是重新设计整台发动机。

**论文摘要**

In-Place TTT 框架将 MLP 最终投影矩阵作为 fast weights，无需重新预训练即可为现有 LLM 赋予 Test-Time Training 能力。通过将训练目标从通用重建损失替换为与自回归语言建模一致的 Next-Token-Prediction 损失，并引入 chunk-wise 分块更新机制兼容 context parallelism，在 128k 长上下文任务中显著提升 4B 模型性能。

**核心方法**

- **方法框架**：In-Place TTT 框架将 LLM 中 MLP 块的最终投影矩阵标记为 fast weights，在推理时通过反向传播对其进行小批量更新。训练目标从通用重建损失替换为与自回归语言建模一致的 Next-Token-Prediction 损失，并引入 chunk-wise 分块更新机制以降低内存和计算开销，同时兼容 context parallelism 扩展。
- **技术细节**：Fast weights 更新仅发生在推理阶段，每个 chunk（若干 token 的窗口）计算一次梯度并原地更新投影矩阵，避免了存储完整反向传播图的内存代价。这在某种程度上类似于在线学习中的小批量 SGD，只不过"学生"是推理中的 MLP 层，"教材"是当前输入上下文的 next-token 监督信号。

**实验成果**

- 提出 In-Place TTT，以 MLP 最终投影矩阵为 fast weights，无需从头重训即可为任意 LLM 添加 TTT 能力
- 设计与 Next-Token-Prediction 对齐的理论有依据目标，结合 chunk-wise 更新机制，支持 context parallelism
- 4B 参数模型在 128k 上下文任务上超越竞争基线；从头预训练时也持续优于 TTT 相关方法

**总结与反思**

- **结果总结**：作为即插即用增强方案，In-Place TTT 使 4B 模型在 128k 长上下文任务上取得领先表现；从头预训练版本在多项 TTT 相关基准上也持续超越竞争方法。

### RAGEN-2：揭示 Agentic RL 中的 Template Collapse 失效模式

- **信号源**：Jiajun Wu @ Stanford University
- **链接**：[RAGEN-2: Template Collapse in Agentic RL](http://arxiv.org/abs/2604.06268v1)

**认知提取**

研究揭示了 agentic RL 训练中一种此前被忽视的失效模式——模型看似输出多样但实为与输入无关的固定模板。这类似于学生背了很多不同模板答题，却完全没有根据题目思考；仅用 Entropy 衡量推理质量，相当于只看答案字数而忽视是否"对题作答"。

**论文摘要**

RAGEN-2 发现多轮 LLM agent RL 训练中的"template collapse"失效模式：模型 Entropy 稳定但推理与输入无关，现有指标均无法检测。研究将推理质量分解为 within-input diversity（Entropy）和 cross-input distinguishability（Mutual Information）两个维度，并提出 SNR-Aware Filtering 方法，在规划、数学推理、网页导航、代码执行等多类任务上一致提升性能。

**核心方法**

- **方法框架**：RAGEN-2 将推理质量拆解为两个维度：Entropy（同一输入内的多样性）和 Mutual Information（跨输入的可区分性）。在 RL 训练的在线阶段引入 MI proxy 进行实时诊断，同时用 signal-to-noise ratio（SNR）机制解释 template collapse 的成因——低 reward variance 导致任务梯度减弱、正则化项主导，抹除跨输入的推理差异。
- **技术细节**：SNR-Aware Filtering 在每次迭代中按 reward variance 对 prompt 排序，过滤低方差（低信号）样本，确保梯度更新来自真正能区分输入的样本。这在某种程度上类似于在训练时剔除"送分题"，让模型只在真正有辨别度的样本上更新权重，从而保留跨输入的推理差异。

**实验成果**

- 发现 template collapse 失效模式：模型 Entropy 稳定但推理与输入无关，现有指标均无法检测
- 将推理质量分解为 within-input diversity（Entropy）和 cross-input distinguishability（Mutual Information），MI 与最终任务表现的相关性显著强于 Entropy
- 提出 SNR-Aware Filtering，用 reward variance 筛选高信噪比 prompt，在规划、数学推理、网页导航、代码执行任务上一致提升性能

**总结与反思**

- **结果总结**：在规划、数学推理、网页导航和代码执行四类任务上，SNR-Aware Filtering 一致提升了模型的输入依赖性（MI 指标）和最终任务表现，验证了 MI 作为推理质量代理指标的有效性。

### SkillGraph：从 Agent 轨迹中挖掘工具执行图谱，解决工具序列排序问题

- **信号源**：Hao Liu 等 @ arXiv
- **链接**：[SkillGraph: Tool Sequence Recommendation](http://arxiv.org/abs/2604.19793v1)

**认知提取**

纯语义相似度无法捕捉工具调用的先后依赖关系，就像知道菜谱里有哪些食材却不知道烹饪顺序。SkillGraph 从历史成功轨迹中挖掘工具执行转移图，将流程依赖编码为可复用的先验，再配合两阶段检索+重排框架完成正确的工具调用排序。

**论文摘要**

SkillGraph 从 49,831 条 LLM agent 轨迹中挖掘有向加权执行转移图，作为工具序列推荐的 graph foundation prior。通过两阶段解耦框架（GS-Hybrid 检索 + 学习型 pairwise reranker），解决纯语义方法无法捕捉工具间数据依赖的排序问题，在 API-Bank 和 ToolBench 两个基准上均显著超越语义基线。

**核心方法**

- **方法框架**：SkillGraph 从 49,831 条成功 LLM agent 轨迹中挖掘有向加权图，节点为工具/API，边权重编码执行转移频率，形成可复用的 graph foundation prior。在此基础上构建两阶段解耦框架：第一阶段 GS-Hybrid（图结构辅助混合检索）负责候选集筛选，第二阶段使用学习型 pairwise reranker 对候选工具进行排序。
- **技术细节**：GS-Hybrid 检索将图结构的工具共现先验与语义相似度融合，弥补纯语义检索对工具间数据流依赖不敏感的缺陷；pairwise reranker 以工具对作为输入，通过学习历史轨迹中的排序偏好来判断两工具的调用先后关系，类似于用比赛积分制替代仅靠外观排名球员。

**实验成果**

- 在 API-Bank 上，Kendall-τ 从语义方法的 -0.433 大幅提升至 +0.613，排序质量显著改善
- 在 ToolBench（9,965 测试样本，约 16,000 工具）上达到 Set-F1=0.271、Kendall-τ=0.096
- 学习型 pairwise reranker 在相同 Stage-1 输入下优于 LLaMA-3.1-8B 的重排结果

**总结与反思**

- **结果总结**：SkillGraph 在 ToolBench 和 API-Bank 两个基准上均超越语义基线，尤其在排序指标 Kendall-τ 上实现了从负值到正值的跨越式提升，验证了图先验在工具序列推荐中的有效性。

### Action Images：将机器人动作编码为像素对齐视频，实现端到端 Zero-Shot Policy

- **信号源**：Chuang Gan @ UMass Amherst / MIT-IBM Watson AI Lab
- **链接**：[Action Images for World Action Models](http://arxiv.org/abs/2604.06168v2)

**认知提取**

研究者将机器人控制信号转化为可视化的"动作图像"（追踪机械臂运动轨迹的多视角视频），让视频生成 backbone 直接充当 zero-shot policy——这在某种程度上类似于把操作指令变成一段动画示意图，让模型通过"看懂动画"就知道该怎么动。这样做既充分利用了预训练视频模型的知识，又天然支持跨视角和跨环境的迁移。

**论文摘要**

Action Images 框架将 7-DoF 机器人动作编码为像素对齐的多视角动作视频，使视频生成模型本身即可作为 zero-shot policy 使用，无需独立 action module。在 RLBench 上实现当前最强 zero-shot 成功率，同一统一模型支持 video-action 联合生成、action-conditioned 视频生成和 action labeling 四类任务。

**核心方法**

- **方法框架**：Action Images 框架将 policy learning 建模为多视角视频生成任务：把 7-DoF 机器人末端执行器动作解码为追踪机械臂运动的 2D 像素级动作视频（action images），使视频 backbone 本身即可作为 zero-shot policy 使用，无需额外的 action module 或 policy head。整个框架统一支持 joint generation、conditioned generation 及 action labeling。
- **技术细节**：具体而言，7-DoF 动作被投影并渲染为多视角视频帧中的运动轨迹覆盖（overlay），形成与原始观测像素对齐的 action image sequence。视频生成模型在训练时同时学习场景动态和动作轨迹，推理时只需生成对应 action image 即可读出控制信号——类似于让模型在"画线描述路径"的方式中学会控制，而非输出抽象低维向量。

**实验成果**

- 提出 pixel-grounded action representation，将 7-DoF 机器人动作转为多视角动作视频，无需独立 policy head
- 在 RLBench 上实现当前最强 zero-shot 成功率，优于已有 video-space world models
- 同一统一模型支持 video-action 联合生成、action-conditioned 视频生成和 action labeling 四类任务

**总结与反思**

- **结果总结**：在 RLBench benchmark 和真实机器人场景中，Action Images 取得最高 zero-shot 成功率，并在 video-action 联合生成质量上超越已有 video-space world model 方法。

### PAPS：深度学习伪解析框架求解瞬态 Fokker-Planck 方程，推理速度提升 4 个数量级

- **信号源**：Xiaolong Wang 等 @ arXiv
- **链接**：[PAPS: Pseudo-Analytical Probability Solver](http://arxiv.org/abs/2604.06001v1)

**认知提取**

Fokker-Planck 方程描述随机动力系统中概率分布随时间的演化，传统数值方法每换一套参数就需重新求解。PAPS 将初始分布编码为低维 latent 表示，再用单一神经网络在该空间中建模全参数空间的瞬态动力学，相当于把"逐点查表"变成了"一张全景地图"。

**论文摘要**

PAPS 提出基于深度学习的伪解析概率求解框架，单次训练即可同时求解任意参数和初始分布下的瞬态 Fokker-Planck 方程。框架分为约束保持 autoencoder（将 Gaussian mixture distribution 参数双射映射到无约束低维空间）和 physics-informed evolution network 两个解耦模块，推理速度比 GPU 加速 Monte Carlo 快 4 个数量级。

**核心方法**

- **方法框架**：PAPS 分为两个解耦模块：（1）约束保持 autoencoder，将 Gaussian mixture distribution (GMD) 参数双射映射到无约束低维 latent 表示；（2）physics-informed evolution network，在 latent 空间中对任意初始条件和系统参数的瞬态概率动力学进行统一建模。训练时将初始分布、瞬态分布、稳态分布均用 GMD 统一表示，从而覆盖完整的时间演化过程。
- **技术细节**：autoencoder 通过双射映射保证 GMD 参数（权重非负且归一、协方差正定等）的硬约束，使 latent 空间无约束可微，便于神经网络优化。evolution network 在该 latent 空间中以物理信息损失（FPE 残差）监督训练，类似于在"压缩地图"上学会预测每一时刻的概率密度快照。

**实验成果**

- 提出 PAPS 框架，单次训练覆盖任意多峰初始分布、系统参数和时间点，实现瞬态 FPE 的批量并行求解
- 引入约束保持 autoencoder，将 Gaussian mixture distribution (GMD) 参数双射映射到无约束低维 latent 空间，保证物理约束
- 在多个范式随机系统上推理速度比 GPU 加速 Monte Carlo 快约 4 个数量级，同时保持高精度，可实现实时参数扫描与随机分岔分析

**总结与反思**

- **结果总结**：在多个典型随机动力系统上，PAPS 保持与参考解相当的精度，同时推理速度较 GPU 加速 Monte Carlo 提升约 4 个数量级，使原本难以处理的实时参数扫描和随机分岔系统研究成为可能。
