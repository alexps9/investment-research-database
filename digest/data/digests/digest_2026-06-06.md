<h3>Vortex：AI agent 自动搜索并部署稀疏注意力算法，GLM-4.7-Flash 吞吐提升 4.7×</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.06453">Vortex: A Programmable and Deployable Sparse Attention System for LLM Serving</a></li>
  <li><b>信号源</b>：<b>CMU Beidi Chen</b>（资深合作）· Zhuoming Chen（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>LLM serving 领域长期存在一个工程上的矛盾：稀疏注意力（sparse attention）理论上能大幅提升吞吐，但从算法构思到实际上线的调优路径极长，每换一种模型架构就要重新手工适配。<b>Vortex 把这个流程变成了一个带反馈回路的自动搜索系统</b>——AI agent 通过统一 API 调用生成稀疏 mask 方案，在真实 serving 环境里快速评估吞吐和精度，再迭代。这意味着稀疏注意力不再是少数 GPU kernel 专家的特权，而变成了可以规模化复用的工具链。往上游看，这对 vLLM 这类通用 serving 框架的地位是个结构性挑战——如果算法搜索本身能自动化，那通用框架需要提供的不仅是执行，更是可编程接口；往下游看，对依赖全量注意力的长上下文推理产品，成本压缩的空间正在被系统性地打开。</p>
</callout>

<p>　　LLM 推理（inference）领域的标准路径是：工程师手工设计稀疏 mask、编写 CUDA kernel、再与 vLLM 等 serving 框架对接。这条路径对每种新模型架构（如多头潜注意力 MLA）都需要重复，迭代周期很长。Vortex 提出了一套可编程系统来解决这个问题：前端是嵌入 Python 的领域专用语言（DSL），用以页为单位的张量（page-centric tensor）抽象来描述稀疏模式；后端与现代 serving 栈深度集成，将前端描述直接映射到 GPU 内核执行，避免额外索引转换开销。</p>

<p>　　系统进一步暴露统一接口供 AI agent 调用，形成"算法生成 → serving 环境评估 → 反馈迭代"的闭环搜索流程，类似于把神经架构搜索（NAS）的思路用到注意力稀疏模式上。实测结果：在 MLA 架构的 GLM-4.7-Flash 上，AI agent 搜索出的最优方案吞吐提升最高 <b>4.7×</b>；agent 自动生成方案在保持精度的前提下吞吐最高 <b>3.46×</b> vs 全注意力；在 229B 参数的 MiniMax-M2.7 上也实现 <b>1.37×</b> 提升（NVIDIA B200 GPU）。</p>

<h3>LadderMan：人形机器人两阶段蒸馏攀梯策略，zero-shot 真实部署并支持攀梯中操作</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05873">LadderMan: Humanoid Ladder Climbing and Manipulation</a></li>
  <li><b>信号源</b>：<b>Berkeley Pieter Abbeel</b>（合作）· Siheng Zhao（一作，USC / MIT PhD 在读）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>人形机器人研究的主流判断是：全身协调任务（爬梯、翻越障碍）和操作任务（抓取、拧螺丝）是两条训练管线，难以统一。LadderMan 打破了这个假设——<b>同一套系统能让机器人爬上梯子，同时在攀梯约束状态下完成操作任务</b>。关键不在于更大的模型，而在于结构上的解耦：先用 hybrid motion tracking 从单一参考动作学出多个专家策略覆盖不同梯子几何形状，再通过强化学习（RL）蒸馏为一个统一的基于深度的视觉运动策略（depth-based visuomotor policy）；感知端借助视觉基础模型（vision foundation models）处理深度图像，填平仿真与真实环境之间的感知鸿沟，实现无需真实数据微调的 zero-shot 迁移。这意味着"仿真中训练、现实中部署"的路线在复杂全身协调场景下可能比此前预期更快成立，工业检修、建筑施工等需要攀爬并操作的场景从原型到落地的距离正在缩短。</p>
</callout>

<p>　　人形机器人此前在梯子攀爬这类任务上面临双重挑战：梯子几何形状千变万化（横档间距、倾斜角、宽度各异），而仿真环境的深度感知与真实相机存在明显 domain gap，导致策略难以直接迁移。现有方法通常要么针对特定梯子专门设计策略，要么牺牲操作能力换取移动鲁棒性。</p>

<p>　　LadderMan 采用两阶段框架：第一阶段，hybrid motion tracking 从单一参考动作出发，为不同梯子几何形状学出多个专家攀爬策略（expert climbing policies）；第二阶段，通过混合模仿学习（hybrid imitation）与强化学习（RL）将多专家蒸馏为统一的深度视觉运动策略（depth-based visuomotor policy）。操作部分引入双智能体框架（dual-agent formulation），将"攀爬中的机器人"与"操作臂"解耦建模，独立训练操控策略，支持遥操作引导。感知端用视觉基础模型处理深度图像，大幅缩小仿真与真实的感知差距，最终实现 zero-shot 迁移到真实人形机器人硬件，并在攀梯约束条件下成功完成多种操作任务。</p>

<h3>PERSUASIONTRACE：贝叶斯网络追踪 LLM 多轮说服过程，模拟器人类相似度 81 分</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05330">PERSUASIONTRACE: A Process-Level Framework for Evaluating LLM Persuasion</a></li>
  <li><b>信号源</b>：<b>Stanford Noah D. Goodman</b> · Jared Moore（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>说服研究此前的主流做法是测量"说服前后的信念差值"，这个方法的问题在于：它告诉你结果，但不告诉你为什么某轮对话奏效，也无法区分"一开始就快被说服"和"逐步被有效论据打动"的情形。PERSUASIONTRACE 把单点测量换成了过程追踪——<b>贝叶斯网络（Bayesian network）在每轮对话后更新目标用户的潜在信念状态，而不是只看入场和出场两个时刻</b>。更值得关注的是，这套框架下的模拟器人类相似度评分达到 81 分，与真实人类参考值 80 分几乎持平，而直接用 LLM 模拟目标用户仅得 64 分。这意味着大规模评估 LLM 说服能力时，"用更好的人类模拟器"这一路线在技术上已经可行，下一步的关键是谁在控制这套基础设施，以及其输出被用于什么场景。</p>
</callout>

<p>　　说服研究与 AI 安全存在直接交叉：当 LLM 被用于社交媒体内容生成、对话式营销或政治舆论场景时，其说服机制需要能被精确追踪和审计，而不仅仅是被标记为"有效"或"无效"。PERSUASIONTRACE 基于 web 实验平台，包含三层：多轮信念报告收集工具记录对话每轮后目标用户的信念变化；说服者话轮按逻辑（logos）、情感（pathos）、权威（ethos）三类修辞维度标注；评估时以真实人类信念动态作为参考，对各类模拟器进行过程保真度评估。</p>

<p>　　核心技术是贝叶斯网络（Bayesian network）模拟目标：模型维护一个随时间演化的显式潜在信念状态，每条说服者消息触发概率更新，产生逐步演化的认知轨迹，而非在对话结束时才输出单一差值。研究发现人类被说服目标可聚类为两种多轮信念更新模式，对三类修辞策略均表现出易感性；LLM 在通用话题、个性化内容、文本和音频模态、多轮交互中均展现出显著说服效果。</p>

<h4>NF-CoT：normalizing flows 在 LLM 内部建模连续潜在思维，代码生成超越显式推理链</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.06447">NF-CoT: Continuous Latent Reasoning via Normalizing Flows in LLMs</a></li>
  <li><b>信号源</b>：<b>Apple Jiatao Gu</b>（资深合作）· Xiangjun Fu（共一，导师 Hongwei Xi）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>链式推理（chain-of-thought，CoT）的隐含假设是：模型必须把每一步中间思考转成可读 token 才能继续推理，但这在 token 消耗和推理延迟上代价极高。NF-CoT 表明这个假设可以被打破——<b>通过在 LLM 骨干网络中嵌入一个 TARFlow 风格的 normalizing flow 头，模型可以在连续潜空间里完成中间推算，只在最后才输出文本</b>，同时保持 KV cache 和左到右解码的完整工程兼容性。这对"需要密集推理链才能解决的任务"这类场景的成本预期需要重新评估——如果潜空间推理成立，推理 token 开销的下降幅度可能比多步蒸馏路线更彻底。</p>
</callout>

<p>　　当前主流大语言模型（LLM）的推理优化围绕两条路线：一是让模型"想得更多"（更长的推理链、更多计算预算），二是把大模型的推理能力蒸馏进小模型。NF-CoT 提出了第三条路：在 LLM 的自回归生成流中插入一个 normalizing flow 头，专门负责"连续思维"位置的生成；文本位置仍由原始语言模型头处理，两类头共享同一因果流（causal stream）与 KV 缓存（KV cache），彼此透明。连续思维通过显式推理链蒸馏而来，训练时可用策略梯度（policy-gradient）在潜空间直接优化推理策略。</p>

<p>　　NF 头采用 TARFlow 风格的可逆变换，将高斯潜变量映射为连续思维向量，每个连续思维位置具备精确对数似然，可用于监督蒸馏和强化学习（RL）优化。实测在代码生成任务上，NF-CoT 的通过率超过显式推理链（CoT）及现有潜空间推理（latent reasoning）基线，并显著降低中间推理 token 开销。</p>

<h4>Asuka-Bench：多轮修正 web 开发 Code Agent 评测，最强模型三轮后完成率仅 52%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05920">Asuka-Bench: Evaluating Code Agents for Web Development with Underspecified Intent</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b>（一作）· Zhendong Mao（通讯，USTC / 清华教授）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>现有代码 agent 评测假设用户需求是完整、清晰的，这与真实 web 开发场景相差很远。Asuka-Bench 把"不完整需求 + 多轮人类反馈"变成了一个可测量的闭环：Code Agent 生成代码、UI Agent 在浏览器里执行测试、User LLM 把结果翻译成自然语言反馈。<b>最强模型经过三轮修正后完成率仍只有 52%，而且模型间差距高达 38 个百分点，说明"看反馈、纠错、再生成"这条链路目前仍是代码 agent 能力的核心分水岭</b>。对代码 agent 产品来说，这意味着当前的评测分数可能严重高估了模型在真实用户使用场景中的可靠性。</p>
</callout>

<p>　　代码生成领域的主流评测（如 HumanEval、SWE-bench）针对的是"给定完整规范、一次生成"的场景。但真实软件开发中，用户通常是看到初版结果后才逐步明确需求并反馈。Asuka-Bench 以闭环多智能体架构为核心解决这个测试盲区：Code Agent 生成 web 项目，UI Agent 在浏览器渲染环境中逐条验证预定义的 expected outcomes（类似自动化 UI 测试），User LLM 将测试结果转化为自然语言修改要求传回 Code Agent，驱动下一轮修正。每个任务的初始描述刻意保持不完整，以模拟真实用户模糊需求。</p>

<p>　　在 2 种 agent 框架下评测 8 个 LLM，加权任务通过率（weighted Task Pass Rate）模型间差距达 38 个百分点；即使最强模型经过三轮修复后完成率也只有 52%。各模型在"根据反馈自我修复"的能力上差异最为显著，这一维度是当前代码 agent 系统的主要瓶颈。</p>

<h4>YouZhi-LLM：逐层自适应 GQA-to-MLA 转换压缩 KV cache，金融场景并发量提升 2.69×</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05868">YouZhi-LLM: Layer-Adaptive GQA-to-MLA Conversion for High-Concurrency Financial Inference</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b>（合作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>大多数 KV 缓存（KV cache）压缩方案是在训练后统一处理所有层，但不同 Transformer 层对注意力头数的敏感度差异很大。YouZhi-LLM 的 FreqFold 机制按每层的频率特性动态分配压缩比，再通过蒸馏和领域微调（SFT）恢复表征质量。<b>在 Huawei Ascend NPU 上，这套方案把金融高并发场景的最大并发量推到了 GPU serving 方案难以轻易复制的水平——这是"架构优化 + 国产硬件生态"双重绑定的信号</b>。对金融行业用户来说，追踪 Ascend 生态在高并发 serving 场景的能力成熟度，比单纯看模型精度更值得关注。</p>
</callout>

<p>　　金融场景对 LLM serving 的核心诉求是高并发低延迟，而 KV cache 的内存占用是限制并发量的主要瓶颈。标准多头注意力（MHA）或分组查询注意力（GQA）在并发场景下内存压力很大；多头潜注意力（MLA，Multi-head Latent Attention）通过低秩分解显著压缩 KV cache，但直接替换会造成表征损失。</p>

<p>　　YouZhi-LLM 提出逐层自适应的 GQA-to-MLA 转换框架：FreqFold 机制分析各层注意力头的频率特性，为不同层动态分配不同的压缩比；随后用知识蒸馏（knowledge distillation）从原 GQA 模型保留通用知识，再用金融领域数据做监督微调（SFT）注入领域能力。在 Huawei Ascend NPU 上通过 vLLM-Ascend 验证：YouZhi-7B 金融基准平均得分提升 <b>12.3%</b>，最大并发量提升 <b>2.69×</b>；YouZhi-14B 精度提升 <b>7.0%</b>，并发量提升 <b>2.43×</b>。层间不均匀压缩使困惑度（perplexity）退化比均匀基线降低最高 <b>35%</b>。</p>

<h4>CL-Bench：LLM agent 持续学习评测，专用记忆系统反不如朴素上下文学习（ICL）</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05661">CL-Bench: A Benchmark for Continual Learning in LLM-Based Agent Systems</a></li>
  <li><b>信号源</b>：<b>Berkeley Matei Zaharia</b>（合作）· Parth Asawa（一作，Berkeley PhD 在读，导师 Matei Zaharia）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>agent 系统研究普遍假设"专门设计的记忆管理模块能让 agent 从经验中持续进步"，CL-Bench 的结果直接挑战了这个假设：在 6 个覆盖软件工程、疾病预测、战略博弈等真实领域的评测中，专用记忆系统的持续学习增益不如朴素的上下文学习（in-context learning，ICL）。<b>这意味着"记忆架构越复杂 = 持续学习能力越强"的直觉可能是错的——当前专用记忆系统更多在过拟合即时观测而非真正提炼跨任务可复用的知识</b>。对 agentic 产品路线来说，这是一个需要重新检验的设计假设。</p>
</callout>

<p>　　持续学习（continual learning）能力——即 AI 系统能否像人类一样从序列经验中逐步进步——是 agentic AI 系统能否真正替代重复性知识工作的关键能力。现有评测通常是一次性任务，无法测试跨实例的经验积累。CL-Bench 将持续学习形式化为"有状态序列任务"：每个领域设计了共享的可学习隐结构（如代码库布局、对手策略、疾病动态），stateless 系统天然无法发现这些规律。</p>

<p>　　评估引入增益指标（gain metric）：比较同一模型在"有历史经验输入"与"无历史经验输入"两种条件下的性能差值，从而与模型本身的基础能力解耦。结果显示，在所有 6 个领域的评测中，前沿 LLM agent 的持续学习增益普遍有限；更出乎意料的是，朴素上下文学习（ICL）在大多数场景下持续学习表现优于专用记忆管理系统——后者频繁过拟合即时观测或无法跨实例复用知识。</p>

<h3>多模态智能 (Multimodal Intelligence)</h3>

<h4>Brick-Composer：三路信号训练 MLLM 积木组装，砖块选择准确率提升 3 倍、步骤成功率从 1% 升至 15%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05445">Brick-Composer: Teaching MLLMs to Assemble LEGO-Style Bricks</a></li>
  <li><b>信号源</b>：<b>Northwestern Manling Li</b>（合作）· Jiateng Liu（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>MLLM（多模态大语言模型）在静态视觉问答上已接近人类水平，但 Brick-Composer 的评测结果揭示了一个清晰的能力边界：<b>当任务需要精细空间推理加序列决策时，当前最强 MLLM 步骤级成功率不足 1%，几乎完全失效</b>。这个边界比大多数人预期的更硬——积木组装要求同时判断"选哪块"（砖块分类）和"放在哪里"（6 自由度位姿估计），两个子任务都需要精确的 3D 空间感知。三路训练信号（人类施工示范、物理世界反馈、合成数据）的组合把步骤成功率拉到 ~15%，是一个可以继续迭代的立足点，但与真实装配任务所需的可靠性仍有一个数量级的差距。对机器人操作和工业装配这类下游场景来说，"视觉空间推理 + 序列决策"的联合能力将是下一个关键卡点。</p>
</callout>

<p>　　MLLM 的空间推理能力是当前具身智能应用落地的核心瓶颈之一：模型能描述图片，但无法精确判断三维空间中物体的相对位置和可接触点（affordance）。积木组装任务将这个挑战具体化为一个可量化的闭环：给定目标 3D 模型，模型需要逐步选出正确砖块并预测每块的放置位置和朝向，形成完整的施工序列。BC-Bench 同步发布，覆盖多种积木对象设计，输出结果可程序化验证。</p>

<p>　　Brick-Composer 将任务分为砖块选择（brick selection）和砖块位姿估计（brick pose estimation）两个子任务，引入三类互补训练信号：Human Design Sparks 提供富含可接触点（affordance）信息的施工示范；World Feedback 将预测动作的结果与视觉和物理后果对齐，形成闭环修正；Synthetic Experience 利用合成数据扩展真实标注数据的分布覆盖。以 Qwen-3-8B 为基础模型微调后，砖块选择准确率提升超 <b>3 倍</b>，步骤级成功率从不足 <b>1%</b> 升至约 <b>15%</b>，完整对象组装步骤正确率达 <b>42%</b>。</p>

<h4>UniVoice：条件流匹配统一语音与歌声生成，TTS 精度与专用系统持平</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05852">UniVoice: Unified Speech and Singing Voice Generation via Conditional Flow Matching</a></li>
  <li><b>信号源</b>：<b>Hao Liu</b>（合作）· Junjie Zheng（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>语音合成（TTS）和歌声合成（SVS）长期以来被当作两个独立子领域，各自维护独立的模型和训练管线。UniVoice 的结果表明，<b>用一个共享 backbone 同时处理两类任务，精度可以达到或超过专用系统——这意味着"统一语音 backbone"的路线已经在精度上站住脚</b>。null melody token 的设计是关键机制：歌声时强制对齐 MIDI 音符，语音时替换为可学习的空旋律 token，让模型自由推断韵律。对语音生成基础设施来说，这是一个值得跟踪的信号——下一个问题是统一模型在推理延迟和音色泛化上的表现是否能跟上专用系统。</p>
</callout>

<p>　　语音和歌声在本质上共享内容（说了什么）、音色（谁的声音）两个维度，差异主要在旋律（是否需要严格对齐音高和节拍）。UniVoice 基于条件流匹配（conditional flow matching）框架，以共享的扩散 Transformer（Diffusion Transformer，DiT）为骨干网络，将生成条件分解为内容（content）、旋律（melody）、音色（timbre）三个独立因子，分别由适配各模态的编码器处理；训练数据包含 30k 小时语音和 35k 小时歌声。</p>

<p>　　歌声生成时，旋律条件由 MIDI 音符序列表示，强约束音高与节拍；语音生成时，旋律条件替换为可学习的 null melody token，让模型从语言与声学上下文中自由推断韵律。论文将 null melody token 解释为对条件流中旋律边缘化（marginalization）的近似，提供了理论支撑。在 TTS 任务上，字素错误率（PER）达 <b>5.26%</b>，与专用系统 F5-TTS（5.21%）和 CosyVoice3（5.30%）相当；在歌声生成上 PER 为 <b>16.22%</b>，优于统一基线 Vevo1.5（24.72%）。</p>

<h4>CLEAR：Drive-JEPA + 单步 VAE 漂移替代 diffusion 去噪，NAVSIM v1 自动驾驶规划 93.7 PDMS</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.06219">CLEAR: Conditional Latent Drift for End-to-End Autonomous Driving Planning</a></li>
  <li><b>信号源</b>：<b>清华 Zhiyuan Liu</b>（合作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>自动驾驶规划领域里，基于 diffusion 的轨迹生成因为多步去噪的推理延迟一直难以达到实时部署标准。CLEAR 把这个问题绕过去了——<b>用变分自编码器（VAE）潜空间中的单步条件漂移（conditional drift）取代迭代去噪，同时用微调后的小型语言模型提取场景语义来动态调度采样策略</b>，在 NAVSIM v1 基准达到 93.7 PDMS，进入当前最优序列。这套框架的信号是：大语言模型（LLM）作为"场景理解 + 策略调度"的语义路由层，而不只是端到端生成器，在自动驾驶这类实时约束强的场景里可能是更实用的接入方式。</p>
</callout>

<p>　　端到端自动驾驶规划的核心挑战是：需要同时保证多模态行为覆盖（能生成多种合理轨迹候选）和实时推理约束（单次推理延迟要足够低）。基于 diffusion 的方法多模态性好但太慢；确定性方法快但难以覆盖边缘场景。CLEAR 提出一套三模块框架：Drive-JEPA 作为视觉编码器提取视觉特征；VAE 潜空间中的单步条件漂移（conditional drift）生成多模态候选轨迹，通过条件系数 α 平衡多样性与专家精度；Qwen 3.5 0.8B 全量微调后从驾驶问答对中提取场景感知的隐藏状态（scene-aware hidden states），驱动自适应调度器（Adaptive Scheduler）选择 α 和采样数 N，以及交叉注意力（cross-attention）评分器从候选轨迹中选出最优轨迹。</p>

<p>　　自适应调度器从预定义的离散方案集合中动态选取参数，避免了固定参数对不同场景的泛化不足。整套方案无需密集几何标注，在 NAVSIM v1 基准达到 <b>93.7 PDMS</b>，达到当前最优（state-of-the-art）水平，同时满足实时推理约束。</p>

<h4>Stanford Jiajun Wu 团队：LLM 合成 weak verifier 聚合，空间布局生成质量提升 66.2%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05268">Aggregating LLM-Generated Weak Verifiers for Spatial Layout Generation</a></li>
  <li><b>信号源</b>：<b>Stanford Jiajun Wu</b>（合作）· Sharon Zhang（一作，导师 Bill Freeman）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>让 LLM 自动生成验证程序（verifier）本身并不新鲜，但这些程序往往精度有限。这篇论文的关键发现是：<b>与其让 LLM 生成一个"好"的 verifier，不如生成大量"不太好"的 verifier，再用极少量人工标注（约 10 条）学习如何把它们聚合成一个强验证器</b>——这是弱学习（weak learning / boosting）思想在 LLM 生成程序上的直接应用，F1 最高达到直接用 LLM judge 的 7 倍。对 AI 生成内容的质量控制和自动评测来说，这是一套值得关注的低人工成本路线：它把"需要高质量标注"的瓶颈从"每条输出"转移到了"学习聚合策略"这个一次性成本上。</p>
</callout>

<p>　　空间布局生成（3D 房间布局、2D 海报设计等）任务的核心挑战是缺乏高质量自动评测：人工评估成本高、LLM judge 精度有限。该研究借鉴弱监督学习（weak supervised learning）理论，提出三步流程：首先让 LLM 基于布局验证领域专用语言（DSL）合成多个不完美的 weak verifier 程序；然后用弱学习方法（类似 Snorkel 风格的 label model）从约 10 条人工标注样本中学习聚合策略，把多个 weak verifier 融合为 strong verifier；最后将 strong verifier 的自然语言反馈作为信号，对布局生成器进行迭代引导。</p>

<p>　　在 3D 房间布局和 2D 海报设计两类任务上，聚合后的 strong verifier F1 分数最高达到直接用 LLM judge 的 <b>7 倍</b>；以 strong verifier 反馈引导布局生成器后，布局质量人工评估提升最高 <b>66.2%</b>；整个聚合策略仅需约 <b>10 条</b>人工标注样本，样本效率极高。</p>

<h4>生物医学世界模型范式：干预条件动态建模支持虚拟细胞、虚拟患者与手术模拟</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.05925">Biomedical World Models: Learning State Representations and Intervention-Conditioned Dynamics</a></li>
  <li><b>信号源</b>：<b>NTU Ziwei Liu</b>（合作）· <b>DeepMind（前）Ting Chen</b>（合作）· Guangyu Wang（共一）· Yong Li（通讯）</li>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>

<p>　　论文梳理了这一范式在四类应用场景中的角色：作为数据引擎（虚拟细胞生成训练数据）、作为实验平台（虚拟类器官测试药物）、作为临床规划器（虚拟患者模拟治疗方案）、作为手术模拟器（外科技能训练和规划）。文章同时识别了实现该范式所需的数据基础设施、评测基准（benchmark）、安全约束和治理框架等关键缺口。本文为展望性框架论文，未报告具体实验数字。</p>
