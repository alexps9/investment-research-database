- 1M context window：支持超长上下文输入，覆盖大规模文档处理和长时任务
- 长时运行任务 SOTA：Greg Brockman 声称在 long-running tasks 和 cursorbench 上均排名第一

**技术路径**

原文未披露具体技术架构变更。从产品描述推断，核心改进方向在于推理时的 token 利用效率优化和多步骤任务规划能力。Pro 版本可能采用更大规模计算或更精细的推理策略。

**行业意义**

GPT-5.5 的发布标志着 OpenAI 进入"半代迭代"节奏，在主版本之间通过效率和能力的增量提升保持竞争压力。token 效率的显著改善直接降低了开发者的 API 成本，agentic 能力的强化则进一步推动 LLM 从问答工具向自主执行工具的演进。

### UC Berkeley 提出 RLT 框架：通过 RL token 接口对 VLA 模型进行高效 online RL 微调

- **信号源**：Sergey Levine @ UC Berkeley
- **链接**：[RLT: RL Tokens for VLA Fine-Tuning](http://arxiv.org/abs/2604.23073v1)

**认知提取**

研究者在 VLA 模型中插入一个紧凑的"RL token"表征作为 RL 接口，既保留预训练知识，又允许小型 actor-critic head 对动作进行 online 精调——这类似于在成熟专家大脑中安装一个小型快速反馈回路，而非重新训练整个大脑。实验显示在精密插接任务上速度提升最高 3x，部分任务甚至超过人类遥操作速度。

**论文摘要**

预训练 VLA（Vision-Language-Action）模型在机器人操作中展现出强大的泛化能力，但在精密操作任务中仍存在精度和速度瓶颈。传统 online RL 微调方法要么需要对整个大模型进行梯度更新（计算成本极高），要么需要从头训练策略（丧失预训练知识）。RLT 提出了一种折中方案：从 VLA 内部提取一个低维"RL token"——一种保留任务语境信息的紧凑表征——作为小型 actor-critic head 的输入接口，仅对该 head 进行 online RL 训练，同时通过 policy anchoring 约束防止偏离预训练先验。

**核心方法**

- **方法框架**：RLT 框架分两步：(1) 适配预训练 VLA，提取一个紧凑的"RL token"——一种保留任务相关知识的读出表征；(2) 在该 token 之上训练轻量级 actor-critic head，通过 online RL 对动作进行精化，同时以策略锚定（policy anchoring）防止偏离预训练 VLA 的先验知识。整体方法与 VLA 骨干解耦，适用于大型 VLA 模型。
- **技术细节**：RL token 是从 VLA 内部提取的低维向量，相当于 VLA"告诉" actor-critic head 当前任务语境的摘要；actor-critic head 规模极小，仅需在线交互数据就可收敛。策略锚定类似于 RLHF 中的 KL 约束，防止 RL 过拟合导致灾难性遗忘。

**实验成果**

- 提出 RL token 机制，将 VLA 的紧凑表征暴露为 online RL 接口，保留预训练知识的同时大幅降低 RL 微调成本
- 在四项真实机器人精密操作任务（螺丝安装、扎带、充电器插入、网线插入）中，任务最难阶段速度提升最高 3x
- 仅需数分钟至数小时真实机器人练习，成功率显著提升，部分任务速度超越人类遥操作

**总结与反思**

- **结果总结**：RLT 在四项真实机器人精密插接任务上实现了最高 3x 的速度提升和显著的成功率提升，并在数小时内完成收敛，验证了 RL token 作为 VLA 在线微调接口的有效性。

### ChronoScope：评估 LLM 多轮对话中时间范围一致性的百万级 Benchmark

- **信号源**：Steven Johnson 等 @ arXiv
- **链接**：[ChronoScope](http://arxiv.org/abs/2604.23051v1)

**认知提取**

该研究关注多轮对话中的"时间范围稳定性"问题——当用户在对话中建立了特定时间背景后，模型是否能在后续轮次中正确延续或切换该时间假设。结果发现主流模型普遍存在向"当前时刻"漂移的倾向，这类时序一致性失误随对话长度加剧。

**论文摘要**

ChronoScope 基于 Wikidata 中的时间限定事实，通过程序化生成构建超百万条多轮问题链，覆盖隐式时间延续、显式范围切换、跨实体时间迁移和长时间轨迹四类场景。评估发现即使在 oracle context 条件下，主流 SOTA 模型仍频繁违反 temporal scope stability，揭示了单轮事实准确性与多轮时序推理之间的系统性差距。

**核心方法**

- **方法框架**：ChronoScope 基于 Wikidata 中的时间限定事实，通过程序化生成构建超百万条多轮问题链，每条链包含显式时间范围建立轮次与后续省略时间引用的追问轮次，系统覆盖四类场景：隐式时间延续（implicit carryover）、显式范围切换（explicit scope switching）、跨实体时间迁移（cross-entity transfer）和长时间轨迹（longer temporal trajectories）。
- **技术细节**：问题链的生成完全基于 Wikidata 图谱中的结构化时间标注，通过模板组合确定性生成，确保 ground truth 可验证。评估时对比模型在携带完整上下文（oracle context）与仅有对话历史两种条件下的表现，以此区分记忆失效与推理失效。

**实验成果**

- 提出 ChronoScope benchmark，含超 100 万条确定性生成的多轮问题链，覆盖 implicit carryover、scope switching 等场景
- 主流 SOTA 语言模型在受控多轮设置中频繁违反 temporal scope stability，倾向于偏向现实时间假设
- 时序一致性失败随对话轮数增加而加剧，且在 oracle context 条件下仍然存在，揭示单轮事实准确性与多轮时序推理之间的差距

**总结与反思**

- **结果总结**：实验表明现有 SOTA 语言模型在多轮时序推理中存在系统性漂移缺陷，对话越长失败率越高，仅凭单轮事实准确性无法预测多轮时序一致性表现。

### Importance Sampling 估算 LLM 有害输出尾部概率：效率提升 10-20x

- **信号源**：He He 何河 @ arXiv
- **链接**：[Importance Sampling for LLM Safety](http://arxiv.org/abs/2604.22167v1)

**认知提取**

当 LLM 每天被查询数十亿次时，即使有害输出概率极低（如 10^-4），绝对数量仍可能很大。这项研究将经典统计学中的 importance sampling 技术引入 LLM 安全评估，通过临时构建"风险放大版"模型来高效探测尾部有害行为——类似于用模拟极端天气的工具来预测罕见洪水，而非等待洪水自然发生。

**论文摘要**

研究者针对目标模型构建若干"unsafe 版"模型变体（通过 fine-tuning 或 steering 使有害输出更高概率出现），以这些变体作为 proposal distribution 进行 importance sampling，再用 importance weight 校正回目标分布，从而在极少样本下得到准确的有害输出概率估计。在 misuse 和 misalignment 基准上，仅 500 个样本即可估算量级约为 10^-4 的有害输出概率，相比暴力 Monte Carlo 方法效率提升 10-20x。

**核心方法**

- **方法框架**：方法核心是将 importance sampling 应用于 LLM 输出分布的尾部风险估计。研究者针对目标模型构建若干"unsafe 版"模型（即通过 fine-tuning 或 steering 使有害输出更高概率出现的模型变体），以这些变体作为 proposal distribution 进行采样，再用 importance weight 校正回目标分布，从而在较少样本下得到准确的有害输出概率估计。
