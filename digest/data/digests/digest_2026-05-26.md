<h3>AutoResearch：AI 驱动科研自动化全链路综述，提出 Vibe Research 谱系与五维评估体系</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.23204">AutoResearch: A Survey on AI-Driven Scientific Research Automation</a></li>
  <li><b>信号源</b>：<b>Northwestern Manling Li</b> · <b>Microsoft Jianfeng Gao</b></li>
</ul>
<callout emoji="🧬">
  <p><b>Our Insights</b></p>
  <p>过去两年 AI 科研工具的讨论基本停留在"某个环节比人快多少"——文献综述、代码生成、实验设计各自孤立。这篇综述的核心贡献是把视角从单点工具拉升到全链路工作流：AutoResearch 谱系将人工主导（Vibe Research）到 AI 自主科研之间的连续空间系统化，并用 novelty / validity / impact / reliability / provenance 五维指标框架评估各类系统。</p>
  <p>影响在两条线上：(1) <b>AI 科研自动化平台</b>（如 Sakana AI Scientist、FutureHouse 等）的估值叙事将从"提速"转向"全流程闭环能力"，评估维度会向 provenance 追溯和跨域鲁棒性收紧；(2) 综述明确指出 AI 在具身实验、伦理审查、机构问责场景仍无法独立运作——这意味着"全自动 AI 科学家"作为短期投资叙事存在高估风险，<b>混合人机协作（Vibe Research 区间）的工具型公司</b>在近 2-3 年更有落地可能。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>排查已接触的 AI 科研自动化标的，评估其产品能力是否覆盖完整五大工作流条件（尤其 provenance tracking 和 feedback/validation 环节），能力碎片化的标的需下调估值预期。</li>
    <li>关注 Manling Li 团队后续是否配套发布 AutoResearch benchmark——若发布，将成为赛道标准化评估的重要节点，提前跟进。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>综述论文以 AutoResearch 为核心框架，系统梳理 AI 驱动科研自动化的全链路进展，覆盖文献挖掘与研究 grounding、假设生成与规划、实验与工具使用、反馈验证与评审、报告与知识传播五大工作流条件。论文引入 Vibe Research 概念，特指基于 prompt 辅助与人工验证的人机协作区间，与更高自主度的 AI 主导模式相区分，并构建 novelty / validity / impact / reliability / provenance 五维评估体系，指出当前 AI 在结构化可执行领域更可信，在具身或需机构问责场景仍有明显局限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：综述形式构建 AutoResearch 分析框架，按五大工作流条件系统梳理现有 AI 科学家系统、混合主动协作框架、benchmark、领域部署及开源基础设施。</li>
  <li><b>技术细节</b>：引入 Vibe Research 概念，通过控制权、证据链、执行、验证、问责五维度分析各系统如何在工作流中重新分配人机责任。</li>
  <li><b>主结果</b>：提出 AutoResearch 谱系，涵盖从 Vibe Research（人工主导）到 AI 完全主导的全范围。</li>
  <li><b>关键结果点</b>：综述揭示当前 AI 科研自动化系统碎片化严重，在证据保存、可重复性、跨领域鲁棒性方面存在共性短板；AI 自主性在具身或伦理场景仍难以独立运作。</li>
</ul>

<h3>EvalVerse：将电影专业美学系统化注入 VLM，构建电影级视频生成评测框架</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.23271">EvalVerse: A Comprehensive Evaluation Framework for Professional Cinematic Video Generation</a></li>
  <li><b>信号源</b>：<b>Songlin Yang</b>（共一） · Anyi Rao（通讯）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>视频生成 benchmark 长期停留在"prompt 指令遵循"维度——模型能不能生成对应画面。这是一个够用但远不够的评估框架，因为真正决定商业价值的是"电影级质量"：构图、节奏、表演、镜头语言。EvalVerse 的突破是把专业影视人士的主观审美判断结构化，再通过 expert-calibrated fine-tuning（专家校准微调）注入 VLM（视觉语言模型），使评测系统具备 Chain-of-Thought（链式推理）推理输出评分理由的能力。</p>
  <p>这件事的格局意义在于：凭借"专业影视知识结构化 + 大规模标注 + VLM 微调"这套路径，EvalVerse 实际上是在为视频生成领域建立 reward model 基础设施。对<b>视频生成创业公司</b>（Runway、Pika、国内的即梦、可灵等）而言，这类评测框架一旦被采纳为行业标准，将倒逼它们在 cinematic quality 维度上公开对标——而非仅比拼 FID 或用户评分。<b>拥有专业影视数据标注资源的公司</b>护城河将因此增厚。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 EvalVerse 是否开放 benchmark 数据集及评测排行榜——若开放，可作为视频生成赛道横向对比的客观锚点，用于尽调已接触标的。</li>
    <li>评估国内视频生成标的（可灵、即梦等）在 cinematic goodness 维度的真实能力，判断其产品定位是否支撑"电影级"叙事。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>EvalVerse 提出面向专业电影级视频生成的评测框架，核心创新是把专业影视人士的主观审美判断系统化数字化：先按前期制作/拍摄/后期三阶段构建电影专业分类体系，再通过大规模人工专家标注建立校准数据集，最后以 expert-calibrated fine-tuning 将这些信号注入 VLM，使评测模型在输出评分时同步输出 Chain-of-Thought 推理过程。评测范围覆盖多镜头序列与音视频整合等复杂场景，显著超越现有 benchmark 的单镜头 prompt-following 局限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：三层架构——知识组织层（电影专业分类体系）/ 数据层（专家标注校准数据集）/ 模型层（expert-calibrated fine-tuning 注入 VLM）。</li>
  <li><b>技术细节</b>：VLM 评测时输出 Chain-of-Thought 推理过程而非直接打分，同时保持与传统"rightness"指标的兼容性，提升可解释性与可信度。</li>
  <li><b>主结果</b>：构建覆盖前期/拍摄/后期全流程的评测分类体系，填补传统 benchmark 忽视的 cinematic quality 维度。</li>
  <li><b>关键结果点</b>：评测范围拓展至多镜头序列与音视频整合，为视频生成领域 reward model 和 evaluator agent 提供基础评测基础设施。</li>
</ul>

<h3>Fast-dDrive：Block-Diffusion VLA 自动驾驶规划，nuScenes L2 误差 0.32m，推理吞吐提升 12×</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.23163">Fast-dDrive: Block-Diffusion VLA for Efficient and Accurate Autonomous Driving</a></li>
  <li><b>信号源</b>：<b>MIT Song Han</b>（合作） · Kewei Zhang（共一） · Jie Jin Wang（共一）</li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>AR（自回归）模型的 KV cache（键值缓存）效率和 diffusion 模型的双向精炼能力此前被认为是互斥的——两套架构各有优势但难以融合。Fast-dDrive 凭借 block-diffusion 架构解决了这个卡点：语义单元内部做双向 diffusion 精炼，单元之间保持严格因果顺序，从而同时复用 KV cache 优势并规避全序列 diffusion 的"逻辑泄漏"。</p>
  <p>影响落在两条线：(1) <b>自动驾驶端到端栈公司</b>（Wayve、Waabi 以及国内的文远知行、元戎启行等）的 VLA 路线可行性进一步获验证——nuScenes L2 误差 <b>0.32m</b> 叠加 <b>12×</b> 吞吐提升，意味着高精度 VLA 的车端实时部署障碍正在系统性降低；(2) 依赖传统模块化感知-规划-控制架构的供应商差异化空间被压缩，端到端路径的估值叙事将持续强化。MIT Song Han 团队与 SGLang 的深度整合也表明，<b>推理优化框架</b>已成为 VLA 落地的关键使能层。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>跟进 Fast-dDrive 是否有车企或 Tier-1 合作方参与，若有商业化意向，可作为 Song Han 团队下一个产业化节点追踪。</li>
    <li>对已接触的自动驾驶端到端标的，评估其推理部署栈是否已集成 SGLang 类优化框架——未集成的在成本侧将面临显著劣势。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Fast-dDrive 提出 block-diffusion VLA（视觉-语言-动作）框架用于自动驾驶端到端规划，将输出序列切分为语义单元，单元内部进行双向 diffusion 精炼、单元之间保持严格因果顺序，结合 Scaffold Speculative Decoding 和基于单一共享前缀 KV cache fork 多路径采样的低开销 test-time scaling，在 nuScenes 上平均 L2 误差降至 0.32m（基线提升 22%），结合 SGLang 后推理吞吐相比 AR baseline 提升 12×，在 WOD-E2E 测试集上达到 diffusion VLA SOTA 精度。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：block-diffusion 架构——结构性 token 冻结为 section scaffold，单元内双向 diffusion 精炼 + 单元间严格因果序；section-aware 训练策略针对安全关键规划场景优化。</li>
  <li><b>技术细节</b>：Scaffold Speculative Decoding 利用冻结结构 token 作为草稿骨架减少 diffusion 步数；多路径轨迹采样（N rollouts）共享同一前缀 KV cache，以接近边际成本获得方差压缩效果。</li>
  <li><b>主结果</b>：WOD-E2E 测试集 SOTA ADE@3s 和 ADE@5s，diffusion-based VLA 中最高 RFS；nuScenes 平均 L2 误差 0.32m，提升 22%。</li>
  <li><b>关键结果点</b>：结合 SGLang 集成后推理吞吐相比 AR baseline 提升 12×，大幅缩短高容量 VLA 向车端实时部署的距离。</li>
</ul>

<h3>xAI 推出 Grok Build Beta：Plan Mode + 多模态生成 + CLI 编排，面向 SuperGrok 用户全面开放</h3>
<ul>
  <li><b>信号源</b>：<b>xAI</b> 官方</li>
  <li><b>原文</b>：<a href="https://x.com/xai/status/2058973760708091907">x.com/xai/status/2058973760708091907</a></li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>

<h4>Google 论文：LLM 应通过元认知（Metacognition）实现"忠实不确定性"</h4>
<ul>
  <li><b>信号源</b>：<b>Rohan Paul</b>（转述 Google 论文）</li>
  <li><b>原文</b>：<a href="https://x.com/rohanpaul_ai/status/2059040056976032121">x.com/rohanpaul_ai/status/2059040056976032121</a></li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p><b>Google</b> 将 hallucination（幻觉）问题重新定义为"过度自信"而非"知识缺失"——模型目标应是区分"我知道"与"我在猜测"，而非追求完美准确率。"utility tax"概念点出了产品设计的真实困境：消除错误答案必须拒绝大量正确答案，导致商业产品偏向自信。</p>
  <p>这一框架若被采纳为 agentic 系统设计标准，将直接影响<b>AI agent 可靠性评估</b>的标准体系，对投研团队评估 agent 标的护城河的方法论也需相应更新。</p>
</callout>
<p>Google 论文提出 metacognition（元认知）框架，将 hallucination 本质定义为过度自信而非知识不足，目标应转向 faithful uncertainty（忠实不确定性）——模型语言输出须与内部置信度对齐。论文引入"utility tax"概念解释模型为何倾向于自信，并指出在 agentic 场景下，不确定性应作为触发验证或停止行动的决策信号。</p>
