### NVIDIA Vera Rubin NVL72：Agentic AI Inference 成本降至十分之一

- **链接**：[Dell Technologies World 主题演讲](https://blogs.nvidia.com/blog/dell-technologies-agent-enterprise-ai/)

> **Our Insights**
>
> 当前企业部署 agentic AI 的核心瓶颈不在模型能力，而在 inference 的单位经济性——多 agent 编排、tool calling、长 context 交互使得 token 消耗量级比单轮对话高出数倍至数十倍，inference 成本成为大规模商业化的硬天花板。NVIDIA 通过 Vera Rubin NVL72 将 agentic inference 成本压缩至上一代 1/10、sandbox 延迟降低 50%，本质上是在硬件层面为"agent 密集调用"这一新工作负载模式定制经济模型——这意味着此前因成本过高而无法落地的多 agent 企业场景（复杂工作流编排、实时数据查询 agent）将在 12-18 个月内具备商业可行性。对依赖 inference-time compute 做差异化的 agentic startup 而言，硬件侧的成本下降既是利好（TAM 扩大），也是威胁（纯成本优势被抹平，需转向 workflow 层壁垒）。
>
> **Potential To-dos**
>

> - 追踪 inference 成本下降后，enterprise AI middleware 层（agent orchestration / observability）的投资窗口是否加速打开——这是下一个 12 个月的潜在结构性机会

NVIDIA CEO Jensen Huang 在 Dell Technologies World 发表演讲，宣布下一代 Vera Rubin NVL72 平台将 agentic AI inference 成本降至现有方案的十分之一，agent sandbox 运行速度提升 50%，企业数据查询响应提速 3 倍。该平台将通过 Dell AI Factory 渠道面向企业级客户交付，明确针对多 agent 工作负载场景定制。

### NVIDIA 首款自研 CPU Vera 完成首批交付，定位 AI Agent 工作负载

- **链接**：[Vera CPU Delivery](https://blogs.nvidia.com/blog/vera-cpu-delivery/)
- **信号源**：**NVIDIA 官方博客**

> **Our Insights**
>
> NVIDIA 长期以 GPU 为核心产品，CPU 领域此前由 Intel/AMD 主导，NVIDIA 仅通过 Grace 做过 ARM 架构的初步尝试。Vera 作为其首款明确定位"AI Agent 工作负载"的自研 CPU，首批客户即为 Anthropic、OpenAI、SpaceXAI 和 Oracle——全部是 inference 密集型的头部买家。这标志着 NVIDIA 从"卖加速卡"向"定义整个 AI 计算节点架构"的战略跃迁：CPU+GPU+NVLink 全栈自研意味着服务器 OEM（Dell、HPE、Supermicro）的议价空间进一步收窄，而 Intel/AMD 在 AI 数据中心 CPU 的份额将面临结构性侵蚀。对一级市场而言，围绕 NVIDIA 生态构建的 AI infra startup（冷却、网络、调度）短期受益于平台锁定效应加深；但同时需警惕 NVIDIA 自身向软件栈延伸的可能性。
>
> **Potential To-dos**

### OpenAI Codex 进入企业本地部署：与 Dell 合作支持混合云场景

- **链接**：[Dell Codex Enterprise Partnership](https://openai.com/index/dell-codex-enterprise-partnership)
- **信号源**：**OpenAI 官方** · **Dell Technologies**

> **Our Insights**
>
> AI coding agent 此前以纯云端 SaaS 形态为主（GitHub Copilot、Cursor、Codex Cloud），金融、国防、医疗等数据敏感行业一直无法采用。OpenAI 选择与 Dell 合作将 Codex 推入 on-premise / hybrid 部署，说明企业 AI coding 的需求已从"是否要用"进入"如何安全地用"阶段。这对纯云端交付的 AI coding startup（Cursor、Augment Code 等）形成直接渠道压力——OpenAI 借助 Dell 的企业销售网络可以触达大量中大型企业 IT 采购决策者。但私有化部署意味着模型更新频率受限、定制化成本上升，垂直场景的 coding agent（如专攻 legacy 代码库迁移、合规审计）仍有差异化空间。
>
> **Potential To-dos**
>
> - 对已投的 AI coding / developer tool 公司，评估其是否具备 on-premise 交付能力——这正在从加分项变为必选项
> - 追踪 Dell 渠道的 Codex 定价模型和 SLA 条款，判断 OpenAI 在企业 market 的 GTM 策略是否会挤压中间层 ISV
> - 主动接触做垂直场景 coding agent 的早期项目（legacy migration、合规审计、特定语言专精），这些是 OpenAI 通用 Codex 短期难以渗透的差异化空间

**摘要**

OpenAI 与 Dell 达成合作，将 Codex AI 编程 Agent 引入企业混合云和本地部署环境，支持安全的企业级软件开发工作流，面向对数据驻留和合规有严格要求的企业客户。

**方法与结果**

- **合作架构**：OpenAI Codex × Dell 企业销售网络与硬件基础设施
- **部署形态**：on-premise / hybrid cloud，支持数据驻留与合规要求
- **目标客户**：金融、国防、医疗等数据敏感行业的企业 IT 部门
- **市场意义**：AI coding agent 从纯云端 SaaS 拓展至私有化部署，企业 GTM 渠道升级
