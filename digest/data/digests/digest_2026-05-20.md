<h3>optimize_anything：用 LLM 搜索替代专门算法的通用文本优化框架</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19633">optimize_anything: LLM-based Universal Text Parameter Optimization</a></li>
  <li><b>信号源</b>：<b>MIT Omar Khattab</b> · Microsoft Lakshya A Agrawal（一作）· Berkeley Matei Zaharia（资深合作）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>几乎所有优化问题都依赖专门设计的算法——遗传算法处理调度、手工调优 CUDA kernel、专用 NAS 探索架构。optimize_anything 挑战了这一前提：只要问题能被表达为"文本产物 + 评分函数"，一套 LLM search 框架就能跨领域替代这些专门算法。实验结果相当有力——<b>ARC-AGI 准确率从 32.5% 跃升至 89.5%</b>，<b>87%</b> 的生成 CUDA kernel 性能达到或超越 PyTorch，云调度成本降低 <b>40%</b>，六类任务全部超越领域专门算法，包括超越 AlphaEvolve 的圆形填充解。</p>
  <p>认知上的关键转变：优化问题的"专业壁垒"在 LLM 搜索面前正在被系统性拆解。多任务搜索通过跨任务知识迁移进一步放大了优势，且收益随相关任务数增加而扩大——这意味着数据积累本身就是护城河。</p>
  <p>对<b>一级市场</b>而言，这直接影响依赖"专业 AI 优化工具"为卖点的垂直 SaaS 标的——当一个通用框架开源且效果持平或超越时，专门工具的差异化空间将显著收窄。同时，<b>Databricks</b> 生态（Khattab 同时在 Databricks 任 Research Scientist）有望将该框架整合进数据/ML 工作流产品，是值得跟踪的商业化节点。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>梳理现有投资组合中依赖"专门优化算法"为核心壁垒的标的，评估 optimize_anything 类通用框架对其护城河的替代风险程度。</li>
    <li>跟踪 Databricks 是否将 GEPA/optimize_anything 框架整合至 MLflow 或 MosaicAI 产品线，作为 Databricks 在 agentic ML ops 赛道布局的观察指标。</li>
    <li>关注 Khattab 在 MIT 的下一篇工作方向——其 DSPy → optimize_anything 的路径指向"自动化 ML pipeline"，是否会延伸至 AutoML 或 agent orchestration 商业化值得提前布局。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>optimize_anything 将优化问题统一建模为"给定文本产物与评分函数，用 LLM 迭代搜索改进方案"，支持单任务、多任务跨迁移和泛化三种模式。框架开源，属于 GEPA 项目，在 ARC-AGI、CUDA kernel 生成、云调度算法等六类跨领域任务上均达到或超越专门领域算法，其中 ARC-AGI 准确率接近三倍提升，验证了 LLM text optimization 作为通用问题求解范式的可行性。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：将优化问题抽象为"文本产物 + 评分函数"二元组，LLM 作为搜索引擎迭代提出候选改进方案；支持三种模式：单任务搜索、带跨问题迁移的多任务搜索、对未见输入的泛化。</li>
  <li><b>技术细节</b>：每轮迭代中 LLM 接收当前文本产物、历史评分及可选的 actionable side information（如失败案例分析），生成新版本候选；消融实验显示带有可操作辅助信息的反馈显著加速收敛。</li>
  <li><b>主结果</b>：ARC-AGI 准确率从 32.5% 提升至 89.5%；87% 生成 CUDA kernel 性能达到或超越 PyTorch；云调度算法成本降低 40%。</li>
  <li><b>关键结果点</b>：多任务搜索通过跨任务迁移优于独立优化，收益随相关任务数增加而扩大；六类任务上均超越专门领域算法，包括超越 AlphaEvolve 的圆形填充（n=26）解。</li>
</ul>

<h3>CODA：将 Transformer 非注意力算子融合进 GEMM epilogue 的 GPU 编程抽象</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19269">CODA: GEMM-Epilogue Abstraction for Transformer Non-Attention Operators</a></li>
  <li><b>信号源</b>：<b>Princeton Tri Dao</b> · Princeton Han Guo（一作）· Princeton Jack Zhang（合作）</li>
</ul>
<callout emoji="⚙️">
  <p><b>Our Insights</b></p>
  <p>LLM 训练和推理的 GPU 效率瓶颈，长期被 FlashAttention 解决了注意力部分，但 normalization、residual、activation 等非注意力算子的显存读写开销一直是"剩余的拖累"。CODA 把这个问题系统化解决：趁 GEMM 输出 tile 还驻留片上 SRAM 时，顺手把后续计算做完——避免了中间张量写回 global memory 再读取的往返开销。</p>
  <p>更值得关注的是 CODA 的可组合 epilogue 原语设计：它不只是一个手工优化 kernel，而是一套让 <b>LLM 也能自动生成高性能 kernel</b> 的编程接口。这意味着 AI 辅助编译器的落地门槛在降低——原本需要深度 CUDA 专家才能做的 operator fusion，正在变成可被 LLM 自动化的流水线任务。</p>
  <p>对 <b>AI infra</b> 赛道的投研意涵：依赖"CUDA 专家手工优化"作为核心壁垒的 inference 优化初创，其差异化空间将被系统性压缩；而提供"AI 辅助 kernel 生成"工具链的方向（如 compiler-as-a-service）值得重新评估其时间窗口。<b>Tri Dao</b> 在 Princeton 的持续工作（从 FlashAttention 到 Mamba 再到 CODA）构成了一条清晰的 infra 创新主线，是跟踪商业化动向的关键节点。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈以"AI 辅助 kernel 自动生成 / AI compiler"为方向的初创团队，CODA 的开源将是该赛道产品能力的重要参照基线。</li>
    <li>评估已投 inference 优化标的的护城河是否依赖手工 CUDA 调优——若是，需重新审视其在 LLM 自动生成 kernel 趋势下的竞争格局。</li>
    <li>跟踪 Tri Dao 团队是否将 CODA 整合进 Flash 系列（FlashAttention / FlashLinearAttention），作为其下一轮技术商业化的信号。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>CODA 提出将 Transformer block 中的 memory-bound 非注意力算子（normalization、activation、residual 等）建模为 GEMM-plus-epilogue 程序：GEMM mainloop 保持不变，epilogue 阶段在输出 tile 仍驻留片上 SRAM 时执行缩放、规约、逐元素变换和累加等操作。框架提供一组有限但可组合的 epilogue 原语，使用户或 LLM 可通过组合原语描述复杂算子，无需手写完整 CUDA kernel，人工与 LLM 自动生成版本均达到高性能。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：将 Transformer block 的非注意力计算建模为 GEMM-plus-epilogue 程序；epilogue 阶段在输出 tile 驻留片上 SRAM 时执行，覆盖标准 Transformer block 前/后向几乎全部非 attention 计算。</li>
  <li><b>技术细节</b>：关键在于"代数重参数化"——LayerNorm 均值/方差规约、残差加法、activation 函数等可被重新推导为对 GEMM 输出 tile 的局部操作，避免中间张量写回 global memory；暴露 scaling、reduction、pairwise transformation、accumulation 四类可组合 epilogue 原语。</li>
  <li><b>主结果</b>：人工与 LLM 自动生成的 CODA kernel 在代表性 Transformer workload 上均实现高性能，验证 GEMM-epilogue 编程路径的可行性。</li>
  <li><b>关键结果点</b>：GEMM-epilogue 抽象在保持框架级开发效率的同时能逼近手工优化 kernel 的硬件效率；LLM 自动生成路径成立，为 AI 辅助 kernel 编译开辟实用路径。</li>
</ul>

<h4>optimize_anything：LLM 搜索作为通用文本优化求解器</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19633">optimize_anything: LLM-based Universal Text Parameter Optimization</a></li>
  <li><b>信号源</b>：<b>MIT Omar Khattab</b> · Microsoft Lakshya A Agrawal（一作）· Berkeley Matei Zaharia（资深合作）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>见头条详述。核心是"文本 + 评分函数 = 通用优化问题"这一抽象对专门领域算法的系统性替代，六类任务全部超越领域专门算法，ARC-AGI 准确率从 <b>32.5% 升至 89.5%</b>，CUDA kernel 生成 <b>87%</b> 达到或超越 PyTorch。多任务搜索的跨任务迁移优势随相关任务数增加而放大，构成数据飞轮效应。对 <b>Databricks</b> 商业化路径和垂直 SaaS 差异化壁垒均有直接影响。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>梳理依赖专门优化算法为壁垒的垂直 SaaS 标的，评估替代风险。</li>
    <li>跟踪 GEPA/optimize_anything 框架在 Databricks 产品线的整合进展。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>optimize_anything 将优化问题统一建模为"给定文本产物与评分函数，用 LLM 迭代搜索改进方案"的三模式框架，在 ARC-AGI、CUDA kernel 生成、云调度等六类跨领域任务上均达到或超越专门领域算法。框架开源，支持多种后端，属于 GEPA 项目。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：将优化问题抽象为"文本产物 + 评分函数"二元组，LLM 作为搜索引擎迭代提出候选改进；支持单任务、多任务跨迁移、未见输入泛化三种模式。</li>
  <li><b>技术细节</b>：每轮迭代中 LLM 接收当前产物、历史评分及可选 actionable side information（失败案例分析）；消融实验表明可操作辅助信息显著加速收敛。</li>
  <li><b>主结果</b>：ARC-AGI 准确率从 32.5% 提升至 89.5%；87% 生成 CUDA kernel 性能达到或超越 PyTorch；云调度成本降低 40%。</li>
  <li><b>关键结果点</b>：多任务搜索优于独立优化，收益随相关任务数增加扩大；超越 AlphaEvolve 圆形填充（n=26）解。</li>
</ul>

<h4>PEEK：为 LLM Agent 缓存 Orientation Knowledge 的 Context Map 系统</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19932">PEEK: Persistent Orientation Knowledge for LLM Agents in Long-Context Tasks</a></li>
  <li><b>信号源</b>：<b>MIT Omar Khattab</b> · ETH Zhuohan Gu（一作）· MIT Samuel Madden（资深合作）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>LLM agent 在反复操作同一文档库或代码库时效率低下——每次都从头"定位"，缺乏对上下文结构的持久记忆。PEEK 把这个问题拆解为工程可解的三模块：Distiller 提取线索、Cartographer 写入结构化 context map、Evictor 按优先级淘汰低价值条目。结果是 long-context 任务准确率提升 <b>6.3–34.0%</b>，迭代次数减少 <b>93–145 次</b>，成本比 SOTA 框架 ACE 低 <b>1.7–5.8x</b>。</p>
  <p>这件事的投研意涵指向 <b>agentic coding 与 document processing 初创</b>：PEEK 类技术直接影响"agent 在长周期任务中的成本可控性"——这是企业客户购买决策的核心敏感点。已验证在 <b>OpenAI Codex</b> 生产级 coding agent 上有效，说明该技术具备真实部署价值而非仅 benchmark 刷分。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 Databricks/MIT 是否将 PEEK 整合进企业级 agentic 产品，作为 long-context agent 成本优化的商业化节点。</li>
    <li>对比已投 agentic 标的的上下文管理方案，评估是否存在用 PEEK 类技术降低 token 消耗从而改善单位经济模型的机会。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PEEK 为 LLM agent 在重复访问长上下文场景（如文档库、代码库）中缓存"方向感知知识"（orientation knowledge），以固定 token 预算的 context map 形式常驻 prompt。系统由 Distiller、Cartographer、priority-based Evictor 三模块协同维护，在 long-context 推理和信息聚合任务上显著提升准确率、降低迭代次数和推理成本，跨模型和 agent 架构均有效。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：orientation knowledge 以固定 token 预算的 context map 形式常驻 agent prompt；Distiller 从推理轨迹提取可迁移知识，Cartographer 转化为结构化编辑，priority-based Evictor 在预算约束下执行淘汰。</li>
  <li><b>技术细节</b>：Distiller 监听推理轨迹抽取实体/schema/常量，Cartographer 以结构化 diff 形式写入 context map，Evictor 按优先级淘汰低价值条目维持 map 大小恒定。</li>
  <li><b>主结果</b>：long-context 推理与信息聚合任务上相比强基线提升 6.3–34.0%，同时减少 93–145 次迭代；相比 SOTA ACE 成本降低 1.7–5.8x，solving rate 提升 6.0–14.0%，rubric accuracy 提升 7.8–12.1%。</li>
  <li><b>关键结果点</b>：context map 方法泛化性强，适用于多种 LM 和 agent 架构，包括 OpenAI Codex 生产级 coding agent。</li>
</ul>

<h4>预训练数据配比实验：代码提升推理能力的真正来源是结构化推理轨迹而非可执行代码</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19762">Code vs. Structured Reasoning Traces: Disentangling the Sources of Reasoning Gains in Pretraining</a></li>
  <li><b>信号源</b>：<b>OSU Kai Zhang</b> · ETH Yuze Zhao（一作）· ETH Enhong Chen（资深合作）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>过去两年，"在预训练数据中加入大量代码能提升模型推理能力"几乎是业界共识——这是 StarCoder、Code Llama 等工作的核心发现。这篇研究在 <b>10T token</b> 语料上做了严格的受控实验，用域分离设计剥离了混杂因素，得出更精确的结论：<b>起作用的不是可执行代码本身，而是 code-text 和 math-text 混合带来的跨域结构化推理轨迹</b>。纯可执行代码不仅不提升通用推理，还与复杂数学推理存在竞争关系。</p>
  <p>MoE routing 分析进一步提供了机制层面证据，不只是 benchmark 上的相关性。这意味着预训练数据配比的优化空间仍然存在，且方向比之前认知的更精确。</p>
  <p>对一级市场而言，这影响的是<b>基础模型厂商的数据策略</b>和依赖"高质量代码数据"为核心壁垒的数据供应商——若结构化推理轨迹（code-text / math-text 混合）才是关键，则纯代码数据集的壁垒价值需要重估。同时，这为 <b>MoE 架构</b>的数据路由优化提供了新的设计依据。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>评估现有数据供应商标的的核心资产：若其壁垒在"高质量可执行代码"而非"结构化推理轨迹"，需重新评估其对基础模型厂商的差异化价值。</li>
    <li>关注 Kai Zhang 团队后续工作，其在预训练数据配比方向的系统性实验有望成为基础模型数据策略的重要参考文献。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>研究在 10T token 语料上做受控预训练实验，通过细粒度域分离设计和 MoE routing 分析，发现"代码训练提升推理能力"的真正来源是 code-text 和 math-text 混合带来的跨域结构化推理轨迹，而非可执行代码本身。纯可执行代码与复杂数学推理存在竞争关系，优化结构化样本密度可在固定预算下显著提升难数学推理并基本保留编程性能。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：在 10T token 语料上进行受控预训练实验，对 code、math、NL 域进行细粒度分离，系统控制 Code-NL 数据比例，分析各数据组分对不同能力维度的独立贡献。</li>
  <li><b>技术细节</b>：通过"受控域分离"排除混杂因素，逐步替换或隔离 code/math/structured trace 三类数据；借助 MoE 模型 expert activation routing 分析揭示数据组合在模型内部的竞争或协同机制。</li>
  <li><b>主结果</b>：排除 Code-NL 数据干扰后，纯可执行代码仅提升编程能力，与复杂数学推理存在竞争关系；数学推理增益主要由 code-text 和 math-text 跨域结构化推理轨迹解释。</li>
  <li><b>关键结果点</b>：在固定 math 预算内提高结构化 math 样本密度可显著提升难数学推理同时基本保留编程性能；MoE routing 分析提供机制层面证据。</li>
</ul>

<h4>TDAA：针对预训练 Encoder 的有目标 Downstream-Agnostic 对抗攻击</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19446">Targeted Downstream-Agnostic Adversarial Attack on Pre-trained Encoders</a></li>
  <li><b>信号源</b>：<b>Microsoft Yi Zhang</b> · Peking Zhuxin Lei（一作）· ETH Ziyuan Yang（合作）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>预训练 encoder 的对抗攻击研究通常只做"无目标"版本——让模型分类出错，但不控制错向哪里。TDAA 将攻击升级为"定向版本"：攻击者预选一张 threat image，生成器为每张输入定制专属扰动，迫使 victim encoder 将被攻击样本特征与目标图像对齐，且对任何下游任务均有效。在 <b>10 种</b>自监督方法、<b>3 个</b> benchmark 数据集上验证，攻击成功率和扰动不可见性均优于现有 DAA 方法。</p>
  <p>对 AI 安全赛道：TDAA 揭示了当前广泛部署的 CLIP、SAM 等 foundation model encoder 在定向攻击下的系统性漏洞，这直接影响依赖 encoder 特征进行内容审核、生物识别、医疗图像分析的应用场景的安全评估逻辑。对 <b>AI 安全审计</b>和 <b>adversarial robustness</b> 方向的初创是潜在商业化机会窗口。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注依赖 CLIP/SAM 类 encoder 进行核心功能的垂直应用（内容安全、医疗影像）——评估其是否有 adversarial robustness 评估需求，作为 AI 安全审计服务标的的切入点。</li>
    <li>跟踪 Yi Zhang（MSR）团队在 encoder 安全方向的后续工作，其 example-specific 扰动生成方法有商业化潜力。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>TDAA 在严格威胁模型下实现有目标且 task-agnostic 的对抗攻击：攻击者预先选定一张 threat image，训练生成器为每个输入样本生成 example-specific 扰动，迫使 victim encoder 对扰动样本的特征与 threat image 对齐。在 10 种自监督预训练方法和 3 个 benchmark 数据集上，TDAA 在攻击成功率和扰动不可见性方面均优于现有 DAA 方法，揭示预训练 encoder 在定向攻击下的严峻安全风险。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：攻击者事先选定 threat image，训练生成器为每个输入样本生成 example-specific 对抗扰动，使 victim encoder 对扰动后样本的输出特征与 threat image 特征尽可能一致，在不知道下游任务的前提下实现定向攻击。</li>
  <li><b>技术细节</b>：生成器以原始图像为输入，输出与图像内容相关的专属扰动（而非共用扰动），通过最小化 DAE 与 threat image 在 encoder 特征空间中的距离来优化，类似为每个伪装对象量身定制面具。</li>
  <li><b>主结果</b>：在 10 种自监督方法（对比学习类）及 3 个 benchmark 数据集上，TDAA 攻击成功率和扰动不可见性均优于现有 DAA 方法。</li>
  <li><b>关键结果点</b>：example-specific 扰动相比单一共享扰动在攻击成功率和不可见性上均有提升；揭示预训练 encoder 在定向攻击下的显著安全漏洞。</li>
</ul>

<h3>多模态智能 (Multimodal Intelligence)</h3>

<h4>DarkLLM：用 1B LLM 统一驱动跨模型跨任务的视觉对抗攻击</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.18868">DarkLLM: A Unified Adversarial Attack Framework via LLM-driven Visual Perturbations</a></li>
  <li><b>信号源</b>：<b>UCSC Xin Wang</b> · Harvard Ye Sun（一作）· Google Yu-Gang Jiang（资深合作）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>传统视觉对抗攻击需要为每个目标模型单独设计——针对 CLIP 有 CLIP 的攻击方式，针对 SAM 又是另一套。DarkLLM 用一个经过 instruction tuning 的 <b>1B</b> 参数 LLM 充当"通用攻击翻译器"，把自然语言攻击意图转化为跨模型通用的视觉扰动，一套框架覆盖 <b>4 类</b>攻击任务、<b>13 个</b>数据集、<b>15 个</b>模型。</p>
  <p>认知上的转变：foundation model 的安全边界不再是单点的，而是系统性的——一个轻量攻击模型就能同时威胁 CLIP、SAM 及主流 LLM。这对部署多模态 AI 的企业（医疗影像、自动驾驶感知、内容安全审核）构成需要认真评估的系统性风险。</p>
  <p>对 <b>AI 安全</b>赛道：DarkLLM 的存在反过来为 adversarial robustness 评估和 red-teaming 服务提供了商业化需求背书，依赖 CLIP/SAM 类 encoder 的关键应用场景需要标准化安全评估流程。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈专注多模态 AI 安全评估 / red-teaming 的初创团队，DarkLLM 类威胁模型的出现为该方向提供了更强的商业化叙事支撑。</li>
    <li>评估已投多模态应用标的（尤其医疗影像、内容安全）是否需要将 adversarial robustness 纳入产品合规评估流程。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>DarkLLM 将一个 1B 参数 LLM 进行 instruction tuning，使其接受自然语言攻击指令并输出 latent attack vector，再经解码器生成视觉 adversarial perturbation，施加于输入图像以欺骗目标模型。框架统一了 targeted、untargeted、segmentation 和 multi-model 四类攻击，在 4 类任务、13 个数据集、15 个模型的大规模评测中展示出高有效性和跨模型泛化能力。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：1B LLM 经 instruction tuning 接受自然语言攻击指令，输出 latent attack vector；该向量经解码器生成视觉 adversarial perturbation，统一覆盖多种攻击类型，无需为不同模型/任务重新设计。</li>
  <li><b>技术细节</b>：攻击目标从硬编码损失函数转化为可解析自然语言指令，LLM token 空间充当攻击意图的语义编码层，latent vector 解码为像素空间扰动时借助类似 diffusion decoder 或线性投影结构。</li>
  <li><b>主结果</b>：在 4 类攻击任务、13 个数据集、15 个模型的评测中，DarkLLM 展示高有效性和跨模型泛化能力，揭示 CLIP/SAM 及主流 LLM 的系统性安全风险。</li>
  <li><b>关键结果点</b>：仅 1B 参数模型即可有效攻击跨架构多模态 foundation model；统一框架无需针对每个模型/任务单独设计攻击策略。</li>
</ul>

<h3>世界模型 (World Model)</h3>

<h4>SWEET：用 Image Editing 替代 Video Generation 的稀疏视觉世界模型用于机器人操控</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19319">SWEET: Sparse World Model via Image Editing for Robot Manipulation</a></li>
  <li><b>信号源</b>：NUS Mike Zheng Shou（资深合作）· Google Yiren Song（一作）· MIT Yihan Wang（合作）</li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>机器人操控中的视觉世界模型几乎默认用 video generation 建模——完整视频序列既昂贵又对任务执行来说信息过度。SWEET 提出了一个反直觉但有效的替代：用 image editing 模型（FLUX-Kontext）生成稀疏 keyframe，而非完整视频。实验显示，FLUX-Kontext 生成的 task-level keyframe 在视觉保真度上优于 video generation 模型 Wan2.2，同时推理成本显著更低。</p>
  <p>在 DROID 和 RoboMimic benchmark 上，SWEET 在 seen 和 unseen 场景下均改善了 keyframe 预测质量，并验证了从 sparse keyframe 规划到可执行 action chunk 的完整 pipeline 可行性。mixed-training 策略（混合真实帧和 edited 帧）是消除分布偏差的关键工程细节。</p>
  <p>对<b>具身智能</b>赛道：这降低了 world model 驱动的 robot manipulation 系统的推理成本基线，利好依赖视觉规划的机器人软件栈初创；同时也意味着 image editing 模型（如 FLUX 生态）在机器人仿真数据生成中有新的应用场景。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>评估 SWEET 类"稀疏 keyframe 规划"方法对已投机器人操控软件栈的适用性——是否可降低其 world model 推理成本从而改善单位经济。</li>
    <li>关注 FLUX-Kontext 在机器人仿真数据生成场景的商业化应用，是否有创业团队在此方向切入。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>SWEET 用 image editing 模型（FLUX-Kontext）替代 dense video generation 作为稀疏视觉世界模型，通过自回归生成 manipulation keyframe 序列（稀疏视觉子目标），再用 goal-conditioned diffusion action predictor 将相邻 keyframe 对转化为可执行 action chunk。引入 mixed-training 策略缩小真实帧与 edited 帧的分布偏差，在 DROID 和 RoboMimic 上验证了完整 pipeline 可行性。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：两阶段 pipeline：FLUX-Kontext 以语言指令和可选箭头空间引导为条件自回归生成稀疏 manipulation keyframe；goal-conditioned diffusion action predictor 将相邻 keyframe 对转化为可执行 action chunk。</li>
  <li><b>技术细节</b>：引入 mixed-training 策略，训练 action predictor 时混合使用真实帧和经过筛选的 edited 帧，同时适应两种输入分布；FLUX-Kontext 相比 Wan2.2 在 task-level keyframe 保真度更高、推理成本显著更低。</li>
  <li><b>主结果</b>：对比实验显示 image editing 模型生成的 task-level keyframe 视觉保真度优于 video generation 模型，推理成本显著更低。</li>
  <li><b>关键结果点</b>：在 DROID 和 RoboMimic 基准上，seen 和 unseen 场景下 keyframe 预测均有提升；验证从 sparse keyframe 规划到可执行 action chunk 的完整流程可行性。</li>
</ul>

<h4>CODA：GEMM-Epilogue 抽象让 LLM 自动生成高性能 Transformer Kernel</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.19269">CODA: GEMM-Epilogue Abstraction for Transformer Non-Attention Operators</a></li>
  <li><b>信号源</b>：<b>Princeton Tri Dao</b> · Princeton Han Guo（一作）· Princeton Jack Zhang（合作）</li>
</ul>
<callout emoji="⚙️">
  <p><b>Our Insights</b></p>
  <p>见头条详述。CODA 解决了 FlashAttention 之外的"剩余瓶颈"——non-attention memory-bound 算子的显存搬运开销。更重要的是其可组合 epilogue 原语设计，使 LLM 能自动生成高性能 kernel，将 AI 辅助编译器的落地门槛系统性降低。对依赖"CUDA 专家手工优化"壁垒的 inference 优化初创构成直接压力；对 AI compiler-as-a-service 方向是新的商业化信号。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈 AI 辅助 kernel 自动生成方向的初创，CODA 开源后该赛道产品的技术基线将被重新定义。</li>
    <li>跟踪 Tri Dao 是否将 CODA 整合进 Flash 系列产品，作为其下一轮商业化节点。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>CODA 将 Transformer block 中的 memory-bound 非注意力算子建模为 GEMM-plus-epilogue 程序，在 GEMM 输出 tile 驻留片上 SRAM 时执行 normalization、activation、residual 等操作，避免中间张量写回 global memory 的往返开销。框架提供可组合 epilogue 原语，人工与 LLM 自动生成的 kernel 均达到高性能。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：将 non-attention 计算建模为 GEMM-plus-epilogue；GEMM mainloop 不变，epilogue 在输出 tile 驻留片上 SRAM 时执行缩放、规约、逐元素变换和累加；暴露 scaling/reduction/pairwise transformation/accumulation 四类可组合原语。</li>
  <li><b>技术细节</b>："代数重参数化"将 LayerNorm 规约、残差加法、activation 函数重新推导为 GEMM 输出 tile 的局部操作，通过固定接口约束使 operator fusion 系统化、可自动化。</li>
  <li><b>主结果</b>：人工与 LLM 自动生成的 CODA kernel 在代表性 Transformer workload 上均实现高性能，验证 GEMM-epilogue 编程路径可行性。</li>
  <li><b>关键结果点</b>：覆盖标准 Transformer block 前/后向几乎全部非 attention 计算；LLM 自动生成路径成立，为 AI 辅助 kernel 编译开辟实用路径。</li>
</ul>

<h4>EEG Microstates 通用表征学习：将脑电信号离散化为"思维原子"实现跨任务迁移</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20182">Universal EEG Representation Learning via Microstate Tokenization</a></li>
  <li><b>信号源</b>：<b>UCSC Xin Wang</b> · MIT Xinyang Tian（一作）· Waterloo Xuesong Chen（资深合作）</li>
</ul>
<callout emoji="🧬">
  <p><b>Our Insights</b></p>
  <p>EEG 信号分析长期缺乏统一的表征范式——睡眠分期、情绪识别、运动想象分类各用各的特征工程。这项工作把 EEG 信号"分词"：将连续脑电流聚类为持续数十毫秒的稳态模式（microstates），构建通用 tokenizer，使语言模型式的序列建模方法可直接应用于脑电信号。在三类下游任务上，microstate 表征一致优于传统时域/频域特征。</p>
  <p>对 <b>BCI 和数字医疗</b>赛道：通用 EEG tokenizer 的出现意味着下游任务的开发成本显著降低——不再需要为每类任务重新做特征工程。这利好构建"通用脑电分析平台"的标的，同时为脑机接口、神经疾病监测应用提供更可扩展的表征基础。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>评估 BCI / 神经监测领域已接触的标的是否有将通用 EEG tokenizer 纳入技术栈的可行性，作为其产品差异化方向。</li>
    <li>关注 Xin Wang（SimularAI Head of Research）在 BCI/神经信号方向的后续工作，SimularAI 的研究方向值得持续跟踪。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>研究从大规模医疗 EEG 数据集出发，将连续 EEG 信号聚类为离散 microstate token 序列，构建通用 microstate tokenizer，作为统一前端表征模块接入多个下游任务（睡眠分期、情绪识别、运动想象分类）。microstate 表征在多种模型架构和下游任务上一致优于传统时域/频域特征，兼具更好的可解释性和扩展性。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：从大规模医疗 EEG 数据集聚类构建 microstate tokenizer，将连续 EEG 信号离散化为有限个原型状态的符号序列；tokenizer 作为通用前端接入睡眠分期、情绪识别、运动想象分类三类下游任务。</li>
  <li><b>技术细节</b>：EEG microstates 是大脑在数十毫秒时间尺度上维持的准稳态电位分布，聚类将多通道 EEG 帧映射到有限原型状态，形成固定词汇表的离散符号序列，使语言模型式序列建模方法得以直接应用。</li>
  <li><b>主结果</b>：在睡眠分期、情绪识别、运动想象分类三类任务上，microstate 表征均优于传统时域和频域特征。</li>
  <li><b>关键结果点</b>：microstate 表征具有更强可解释性和可扩展性；tokenizer 跨任务通用迁移能力验证了其作为统一 EEG 表征模块的可行性。</li>
</ul>
