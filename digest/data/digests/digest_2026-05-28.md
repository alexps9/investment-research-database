<h3>MiniMax 公开 M2 系列技术报告，首次披露 Forge RL 系统与 M2.7 自我进化探索</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.26494">MiniMax-M2 Technical Report</a></li>
  <li><b>信号源</b>：<b>Junxian He</b> · <b>Wenhu Chen</b> · MiniMax（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这份技术报告真正值得更新的判断，不在于 M2 旗舰的规模，而在于 <b>M2.7 checkpoint 公开披露了"自我进化"早期探索</b>——模型能自主 debug 训练任务、修改自身 scaffold，成为国内首个在技术报告中明示该方向的大模型团队。如果这个方向被后续产品验证，原本"scaling 靠人工数据飞轮"的路线权重可能会下降；能提供可验证 agentic 轨迹生成环境的数据基础设施方向，叙事重要性值得上调。</p>
</callout>

<p>　　大模型推理成本与参数规模之间的权衡，是过去两年 MoE 架构受到持续关注的核心原因。MiniMax 此次公开 M2 系列技术报告，旗舰 M2 总参数 229.9B、每 token 仅激活 9.8B（约 4.3%），是"以小激活量撬动大智能"这条路线的最新落地。</p>
<p>　　报告披露的三大组件中，<b>Forge RL 系统</b>是工程上最有参考价值的部分：通过 windowed-FIFO 调度管理长轨迹 batch、prefix-tree merging 复用公共前缀的键值缓存（KV cache），并将训练、推理、agent 执行三个环节解耦，使强化学习训练可以灵活接入外部黑盒 agent。M2.7 checkpoint 则初步实现自主调试训练流程并修改自身 scaffold，无需人工干预——这是国内团队在技术报告层面首次明确披露"自我进化"探索。</p>

<h3>Stanford Percy Liang 团队：招聘算法单一化（Algorithmic Monoculture）导致系统性种族差异，300 万求职者数据实证</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27371">Algorithmic Monoculture in Hiring</a></li>
  <li><b>信号源</b>：<b>Percy Liang</b>（Stanford） · Rishi Bommasani（一作，Stanford Medicine PhD 在读，导师 Percy Liang）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文提出了一个过去 HR tech 讨论中长期缺失的市场层面视角：当多数雇主同时使用同一家算法供应商时，<b>单个算法的偏差会在整个就业市场被系统性放大</b>——受影响的求职者不论投递多少家公司，都会碰到同一堵墙。如果这一框架被监管机构部分接受，"算法合规"从单点企业义务转变为平台级市场结构义务的叙事权重会上升；HR tech 领域中高度集中化的算法筛选供应商，其监管风险的评估方式可能需要从"单个用户的公平性审计"扩展到"市场集中度层面的结构性影响"。</p>
</callout>

<p>　　招聘算法领域过去的公平性研究大多聚焦于单个雇主或单个系统的偏差，假设求职者可以通过广泛投递来规避某家公司的算法偏见。这项研究则从市场结构出发，当多家雇主集中使用同一供应商时，这个假设就不成立了。</p>
<p>　　研究团队获取了一个包含 300 万求职者、400 万份申请的数据集，所有申请均由同一算法供应商筛选。利用算法的"确定性可复现性（deterministic replicability）"——同一份简历每次都会得到相同结果——研究者模拟了每位求职者在所有职位的反事实结果。结论是：<b>Black 求职者中 25.87% 的申请投向了对其造成不利影响的职位</b>，符合美国就业歧视认定标准；在投递 10 个职位的求职者中，4% 被所有职位系统性拒绝，高于随机概率预期。</p>

<p><em>* 注：数据集来源为单一算法供应商，结果能否推广至其他供应商或更广泛市场，需独立验证。</em></p>

<h3>Yann LeCun 团队理论证明 LeJEPA 可线性识别潜变量，Gaussian 是充要条件</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.26379">Linear Identifiability of LeJEPA World Models</a></li>
  <li><b>信号源</b>：<b>Yann LeCun</b>（NYU / Meta）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>JEPA 路线长期面临一个未解的理论质疑：它学到的表示是否真的在恢复世界的真实结构，还是只是某种任意的等价变换？这篇论文给出了一个条件性的肯定答案——<b>在潜变量服从 Gaussian 分布的世界里，LeJEPA 满足线性可识别性（linear identifiability），且 Gaussian 是使这一保证成立的唯一分布</b>。这意味着 JEPA 路线现在有了一块理论地基：在特定假设下，它的表示学习是有保证的，而不是"也许有效"。对下游 latent-space planning 的研究者而言，这个结果让以 JEPA 为基础构建规划模块的方向，从经验性选择变成了有理论支撑的选择。</p>
</callout>

<p>　　世界模型的表示学习领域存在一个长期争论：自监督方法（如 JEPA）学到的潜在表示，是否能够可靠地反映真实世界的因果结构，还是只是捕捉了某些统计相关性？可识别性理论（identifiability theory）是回答这一问题的严格工具，但此前 JEPA 类方法缺乏相应的理论保障。</p>
<p>　　本文在"加性噪声转移（additive-noise transitions）"的世界假设下，对 LeJEPA（alignment 损失加 Gaussian 正则化）给出了完整的理论证明。正向证明利用谱分解（spectral decomposition）说明，任何非线性成分都会提高 alignment 损失，因此最优解必然是线性映射；逆向则构造反例证明，非 Gaussian 潜变量分布无法满足相同保证——Gaussian 既充分又必要。实验在从 2D 玩具示例到 1024 维 latent 的范围内验证了理论预测。</p>

<h3>hardmaru 团队（ICLR 2026）：将 forward pass 类比 diffusion 去噪，分块训练匹配端到端性能并大幅降低显存</h3>
<ul>
  <li><b>论文</b>：<a href="https://x.com/hardmaru/status/2059648995132367277">Block-wise Training via Diffusion Analogy（ICLR 2026）</a></li>
  <li><b>信号源</b>：<b>hardmaru</b>（个人）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>端到端反向传播（backpropagation）需要在显存中保留整个网络的中间激活值，这是深度模型训练显存墙的根本来源。这篇论文用一个类比绕开了这个问题：把每个网络块的训练信号从"等待全局 loss"替换成"局部去噪目标"，使各块可以独立训练、无需跨层梯度链。<b>在 ViT、DiT、LLM 三类主流架构上匹配端到端性能，同时大幅降低峰值显存</b>——如果这个结果在更大规模上能复现，"训练大模型必须有大显存集群"这一前提的适用边界可能会收窄；边缘侧或低显存环境下训练大模型的可行性值得重新评估。</p>
</callout>

<p>　　深度模型训练的显存瓶颈一直是制约训练规模和硬件门槛的核心约束。目前主流的应对方案（梯度检查点、模型并行、卸载到 CPU 内存等）大多是"省着用"的策略，而非从根本上消除对整网激活的依赖。</p>
<p>　　这项发表于 ICLR 2026 的工作采用了一个不同的视角：将深度网络的前向传播（forward pass）重新解释为类似扩散模型（diffusion model）逐步去噪的过程，把网络切分为若干独立块，每次仅训练一个块，各块之间不依赖完整的反向传播梯度链。训练某个块时，将其输入视为"带噪信号"，目标输出视为"去噪目标"，用局部去噪信号代替跨层反向传播梯度。实验在 ViT、DiT 和 LLM 三类架构上验证，分块独立训练的最终性能与端到端反向传播持平，同时峰值显存占用大幅下降。</p>

<p><em>* 注：原文为 X 推文介绍 ICLR 2026 论文，具体数字和实验设置以正式论文为准。</em></p>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>"用更强教师模型来蒸馏小模型"是当前推理能力传递的主流路线，但这条路依赖高质量标注数据和教师模型的可获取性。这篇论文表明：<b>让模型对无标注 prompt 自我采样、用三阶段级联验证过滤后再训练自身，在数学、科学、编程三个推理域都带来了显著提升，且测试时只需单次推理</b>——这意味着"没有教师模型也能提升推理能力"的路线在小规模模型上已有实证支撑，那种"小模型必须靠大模型才能学会推理"的假设需要被部分修正。</p>
</callout>

<p>　　推理能力增强目前主要走两条路：一是从更强模型蒸馏，二是在推理时投入更多算力（test-time compute）。前者依赖外部教师，后者测试时成本高。Self-Verified Distillation 想走第三条路：模型完全依靠自身生成的数据来提升推理能力。</p>
<p>　　具体做法是对无标注的 seed questions 采样多个候选解，经过三阶段级联验证过滤（循环一致性 cycle-consistency → 事实性 factuality → 正确性 correctness，三关全票通过才接受）——类似于对考卷进行三轮盲评，只收入三轮全优的答案用于再训练。Qwen3-4B 经此方法后，数学推理（AIME26 和 HMMT）pass@1 提升 <b>+16.7 点</b>，科学（GPQA Diamond 和 HLE）提升 <b>+11.1 点</b>，编程（LCBv5 和 LCBv6）提升 <b>+8.3 点</b>，且测试时仅需单次推理，多数场景下优于消耗更多推理算力的 UQ-TTC 基线。</p>

<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27209">NoisyAgent: Robust LLM Agent Training with Environmental Noise</a></li>
  <li><b>信号源</b>：<b>Yi Zhang</b> · Yuxin Chen（共一）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>当前 LLM agent 的评测基准大多是理想化环境——工具调用总是成功、用户表达总是清晰。NoisyAgent 的结果说明：<b>在训练时主动注入现实噪声，不仅能提升噪声环境下的性能，在干净 benchmark 上也有增益</b>——这暗示过去"在干净数据上训练、部署到真实环境"的路线，可能在泛化能力上留了一块系统性缺口。对正在评估 agent 产品可靠性的投研视角来说，"是否做过噪声条件训练"可能会成为一个值得纳入尽调清单的维度。</p>
</callout>

<p>　　LLM agent 在标准化 benchmark 上的表现与真实部署落差，一直是产品化过程中的主要痛点。用户表达模糊、工具调用异常在现实中极为常见，但训练数据往往不覆盖这些场景。</p>

<h4>Jianfeng Gao 团队：Latent Recurrent Transformer（LRT），仅 0.3% 额外参数引入跨 token 循环记忆</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.26797">Latent Recurrent Transformer</a></li>
  <li><b>信号源</b>：<b>Jianfeng Gao</b>（Microsoft Research）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>循环（recurrent）结构与 Transformer 融合的思路并不新鲜，但此前的方案大多需要插入额外的 pause token 或专门的深度循环层，改变了原有架构的推理接口。LRT 的关键设计是：<b>把上一个 token 某层的隐状态直接复用为下一个 token 的记忆，而这个隐状态在正常解码中本来就会算——因此几乎不增加额外参数或推理成本，且与现有的键值缓存（KV cache）接口保持兼容</b>。这意味着"引入循环记忆必须改造推理栈"的假设值得重新检验；现有 serving 基础设施是否能以极低改动成本获得循环记忆带来的长文本建模收益，是一个值得跟踪的工程验证点。</p>
</callout>

<p>　　纯 Transformer 的注意力机制在处理长序列时存在二次方复杂度的问题，循环架构（如 LSTM/SSM）虽然可以用常数内存处理任意长序列，但训练并行性差、与现有 serving 系统不兼容。LRT 尝试在不改变推理接口的前提下引入循环特性。</p>
<p>　　LRT 在标准自回归（autoregressive）Transformer 中增加一条跨位置的循环潜变量通路：每个 token 在特定层（source layer）的隐状态被传递给下一个 token，通过跨层注意力注入记忆，标准注意力机制和 KV cache 接口保持不变。为解决循环结构难以并行训练的问题，作者设计了"交错并行训练（interleaved parallel training）"：先对完整序列做一次初始化前向传播写入共享 buffer，再将序列按位置分为不相交子集并行更新，整体计算开销约为基线的 2 倍。额外参数仅 0.3%，在语言建模 loss 和上下文学习（in-context learning）两项指标上均优于同等有效计算量的基线。</p>

<h4>Hao Zhu 团队：PRISM，用循环网络建模多意图切换的逆强化学习（IRL）方法</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.26998">PRISM: Multi-Intention Inverse Reinforcement Learning</a></li>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>传统逆强化学习（IRL）假设智能体在整段轨迹中只有一个固定目标，这对机器人和人类行为数据来说是个相当强的限制——现实中目标会切换。PRISM 把多意图 IRL 做到了 EM 框架下可以精确求解、不需要变分近似，<b>并首次在大规模机器人操控数据集上恢复出时间连贯的可命名意图</b>。这意味着从无标注示范中自动发现结构化行为模式变得更可行；对机器人技能学习和人类行为分析两个方向，都是值得跟踪的方法论进展。</p>
</callout>

<p>　　标准 IRL 的 Markov 假设要求智能体目标在整段轨迹中保持不变，实际上机器人操控、人类日常行为中目标频繁切换是常态。现有多意图 IRL 方案大多需要手工设计历史窗口或变分近似，泛化性有限。</p>
<p>　　PRISM 用一个轻量循环网络（recurrent network）对每步的意图后验进行推断，并证明期望最大化（EM）目标可精确分解为独立的 per-intention 奖励子问题，每个子问题有闭合解，E 步（E-step）复杂度仅 O(nK)（n 为轨迹长度，K 为意图数量）。在非马尔可夫 gridworld、鼠迷宫和大规模机器人操控数据集 BridgeData V2 三类任务上，PRISM 均取得最高 held-out log-likelihood，并从无标注示范中自动恢复出可命名、时间连贯的意图。</p>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>音频语言模型（ALM）在多模态领域的竞争叙事主要集中在语音理解和语音生成，音乐理解长期是被忽视的子方向。PitchBench 系统测试后发现：<b>当前 frontier ALM 连基础的音高感知（pitch perception）都高度不稳定，随音源、音符时长、响应格式变化剧烈</b>。这说明"音乐理解"作为 ALM 的一项能力，目前基本处于未解决状态——音乐科技领域中依赖 AI 进行音准检测、旋律分析的方向，短期内仍将依赖专门训练的模型而非通用 ALM。</p>
</callout>

<p>　　音频语言模型的评测体系目前主要覆盖语音识别（ASR）、说话人识别和音频描述等任务，对音乐专项能力的系统性评估几乎空白。音高感知（pitch perception）是音乐理解的基础能力，理论上应该是最容易习得的音乐属性之一。</p>
<p>　　PitchBench 构建了 28 项实验，覆盖绝对音高、相对音高、和弦、旋律线条等维度，在音量、音符时长、音源类型、背景噪声等多种声学条件下进行变量控制。评测以开放式问答和多选两种格式出题，避免仅靠猜测选项掩盖模型真实能力。结论是：当前 frontier ALM 在音高识别上高度不可靠，各声学条件下性能差异显著，尚不具备稳定的音高感知能力。数据生成工具以 Python 包形式开源，支持扩展新的声学条件。</p>

<h4>Jiatao Gu（Apple）：wait-think-answer 可学习控制框架，让 streaming 语音 AI 动态决定何时开口</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27190">Learnable Listening: Wait-Think-Answer for Streaming LALMs</a></li>
  <li><b>信号源</b>：<b>Jiatao Gu</b>（Apple，通讯） · Zhiyuan Song（一作，导师 Jiatao Gu）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>流式语音 AI 的主流做法是等语音输入完全结束再开始推理，这在低延迟交互场景下带来明显的响应滞后。这篇论文把"何时开始推理"本身变成一个可以训练的决策——<b>模型学会在流式输入过程中动态判断何时积累了足够证据再给出答案，同时保持推理延迟不增加</b>。这意味着"流式语音 AI 必须在端点后才能思考"的假设被打破；对正在做实时语音 agent 的方向来说，这个方法论提供了一个在准确率和延迟之间联合优化的新路径。</p>
</callout>

<p>　　实时语音交互场景对延迟极为敏感，但当前大型音频语言模型（LALM）的推理流程通常要等完整的音频端点信号才开始思考，导致端到端响应延迟偏高。已有的早期结束检测方案没有把"推理决策"本身纳入训练。</p>
<p>　　本文以 Qwen2.5-Omni-7B 为基础，构建 wait-think-answer 控制公式，控制器在流式音频下做三种动作选择：wait（继续监听）、think（输出紧凑的中间推理更新）或 answer（给出最终答案）。先通过监督微调（SFT）初始化，再用六奖励 DAPO（解耦剪切与动态采样策略优化，Decoupled Clip and Dynamic Sampling Policy Optimization）对完整轨迹进行强化学习——奖励函数综合答案正确性、动作合法性、延迟同步、推理质量等六项指标。结果是在 6 任务语音推理问答（SRQA）基准上，行加权准确率从 67.6% 提升至 70.3%，同时 post-endpoint final-think 长度减少 14%，实现准确率与延迟的双重优化。</p>

<h4>Jiajun Wu 团队：HTMLCure，基于浏览器交互状态的 HTML 质量评估与自动修复，构建 40K 精炼 SFT 数据集</h4>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>空间基础模型（spatial foundation model）的评测过去高度碎片化——每个方向（深度估计、3D 重建、具身导航）各有自己的基准，缺乏跨范式的横向比较。SpatialBench 的结果显示：<b>当前 41 个被评测模型没有一个达到全场景通用水平，具身和第一人称（egocentric）场景下数据质量的影响远大于数据规模</b>。这意味着"做一个通用空间基础模型"在技术上的挑战比目前叙事中描述的要难——专注特定应用域的空间感知方案，短期内仍有合理的生存空间。</p>

<h4>Noah Snavely（Cornell / Google AI）：G3T，重力对齐坐标系替代相机坐标系，大幅提升 3D 重建精度</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27372">G3T: Gravity-Grounded 3D Reconstruction</a></li>
  <li><b>信号源</b>：<b>Noah Snavely</b>（Cornell / Google AI） · Bharath Raj Nagoor Kani（一作，Cornell，导师 Shubham Tulsiani）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>前馈式三维重建（feed-forward 3D reconstruction）近年发展很快，但在长序列和跨视角对齐上的误差积累一直是瓶颈。G3T 的思路不是改模型，而是改坐标系：<b>把"向上"方向固定到重力方向，跨视角对齐的旋转搜索空间从 SO(3) 的三个自由度降到水平面内的一个自由度</b>。这个设计利用了自然场景中几乎普遍存在的物理约束，是一个以极小改动换来大幅性能提升的典型例子。对实际工程部署来说，微调数据和方法都可以在现有前馈重建模型（如 VGGT）基础上叠加，迁移成本低。</p>
</callout>

<p>　　当前主流的前馈式三维重建方法（如 VGGT、DUSt3R）以相机为中心建立坐标系预测点图（pointmap），在跨视角对齐时需要处理完整的旋转自由度（SO(3)，三个方向）。对现实场景来说，大多数场景有明确的重力方向，水平面以外的旋转是多余的自由度。</p>
<p>　　G3T 通过在重力对齐的三维数据上微调现有重建模型，输出"竖直朝上"的点图（upright pointmap）及相机到重力方向的位姿（camera-to-gravity pose）。固定重力轴后，不同视角的点图仅需在水平面内对齐，搜索空间大幅缩减。G3T-Long 基于此构建子图（submap）增量式重建管线，利用竖直轴共享特性在子图合并时显著降低误差积累。在重力感知预测任务上精度显著优于相机中心坐标系基线，G3T-Long 在长序列三维重建场景下重建精度大幅改善。</p>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>多 agent 具身导航的现有研究大多在室内小场景下展开，去中心化、城市级规模的协作规划是一个尚未被充分探索的方向。CoSaR 把大语言模型（LLM）的高层通信规划能力和经典导航算法的精确执行能力层次化组合，<b>在 14 个城市级场景上相比基线同时实现更快集合、更短路径和更高安全性</b>。这对多机器人协作和城市级调度方向是一个有参考价值的方法论——基础模型作为"协调层"而非"执行层"的架构模式，在这类任务上展现出比端到端单一模型更强的实用性。</p>
</callout>

<p>　　多 agent 导航任务通常在封闭环境下设计，缺乏对动态约束（如巡逻障碍物）和大规模开放环境的处理能力，而真实机器人协作（如物流、安保、搜救）往往需要在城市级室外场景中动态调整集合策略。</p>

<h4>Yang You 团队：FA-OPD，对抗双通道在线策略蒸馏（on-policy distillation）解决机器人控制行为克隆的分布偏移</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27095">FA-OPD: Flow-Adversarial On-Policy Distillation for Robot Control</a></li>
  <li><b>信号源</b>：Wan ZhengLin（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>行为克隆（behavioral cloning）在机器人控制中的核心缺陷是分布偏移：策略在专家状态上训练，一旦真实运行偏离专家轨迹就容易失控。FA-OPD 用一个由示范数据训练的流匹配（Flow Matching）教师，对学生策略的真实运行轨迹同时提供奖励信号和动作信号——<b>奖励通道驱动长程探索，动作通道在学生真实访问的状态处提供密集局部目标</b>。在 6 个机器人基准上超越强基线，在噪声和有限示范条件下的鲁棒性尤其突出——这对低数据量场景（如真实机器人采集成本高）的应用方向有直接参考价值。</p>
</callout>

<p>　　行为克隆是机器人模仿学习的基础方法，但它只在专家示范的状态分布上学习，部署时一旦遇到专家轨迹覆盖不到的状态就容易出错。现有的在线策略蒸馏（on-policy distillation）方案要么只依赖奖励信号（稀疏、长程反馈），要么只依赖动作监督（局部、无探索能力），难以兼顾长程稳定性和局部精准性。</p>
<p>　　FA-OPD 将一个从示范数据学习的流匹配（Flow Matching）教师与一个轻量多层感知机（MLP）学生协同训练。教师在学生的在线策略（on-policy）运行轨迹上提供两路信号：奖励通道学习 state-action 对的专家相似度目标，用于长程策略梯度优化；动作通道在学生真实访问的状态处提供密集动作监督目标，防止策略发散。对抗机制体现在教师根据学生运行轨迹动态调整，形成闭环协同。在 6 个涵盖导航、操控和运动的机器人基准上超越强基线，并在噪声示范和有限示范场景下表现出更强鲁棒性。</p>

<h4>Song Han（MIT）：JetViT，后训练注意力搜索（Post-Training Attention Search）将 ViT 转为混合注意力变体，H100 吞吐提升 1.79x</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.26636">JetViT: Post-Training Attention Search for Efficient Vision Transformers</a></li>

</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>视觉 Transformer（ViT）的全注意力机制在高分辨率视觉任务中是推理成本的主要来源，现有的混合注意力方案大多需要从头训练。JetViT 的关键设计是：<b>继承预训练权重，通过搜索算法自动识别哪些全注意力块是冗余的并替换为线性注意力或窗口注意力，在精度无损失的前提下 H100 上实现最高 1.79 倍吞吐提升和 44.81% 延迟降低</b>。"后训练搜索"而非"从头训练"意味着这个方法可以直接叠加到已有的 ViT 部署上，改造成本极低——对视觉推理服务方向是一个直接可用的优化工具。</p>
</callout>

<p>　　视觉 Transformer（ViT）部署的主要成本来自全注意力机制在高分辨率图像上的二次方复杂度。已有研究表明，把部分全注意力块替换为线性注意力（linear attention）或窗口注意力（window attention）可以大幅降低推理成本，但这通常需要从头重新训练混合架构，对已经完成预训练的大型视觉基础模型来说改造成本高。</p>
<p>　　JetViT 采用后训练注意力搜索（Post-Training Attention Search）框架，从预训练全注意力 ViT 出发，继承原始 MLP 和注意力权重，通过三步搜索找到最优混合架构：第一步优化线性注意力块的具体设计，第二步搜索线性注意力和窗口注意力块的最佳组合，第三步识别并保留对性能至关重要的全注意力块。整个过程无需完整重训，计算开销远低于从头训练。在 DINOv3 和 DepthAnythingV2 两个高分辨率视觉基础模型上验证，精度无损失，NVIDIA H100 上最高 1.79 倍吞吐提升和 44.81% 延迟降低。</p>
