<h3>NEO-ov（视觉语言模型）：NTU Ziwei Liu 团队提出无 encoder 原生视觉语言模型，在多图、视频与空间智能任务上追平模块化 VLM</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28820">NEO-ov: Native One-Vision Foundation Model</a></li>
  <li><b>信号源</b>：<b>NTU Ziwei Liu</b>（通讯）· Haiwen Diao（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>现有多模态大模型（MLLM）的主流设计是"先 encode 图像、再交给语言模型"——两个独立模块拼在一起，像素信号在模块边界容易断裂。NEO-ov 这篇论文表明：彻底去掉这条边界，让视觉和语言表征在同一层内交互，模型在多图像、视频理解和三维空间（3D spatial）感知上的表现可以追平那些带独立视觉编码器（ViT）的模块化 VLM。<b>这意味着"必须依赖独立 ViT encoder"作为多模态架构标配的假设可能需要重估</b>——encoder-free 路线如果在更大规模下复现，围绕 ViT 细调的整套多模态适配生态的技术必要性会被稀释。</p>
</callout>

<p>　　多模态大模型过去三年的主流路线是"冻结预训练视觉 encoder + 语言模型对齐"，这条路简单有效，但存在一个结构性代价：视觉特征和语言特征来自不同预训练目标，对齐阶段容易损失细粒度像素级信息。这也是空间感知和细粒度视觉任务上 VLM 一直落后的原因之一。</p>
<p>　　NEO-ov 选择完全端到端的 native 路线：不引入任何外部图像编码器、辅助适配器（adapter）或后融合模块，直接从原始像素出发，在统一的 Transformer 结构内同时学习跨帧的时空关系和像素词汇对应（pixel-word correspondence）。论文系统披露了训练方案（training recipe）细节，并在多图像理解、视频理解、空间智能等多项任务上与模块化 VLM 对比，结果显示性能差距大幅收窄，细粒度视觉感知任务上甚至超越对手。</p>

<h3>Cognitive Taxonomy：Stanford Noah Goodman 团队提出通用智能 10 项认知能力分类框架，为 AGI 进展测量提供实证路线图</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28405">Cognitive Taxonomy for General Intelligence Evaluation</a></li>
  <li><b>信号源</b>：<b>Stanford Noah D. Goodman</b>（合作）· Ryan Burnell（一作，Alan Turing Institute）· Shane Legg（通讯）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>当前 AGI 进展评估高度依赖单一 benchmark 分数，这套方法的缺陷越来越明显：一个模型可能在数学竞赛题上达到人类水平，却在规划或持续记忆上严重欠缺。Cognitive Taxonomy 这篇论文尝试把"通用智能"拆成 10 项来自认知科学的独立能力维度，每项配独立的 held-out 评测任务，最终汇成多维剖面（cognitive profile）。<b>如果这套框架被 AI 评测社区部分采纳，"单一 benchmark 定英雄"的叙事在叙事重要性上的权重会下降</b>；面向监管和治理的多维能力披露路线，在政策层面的讨论优先级可能上升。</p>
</callout>

<p>　　AGI 的测量问题一直是研究社区的痛点：现有 benchmark（数学、代码、常识问答）各自覆盖一个切面，不同模型在不同 benchmark 上此消彼长，很难得出整体性结论。更根本的问题在于，没有人把"通用智能"到底由哪些组件构成这件事说清楚——直到借鉴认知科学的分类框架出现。</p>
<p>　　Cognitive Taxonomy 的结构分两层：第一层参照心理学、神经科学和认知科学文献，将通用智能拆解为 10 项核心认知能力，涵盖感知、记忆、推理、规划等；第二层为评估协议，针对每项能力设计 held-out 任务（即不出现在训练数据中），评分结果汇聚为雷达图式的认知剖面（cognitive profile），可直观比较不同系统在各维度的差异。论文目前为框架提案，尚未报告具体系统的数字对比，其贡献在于提供了一套可复现、可扩展的路线图。</p>

<h3>基础模型 (Cognitive Models)</h3>

<h4>CORE：Stanford Noah Goodman 团队提出对比反思非参数学习，少样本场景下收敛比 GRPO 等方法更快</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28742">CORE: Contrastive Reflection for Non-parametric Reasoning Improvement</a></li>
  <li><b>信号源</b>：<b>Stanford Noah D. Goodman</b>（合作）· Linas Nasvytis（共一）· Simon Jerome Han（共一）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>当前提升大模型推理能力的主流方向是参数化方法——通过强化学习（RL）或监督微调（SFT）直接更新权重，成本高且需要大量 rollout。CORE 这篇论文表明：不更新任何权重，只让模型对比自己过去成功和失败的推理轨迹，提炼出自然语言策略摘要（insights）并注入上下文，就能在 5 个训练样本场景下达到与 GRPO、episodic 检索增强生成（RAG）等方法相当甚至更优的推理提升。<b>这意味着"推理能力提升必须依赖权重更新"的默认假设存在反例</b>——非参数路线如果在更复杂任务上保持竞争力，轻量自我改进工具链（无需 RL 基础设施）的研发优先级值得上调。</p>
</callout>

<p>　　现有推理增强方法可以分为两大类：一类直接更新模型权重（GRPO、SFT、MemRL 等），另一类依赖上下文（episodic RAG 直接复用历史轨迹）。前者资源消耗大，后者 context 长度随历史积累膨胀，泛化能力有限。CORE 走了第三条路：让模型对比一组成功与失败的推理轨迹，由模型自身提炼出描述策略差异的自然语言 insight（如"此类问题应先检验边界条件"），随后在新问题推理时将这些 insight 作为上下文注入。</p>
<p>　　与直接复用历史轨迹相比，CORE 把对比信息压缩为抽象规则，既缩短了 context 长度，又提升了跨问题泛化能力。在 4 个推理任务上，CORE 使用的 rollout 数量更少、收敛更快，在 5 个训练样本的极端低资源场景下仍与各 baseline 持平甚至更优，context token 消耗也显著低于 episodic RAG。</p>

<h4>CODE：因果叙述替代静态事实覆写，将 LLM 知识编辑后自我反驳率从 95.6% 压至 1.8%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28303">CODE: Causal On-policy Distillation for Knowledge Editing</a></li>
  <li><b>信号源</b>：<b>Kai Zhang</b>（合作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>LLM 知识编辑领域过去的主流假设是"把旧事实替换成新事实就够了"。CODE 这篇论文系统展示了这个假设的结构性缺陷：零失真代理实验下，静态事实覆写后模型的自我反驳率高达 95.6%——模型的先验推理体系无法接受孤立注入的新事实，导致知识冲突。引入因果叙述（"旧知识为何变成新知识"）后冲突率降至 6.6%，再加上非对称 on-policy 蒸馏将因果模式写入参数后，<b>自我反驳率进一步压缩到 1.8%</b>。<b>这意味着知识编辑工具链如果不解决因果一致性问题，在需要多步推理的企业知识库更新场景里仍然不可靠</b>——内嵌因果结构的编辑方法，在实用性上对现有方案有实质替代潜力。</p>
</callout>

<p>　　知识编辑是让大模型在不全量重训的前提下更新特定事实的一类技术，在企业知识库维护、时效性更新等场景有直接需求。但现有方法（ROME、MEMIT 等）本质上是"数据库式覆写"——直接修改模型对某个事实的存储，而不解释为什么旧事实变了，导致模型在多跳推理时用旧逻辑体系推出新事实的矛盾结论。</p>
<p>　　CODE 框架分两步：因果 bootstrapping 为每条知识更新自动构造"旧知识 → 触发原因 → 新知识"三元因果叙述；非对称 on-policy 蒸馏让模型在自身生成的推理轨迹上学习，对因果链路相关 token 施加更强的学习信号，将因果转换逻辑直接写入参数记忆。在 LLaMA-3.1 和 Qwen-2.5 上，自我反驳率从 95.6% 降至 <b>1.8%</b>，多跳推理准确率达 <b>83.5%</b>。</p>

<h4>Log-ICoT：UCB Jiantao Jiao 团队首次给出隐式思维链理论保证，训练阶段数从线性压至对数级</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28600">Log-ICoT: Logarithmic Implicit Chain-of-Thought Curriculum</a></li>
  <li><b>信号源</b>：<b>UCB Jiantao Jiao</b>（合作）· Yixiao Huang（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>隐式思维链（Implicit Chain-of-Thought，ICoT）的核心承诺是：让模型推理时不输出中间步骤，但保持与显式思维链（Chain-of-Thought，CoT）一样的准确率——这对生产部署的 token 成本有直接意义。这篇论文首次从理论上证明 ICoT 可行，并提出 Log-ICoT curriculum：按 2 的幂次递增地删除 thinking token，把训练阶段数从 O(k) 降至 <b>O(log k)</b>，样本效率与显式 CoT 持平。<b>这意味着"推理时必须生成完整 CoT 才能保证准确率"的工程假设有了可信的反例</b>；如果理论结论在更复杂任务上复现，压缩推理 token 的工具链研发优先级值得上调。</p>
</callout>

<p>　　大模型部署时，思维链推理带来的 token 开销是生产成本的重要组成部分。ICoT 的想法是把推理步骤"内化"进模型的隐层状态，推理时不再外显输出——但此前缺乏严格的理论保证，不知道这种内化在多层网络里是否可靠，以及训练需要多少阶段。</p>
<p>　　本文以 k-parity（k 元奇偶性学习）为理论载体，证明 L 层 transformer 在 Log-ICoT curriculum 下以多项式样本数（poly(n)）学会该任务，只需 log₂k 个训练阶段。关键机制是"几何块删除"：标准 ICoT 每次只删一个 thinking token（线性 k 步），Log-ICoT 每次删除的 token 数量按 2 的幂次递增，每一步将剩余任务折半，总阶段数压缩到 log₂k。实验和可视化进一步确认推理能力被逐层吸收进更深的网络层。</p>

<h4>HRBench：首个统一评测框架，系统比较混合推理 LLM 的三类思考模式切换策略</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28398">HRBench: Benchmarking Hybrid-Reasoning LLM Thinking-Mode Switching</a></li>
  <li><b>信号源</b>：Hao Liu（通讯）· Yansong Ning（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>混合推理（hybrid-reasoning）大模型允许在"深度思考"和"快速回答"之间动态切换，这是当前主流 LLM 产品的重要功能特性。HRBench 这篇论文首次在统一实验设置下对三类切换策略（提示词直接控制、外部路由器分配、推测式先轻后重）和四种训练机制进行横向对比。<b>关键发现是：没有任何一种策略在全部场景下最优，最优选择随模型规模和任务领域显著变化</b>——这意味着"选定一种切换策略就万事大吉"的产品设计假设需要被修正；为不同场景定制切换策略组合的能力，会成为推理产品差异化的一个新维度。</p>
</callout>

<p>　　主流前沿模型（Qwen3、Kimi-K2.5、DeepSeek R1 等）都提供了"thinking mode"开关，让用户可以根据问题复杂度选择是否触发深度推理。但各家实现方式不同——有的通过提示词、有的用外部分类器路由、有的用推测式执行（speculative execution，先轻量推理，不够再升级）——在不同任务上性能和效率表现各异，却从未在统一平台下比过。</p>
<p>　　HRBench 沿两个轴构建 12 种受控配置：三类切换策略家族和四种训练机制（无训练、监督微调 SFT、离线强化学习、在线强化学习），复现 12 种以上先前方法，在 Qwen3.5-2B 到 Kimi-K2.5-1.1T 共 6 个模型、数学和代码等 5 个任务领域上系统测试。结果显示：提示词方法在 token-准确率权衡上表现较优，路由方法成本削减更稳定，推测式方法倾向以更多 token 换取更高准确率，且最优组合随模型规模和领域显著变化。</p>

<h4>FABSVer：解题生成与自我验证融合为单次生成，训练成本压缩至现有方法的 51%–71%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28389">FABSVer: Fused Answer-Bearing Self-Verification for Mathematical Reasoning</a></li>
  <li><b>信号源</b>：<b>Yang Song</b>（资深合作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>数学推理的强化学习训练长期存在一个隐性成本：先训练生成模型，再单独训练验证器，两个完整 RL 循环叠加。FABSVer 把这两件事合并进同一次 rollout，共享键值缓存（KV cache）和梯度更新，训练时间直接缩短到现有方法的 <b>51%–71%</b>。同时引入动态参考模型更新（Dynamic Reference Model Update，DRMU），每隔若干步把参考模型替换为当前策略快照，防止 KL 散度约束把策略锁在早期水平，持续抬高 reward 上限。<b>这意味着"验证器必须独立训练"这一工程假设可以被打破</b>；训练基础设施方向上，支持统一多任务 RL 循环的调度框架需求会随之上升。</p>
</callout>

<p>　　让大模型既能解题又能验证答案是当前数学推理的核心诉求，但分开训练两个模块不只是代码工程问题——每个模块各自需要完整的强化学习采样循环，算力消耗翻倍，且两个模块的优化目标未必一致。FABSVer 的核心思路是：在同一次生成过程中同时产出解题序列和验证判断，联合优化两项能力的奖励信号。</p>
<p>　　DRMU 解决了另一个独立的收敛瓶颈：标准人类反馈强化学习（RLHF）用固定参考模型计算 KL 惩罚，策略改进到某个程度后会被参考模型"拉住"停滞，形成 reward plateau。DRMU 动态更新参考模型快照，持续抬高可达 reward 上限。在三种模型规模的数学任务上，FABSVer 在自我验证和推理性能上均优于对比方法，训练成本仅为 51%–71%，且验证与解答 reward 差距随模型规模扩大而收窄。</p>

<h4>Mobile-Aptus：置信度驱动的移动端多模态大模型 agent，离线任务成功率提升 17%，真实场景提升 26%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28629">Mobile-Aptus: Confidence-Driven Mobile-Using Agent</a></li>
  <li><b>信号源</b>：<b>Aston Zhang</b>（合作）· Zheng-Guang Wu（一作）· Zhuosheng Zhang（通讯，SJTU）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>移动端 agent 产品的两大慢性病是"管太多"（自主执行出错）和"问太多"（频繁打断用户确认）。Mobile-Aptus 这篇论文用置信度分数来决定何时自主行动、何时请求人工介入，在真实动态环境下把任务成功率提升 <b>26%</b>，同时把每条指令的人工干预步骤压到 <b>0.64 次</b>。<b>这意味着移动端 agent 的商业化瓶颈之一——用户容忍度——有了一条可量化的改善路径</b>：置信度校准能力会成为评估 agent 产品成熟度的一个新维度，而非只看任务完成率这单一指标。</p>
</callout>

<p>　　当前移动端 agent 产品的用户体验两极分化：过于保守的模型频繁弹出确认对话框，过于激进的模型则在不该点击的地方乱操作。两者都会让用户放弃。置信度驱动的决策框架是解决这个问题的直觉方向，但如何让模型输出可靠的置信度分数本身就是难题——过度自信或过度保守同样有害。</p>
<p>　　Mobile-Aptus 分两阶段解决这个问题：第一阶段通过监督微调（SFT）让多模态大模型（MLLM）同时输出动作和置信度分数；第二阶段用语义相似度检索从历史交互库中找相近案例，再通过直接偏好优化（DPO）对"高估置信度导致过度执行"和"低估置信度导致过度请求"的行为分别施加惩罚，纠正系统性偏差。在 OS-Kairos、AITZ、Meta-GUI、AndroidControl 四项主流 benchmark 上全面达到目前最优（SOTA），离线任务成功率平均提升超 17%，真实场景提升 26%。</p>

<h4>ProtLiD²（蛋白质设计扩散模型）：清华 Zhiyuan Liu 团队提出配体条件化离散扩散模型，联合设计蛋白质序列与结构，口袋设计通过率提升近 4 倍</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27413">ProtLiD²: Ligand-conditioned Discrete Diffusion for Protein Co-design</a></li>
  <li><b>信号源</b>：<b>清华 Zhiyuan Liu</b>（合作）· Wei Chen（一作）· Yang Zhang（通讯）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>蛋白质结合口袋（binding pocket）设计领域过去主流是序列和结构分开优化，或用独立的对接（docking）工具验证。ProtLiD² 这篇论文表明：把配体的化学和三维几何信息直接注入离散扩散过程，序列和结构协同生成，在严格对接阈值下配体通过率从 6% 跃升到 23%。<b>这意味着"序列设计和结构验证两步走"的路线作为口袋设计的标准流程，效率上会被联合生成方案显著压制</b>——如果这个结果在更多靶点上被第三方复现，依赖独立对接工具作为筛选瓶颈的药物发现管线需要重新考虑流程整合节奏。</p>
</callout>

<p>　　功能蛋白质设计，尤其是针对特定小分子配体的结合口袋设计，是药物发现前端最核心的工程挑战之一。现有方法普遍存在序列和结构优化目标不一致的问题：先设计序列、再折叠验证结构、再用对接工具检验配体亲和力，三步之间信息没有闭环传递，容易在后期验证阶段大量淘汰。</p>
<p>　　ProtLiD² 以掩码离散扩散（masked discrete diffusion）为骨架，在氨基酸序列 token 和离散结构 token 的联合去噪过程中，通过几何感知交叉注意力（geometry-aware cross-attention）将配体的化学特征和三维坐标实时融入生成过程。推理阶段引入最大置信度边界引导的 ReMask 解码（maximum confidence-margin guided ReMask）：保留置信度高的 token 预测，对不确定的 token 重新掩码再去噪，形成自校正循环。在全蛋白设计上，TM-score 从 0.672 提升至 <b>0.802</b>；口袋共设计的 active-site BB-RMSD 从 3.46Å 降至 <b>1.97Å</b>；严格对接阈值下 ligand-aware 通过率从 6.08% 提升至 <b>23.49%</b>，超越 Complexa、FAIR、PocketGen 等现有方法。</p>

<p><em>* 注：上述结果来自论文自披露，实验在标准 benchmark 上完成，但尚无独立第三方对真实靶点的验证数据。</em></p>

<h4>CARE：USC Xuezhe Ma 团队提出社区感知反应评估框架，揭示 LLM 模拟在线社区话语存在持续性"真实差距"</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.27388">CARE: Community-Aware Reaction Evaluation for LLM Discourse Simulation</a></li>
  <li><b>信号源</b>：<b>USC Xuezhe Ma</b>（资深合作）· Nuan Wen（一作）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>LLM 在角色扮演和社区模拟场景中的应用越来越广，但"模型能不能真实再现特定亚文化群体的话语风格"这个问题从未被系统测量过。CARE 这篇论文的核心发现是：即使给 LLM 明确的社区提示词，也无法从根本上提升话语模拟的真实度——不同前沿模型表现出各异的行为偏差，说明当前的对齐（alignment）训练目标没有覆盖在线社区的社会语言多样性。<b>这意味着依赖 LLM 做社交媒体用户模拟（如用户研究、政策影响预测）的应用场景，其输出结果的可靠性需要被重新评估</b>；针对特定社区语言风格的定制化微调需求会随之上升。</p>
</callout>

<p>　　LLM 越来越多地被用于模拟用户行为、预测社区反应或生成定向内容，这些应用隐含地假设模型能真实再现特定群体的语言偏好和态度。但亚文化社区的话语有独特的言外之意（illocutionary tone）和态度分布，这类信息往往不在通用预训练数据里——或者即使有，也会被对齐训练平均掉。</p>
<p>　　CARE 框架以"反应中心"设计为核心：收集真实社区对新闻事件的回应，用人机协作方式标注细粒度的言外之意分类（illocutionary tone），再让 LLM 生成对应回复，比较两者的态度分布。实验结果揭示 LLM 在社区语言模拟上存在系统性偏差，显式社区提示词工程效果有限；前沿模型之间的行为差异说明对齐策略未能充分覆盖在线社区的社会语言学多样性。</p>
