<h3>OpenAI 通用推理模型攻克 Erdős 1946 年平面单位距离猜想</h3>
<ul>
  <li><b>信号源</b>：<b>Rohan Paul</b>（独立评论）</li>
  <li><b>原文</b>：<a href="https://x.com/rohanpaul_ai/status/2057354887185481889">x.com/rohanpaul_ai/status/2057354887185481889</a></li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>过去几乎所有数学 AI 突破都依赖专用形式化证明器（如 Lean + AlphaProof），这次 <b>OpenAI</b> 用的是通用推理模型——凭借足够多的推理时间（test-time compute），跨越平面几何与代数数论（infinite class field towers、Golod–Shafarevich theory）两个本来几乎不相交的数学分支，在 Erdős 悬置近 80 年的猜想上实现多项式量级的构造改进。证明已由外部数学家独立核验，这排除了"模型幻觉"的可能性。</p>
  <p>这件事的格局影响落在两个层面：(1) <b>"通用推理 > 专用形式化"窗口已打开</b>——对 AI for Math 赛道，此前估值逻辑"只有专用形式化路线才能产出可验证数学成果"需要修正，通用 LLM + 充足 inference 预算这条路线的可行性大幅上升；(2) 这是 <b>OpenAI</b> 旗舰推理模型在数学研究级别的首次公开背书，对于押注 AI 科学发现（AI for Science Discovery）主题的基金来说，"AI 能否产出人类未知知识"这一核心假设今天得到了第一个强外部验证。</p>
  <p>谁该关心：专注数学/理论物理方向的 AI for Science 初创，以及依赖"专用系统不可替代"护城河的形式化验证工具赛道。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈专注 AI for Math 方向的团队，重点看其对"通用推理 vs. 专用形式化"技术路线的判断是否已更新。</li>
    <li>重新评估形式化证明工具（如 Lean 生态相关创业）的护城河假设——通用 LLM 的 test-time compute 能否继续蚕食其独特性？</li>
    <li>跟进 OpenAI 后续是否公开该结果的技术报告，用于判断其推理模型在数学以外领域（理论物理、组合优化）的泛化能力。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>OpenAI 的通用推理模型（非专用定理证明器）通过分配更多 test-time compute，在平面单位距离问题上找到多项式量级改进的无限构造族，成功推翻 Erdős 1946 年提出的猜想。该突破需要跨越平面几何与代数数论两个领域（涉及 infinite class field towers 和 Golod–Shafarevich theory），证明已由外部数学家独立核验，排除了模型幻觉的可能性。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：使用 OpenAI 通用推理模型，通过扩大 test-time compute 探索跨学科数学证明空间，而非依赖专用形式化证明框架。</li>
  <li><b>核心突破</b>：在平面单位距离问题上找到多项式量级改进的无限构造族，推翻 Erdős 1946 年猜想。</li>
  <li><b>外部验证</b>：证明已由外部数学家独立核验，确保结果真实有效。</li>
</ul>

<h3>IXT：将后训练质量反馈嵌入预训练，计算效率提升 2.8x</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20285">arxiv.org/abs/2605.20285</a></li>
  <li><b>信号源</b>：<b>Yejin Choi</b> · Brandon Cui（共一）· University of Washington Ximing Lu（共一）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>pre-training 与 post-training 一直是两个割裂的阶段：RLHF（人类反馈强化学习）/DPO（直接偏好优化）等质量信号只在训练结束后才注入。IXT 的关键一步是：让 thinking reward model 对每条预训练数据生成自然语言批注并 prefix 到样本前，从而把"质量感知"提前到预训练的第一个 token——凭借的是 offline reward-conditioned RL 的范式迁移，不需要额外优化器，直接嵌入现有 pre-training pipeline。</p>
  <p>格局影响：<b>scaling curve 向左弯折 2.8x</b> 意味着同等性能只需原来约 36% 的算力，这对 pre-training 算力消耗极大的基础模型厂商是直接的成本冲击。依赖"烧更多算力换性能"护城河的小型基础模型团队的卡点因此被部分解除——他们可以用更小的算力预算接近甚至超越此前需要大得多的计算量才能达到的性能上限。</p>
  <p>对已投/关注的标的：pre-training 数据质量筛选赛道（如专注 data curation 的公司）的价值可能被重估——IXT 表明质量信号不只在数据选择阶段有效，在训练过程动态注入同样有效，两种方式可能互补。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈专注 pre-training data curation 或 training efficiency 的团队，了解其对"动态质量注入 vs. 静态数据筛选"技术路线的判断。</li>
    <li>追踪 IXT 是否被 Meta / Mistral 等开源社区复现——若复现成立，pre-training 效率格局将显著改变，头部基础模型的成本优势护城河需重估。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>IXT（Introspective Training）受 offline reward-conditioned RL 启发，引入 thinking reward model 对所有预训练数据自动生成自然语言批注（critique），以 prefix-conditioning 方式拼接到原始样本前进行训练，使模型从预训练阶段就能感知数据质量差异。在 7.5-12B 规模 dense LLM 上从头训练至 18 万亿 token 的完整实验验证了方法有效性，整体计算效率最高提升 2.8x，并在 math、code 等领域突破了普通训练方式的性能上限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：thinking reward model 对每条预训练数据逐样本生成自然语言 critique，以 prefix 形式拼接到输入序列前，进行 prefix-conditioning 条件化语言模型训练，无需在线采样或额外 RL 优化器。</li>
  <li><b>技术细节</b>：prefix 携带结构化质量信号而非任务描述，使 next-token prediction 从一开始就是有条件的，不等权对待所有 token；整个流程可无缝嵌入现有训练 pipeline 任意阶段。</li>
  <li><b>主结果</b>：在 7.5-12B dense LLM 训练至 18 万亿 token 的实验中，计算效率最高提升 2.8x，scaling curve 明显向左弯折。</li>
  <li><b>关键结果点</b>：在 math 和 code 等专项领域达到普通训练方式无法触及的性能上限；方法适用于 pre-training、post-training 等全部训练阶段，具备通用性。</li>
</ul>

<h3>AI 评审 vs. 人类评审：469 小时大规模标注研究揭示 GPT-5.2 超越最高分人类审稿人</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20668">arxiv.org/abs/2605.20668</a></li>
  <li><b>信号源</b>：<b>Xiang Yue</b> · Seungone Kim（一作）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>此前关于 AI 评审的研究大多只看"判决一致性"（AI 接受/拒绝预测是否与人类一致），这项研究改为逐条分析每位审稿人的批评质量——这类似于不看法院判决结果统计而去逐条分析每位法官的裁决理由。结论比"AI 评审通过率"一类的粗粒度指标更有实际意义。</p>
  <p><b>GPT-5.2</b> 综合得分超越最高分人类审稿人（<b>60.0% vs. 48.2%</b>），且能发现 <b>26%</b> 人类未提及的独特问题——但三个 AI 审稿人之间重叠率高达 <b>21%</b>（人类仅 <b>3%</b>），这揭示了一个关键结构性缺陷：AI 审稿人本质上是高度同质化的群体，同时购买三个 AI 审稿服务和购买一个的覆盖度几乎一样。</p>
  <p>投研意涵指向"AI 辅助科学同行评审"工具赛道：单纯以"AI 替代人类审稿人"为卖点的产品护城河脆弱——差异化应建立在提升多样性（如 AI + 人类互补编排、减少同质化的 ensemble 策略）而非单纯替代上。关注该赛道初创时应重点问：其产品如何解决 AI 审稿的同质化问题？</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈面向学术出版社或科研机构的 AI peer review 产品团队，重点考察其多样性策略（人机互补编排、多模型 ensemble 去重）。</li>
    <li>追踪该 benchmark（基于 Nature 系列论文）是否被其他机构的审稿工具用于独立测试，以验证 GPT-5.2 结果的普遍性。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>本研究招募 45 位物理、生物与健康科学领域专家，耗时 469 小时，对 82 篇 Nature 系列论文的 AI 评审（GPT-5.2、Gemini 3.0 Pro、Claude Opus 4.5）与人类评审生成的 2,960 条具体批评意见进行大规模标注，评估维度涵盖正确性、重要性和证据充分性。研究发现 GPT-5.2 驱动的评审智能体综合得分超过每篇论文评分最高的人类审稿人（60.0% vs. 48.2%），但 AI 审稿人之间高度同质化，存在 16 类系统性弱点，综合结论为 AI 评审应作为人类评审的补充而非替代。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：将每条评审拆解为针对论文某一具体方面的"批评单元"，由 45 位领域专家逐条按正确性、重要性、证据充分性三维度评分，并对所有审稿人进行相对排名，避免仅看总分的粗粒度评估。</li>
  <li><b>技术细节</b>：跨审稿人重叠率通过计算不同审稿人提出相同批评的比例衡量，类似于测量多个检索系统"查全率"与"去重后查全率"之差。</li>
  <li><b>主结果</b>：GPT-5.2 综合评分超越最高分人类审稿人（60.0% vs. 48.2%，p=0.009）；AI 能发现 26% 人类未提及的独特问题。</li>
  <li><b>关键结果点</b>：三个 AI 审稿人之间批评重叠率高达 21%（人类仅 3%）；AI 存在 16 类系统性弱点，包括子领域知识不足、多文件长文本管理缺失、对次要问题过度批评等。</li>
</ul>

<h4>IXT：Introspective Training — 后训练质量反馈嵌入预训练，计算效率提升 2.8x</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20285">arxiv.org/abs/2605.20285</a></li>
  <li><b>信号源</b>：<b>Yejin Choi</b> · Brandon Cui（共一）· University of Washington Ximing Lu（共一）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>pre-training 与 post-training 的隔离一直是模型训练效率的核心痛点——RLHF/DPO 等质量信号只在训练结束后才注入，此前的算力消耗无法感知数据质量差异。IXT 通过将 thinking reward model 生成的 critique 以 prefix 形式嵌入每条预训练样本，把质量感知提前到训练第一步，相当于让 scaling curve 在同等算力下向左压缩。</p>
  <p><b>2.8x 计算效率提升</b>意味着同等性能只需约 36% 的算力预算，这对依赖"烧算力换性能"护城河的基础模型团队构成直接挑战。pre-training data curation 赛道需重估：IXT 表明质量信号在训练过程动态注入与静态数据筛选是互补而非替代关系，两者结合可能是下一代 pre-training 的主流路线。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 IXT 是否被开源社区复现（Meta / Mistral 等），若复现成立则需修正"大算力规模即护城河"的判断。</li>
    <li>约谈专注 pre-training data curation 的团队，评估其是否已将动态质量注入纳入产品路线图。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>IXT 受 offline reward-conditioned RL 启发，引入 thinking reward model 对所有预训练数据逐样本生成自然语言 critique（含质量判断与改进建议），以 prefix-conditioning 方式拼接到原始样本前进行条件化语言模型训练，使模型从预训练阶段起便能感知 token 质量差异。在 7.5-12B dense LLM 上从头训练至 18 万亿 token 的完整实验中，整体计算效率最高提升 2.8x，并在 math、code 等领域突破了普通训练方式的性能上限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：thinking reward model 对每条数据生成自然语言 critique 作为 prefix，进行 prefix-conditioning 训练；整体框架借鉴 offline reward-conditioned RL（类似 Decision Transformer），无需在线采样或额外 RL 优化器，可无缝嵌入现有训练 pipeline 任意阶段。</li>
  <li><b>技术细节</b>：prefix 携带结构化质量信号，使 next-token prediction 从一开始就是有条件的，不等权对待所有 token；实验使用从头训练的 transformer dense LLM（7.5B 到 12B 参数），token 消耗最多达 18 万亿。</li>
  <li><b>主结果</b>：计算效率最高提升 2.8x，scaling curve 明显向左弯折。</li>
  <li><b>关键结果点</b>：在 math 和 code 等领域达到普通训练方式无法触及的性能上限；方法适用于 pre-training、post-training 等全部训练阶段，具备通用性。</li>
</ul>

<h4>AIR：Anchor Invariance Regularization — 让 LLM 安全行为不随 prompt 表述方式改变</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20994">arxiv.org/abs/2605.20994</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b> · Yixu Wang（一作）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>LLM 安全对齐的核心漏洞之一是"对抗性改写"——用换一种说法的相同有害意图绕过安全机制。AIR 的核心洞察是：可验证 prompt（如多选题）的反馈质量远高于开放式 prompt 依赖的 reward model，因此用前者作为锚点单向正则化后者，避免了双向对称正则化"把好的拉低到差的水平"的问题。分布外一致性提升 <b>33.49%</b> 是一个实质性指标，说明对抗性改写场景下的鲁棒性有了可量化的改善。</p>
  <p>对 AI safety/alignment 工具赛道：AIR 以即插即用辅助 loss 形式接入 GRPO（Group Relative Policy Optimization）等主流框架，部署成本低，但其在生产环境中的对抗鲁棒性需要更多红队测试验证。当前更适合作为企业安全部署的辅助手段，而非核心防御层。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 AIR 是否被主流安全对齐工具集成（如 Anthropic Constitutional AI 生态），判断其成为行业标准的可能性。</li>
    <li>评估依赖 reward model 质量做对齐的创业标的，其护城河是否因 AIR 类方法（降低对 RM 质量的依赖）而被削弱。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>AIR（Anchor Invariance Regularization）将训练 prompt 分为可验证变体（如多选题，有确定性反馈）和开放性变体（依赖 reward model）两类，以可验证 prompt 的输出作为 stop-gradient 锚点，仅对开放性变体施加单向正则化，使其向锚点对齐。整体框架以辅助 loss 形式接入 GRPO（Group Relative Policy Optimization）等 group-based preference optimization 流程，通过 heterogeneous prompt grouping 联合训练，在 Safety、Moral Reasoning、Math 任务上，分布内 group accuracy 提升 12.71%，分布外一致性提升 33.49%。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：将 prompt 分为 anchor（可验证）和开放式两类，以 stop-gradient 机制截断锚点端梯度，单向驱动开放端向锚点对齐；以辅助 loss 插入现有 GRPO 训练流程，无需修改主优化逻辑。</li>
  <li><b>技术细节</b>：stop-gradient 机制是关键——只有开放端参数被更新，锚点端固定作为"精密量具"，避免对称正则化导致的性能退化。</li>
  <li><b>主结果</b>：分布内 group accuracy 提升 12.71%，分布外一致性提升 33.49%。</li>
  <li><b>关键结果点</b>：方法在 Safety、Moral Reasoning、Math 三类任务上均有效，具备一定通用性；对对抗性表述更加鲁棒。</li>
</ul>

<h4>AGPO：自适应策略优化改进 GRPO，数学推理训练稳定性提升</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20722">arxiv.org/abs/2605.20722</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b> · Miaobo Hu（一作）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>GRPO（Group Relative Policy Optimization）已成为 LLM post-training 主流框架之一，但固定超参数（clip 幅度、解码温度）在不同训练阶段和任务上表现不稳定。AGPO 通过 probe rollout 派生的统计量动态调节这两个关键超参数，核心价值是减少人工调参成本，提升跨模型/跨任务的训练稳定性。在 <b>Qwen2.5-14B</b> 上的结果（GSM8K 67.3%、MATH 40.5%）说明增益是真实的，且在 <b>Llama-3-8B</b> 和 <b>Gemma-2-9B</b> 上均可复现，方法通用性有初步保证。</p>
  <p>对 LLM 训练基础设施赛道：自适应优化器类方法的价值在于降低专家调参门槛，使更多中小团队能稳定复现大模型 post-training 效果；但其差异化相对薄弱，更可能成为开源工具集成而非独立商业化产品。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 AGPO 是否被 vLLM / TRL 等主流训练框架纳入标准配置，若是则说明其实用价值得到验证。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>AGPO（Adaptive Group Policy Optimization）在 GRPO 基础上引入双统计反馈机制：adaptive clipping 控制器根据 reward 离散度、偏斜度、probe vote entropy、policy entropy 和 KL drift 动态设定 trust region 大小；bidirectional adaptive temperature sampling 根据不确定性相对 running baseline 的高低动态升降解码温度。两个控制器共享 probe 派生的统计状态，不引入额外 critic 网络。在 Qwen2.5-14B 上 9 个数学/STEM benchmark 全面超越 PPO 和 GRPO，GSM8K 达 67.3%，MATH 达 40.5%，且在 Llama-3-8B 和 Gemma-2-9B 上验证了泛化性。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：GRPO 的 critic-free 改进，核心为共享 probe 统计状态驱动的两个控制器：adaptive clipping 动态确定 trust region；bidirectional adaptive temperature sampling 动态调节解码探索程度。</li>
  <li><b>技术细节</b>：probe-derived statistical state 汇集 reward 均值、方差、偏斜、entropy 及 step-wise KL drift，为每步更新实时调整 PPO-style clip 范围和采样温度，类似自动调节实验室培养箱的温控系统。</li>
  <li><b>主结果</b>：Qwen2.5-14B 在 9 个数学/STEM benchmark 上持续优于 PPO 和 GRPO；GSM8K 67.3%，MATH 40.5%。</li>
  <li><b>关键结果点</b>：消融实验确认两个模块各自独立贡献，效果互补；在相同 token 预算下优势成立。</li>
</ul>

<h4>PF+BV：伪形式化证明格式 + 逐块验证，超越 LLM-as-judge 基线</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20531">arxiv.org/abs/2605.20531</a></li>
  <li><b>信号源</b>：<b>Stanford Tengyu Ma</b> · Stanford Slim Barkallah（共一，导师 Tatsunori Hashimoto）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>数学证明验证有两个极端：完全形式化（Lean 等，精确但需要大量标注）和直接让 LLM 判断（灵活但不可靠）。PF+BV 在中间开了一条路——把自然语言证明重构为"自包含模块"后逐块验证，每块有明确的输入/输出签名，类似编译器的逐函数类型检查。在奥林匹克和研究级数学两个基准上 Pareto 优于 LLM-as-judge，且发布了 ArxivMathGradingBench 供后续对比。</p>
  <p>对 AI for Math 评测赛道的意义：PF+BV 不依赖 Lean，但比纯 LLM 判断更可靠，部署门槛低。若此方法被学术期刊或 arXiv 预印本平台采纳，将为 AI 辅助同行评审提供一个可落地的数学证明验证子模块——与 AI peer review 工具赛道的应用场景高度相关（参见头条 4 的 AI 评审信号）。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>跟踪 ArxivMathGradingBench 被其他团队使用的情况，判断该基准是否成为数学证明验证的社区标准。</li>
    <li>评估 AI 辅助论文评审产品团队是否已将数学证明验证纳入功能模块，PF+BV 可作为低成本技术插件。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PF+BV 在完全形式化证明语言（如 Lean）与自然语言证明之间架起桥梁：首先由 LLM 将自然语言证明翻译为 Pseudo-Formal（PF）格式——每个模块包含明确的 premises、conclusion 和 sub-proof；然后 Block Verification（BV）算法对各模块独立调用 LLM 验证局部正确性，避免跨步骤错误传播。在奥林匹克和研究级数学两个基准上，PF+BV 在错误发现精确率和召回率双指标上 Pareto 优于 LLM-as-judge 基线，并配套发布了 ArxivMathGradingBench 评测集。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：两步流程——LLM 翻译自然语言证明为 PF 格式（显式声明每模块的前提与结论）；BV 算法对各模块独立调用 LLM 验证，汇总模块级错误信号。</li>
  <li><b>技术细节</b>：PF 的核心设计是"自包含模块"——每块显式列出所有前提，类似函数签名，使 LLM 验证时无需追溯上下文，避免隐式依赖和歧义。</li>
  <li><b>主结果</b>：PF+BV 在奥林匹克和研究级数学验证基准上，错误发现 precision 和 recall 双指标 Pareto-dominate LLM-as-judge 基线。</li>
  <li><b>关键结果点</b>：开源 ArxivMathGradingBench 基准，支持后续研究对比；整个流程不依赖 Lean 等形式语言，仅需 LLM 能力。</li>
</ul>

<h4>Gated DeltaNet-2：解耦 erase/write gate，linear attention 长上下文检索显著提升</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22791">arxiv.org/abs/2605.22791</a></li>
  <li><b>信号源</b>：<b>Yejin Choi</b> · NVIDIA Ali Hatamizadeh（一作）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>linear attention 的竞争力能否持续取决于能否在固定大小的 recurrent state 里做更精细的记忆管理。Gated DeltaNet-2 将原来耦合在一个标量门控里的"擦除旧内容"与"写入新内容"拆开，各用独立的 channel-wise gate 管理——这使得模型可以在不破坏已有记忆的前提下精准写入新信息，长上下文多键检索（RULER needle-in-a-haystack）提升最为显著。</p>
  <p>在 <b>1.3B 参数 / 100B FineWeb-Edu token</b> 设置下综合超越 Mamba-2、Gated DeltaNet、KDA、Mamba-3，说明这一改进在主流训练规模下是真实有效的。对 linear attention / SSM 架构赛道（如 Mamba 生态相关创业）：Gated DeltaNet-2 进一步压缩了 Transformer 在长上下文推理上的优势窗口，SSM 类架构的 inference 成本优势+性能差距正在收窄。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 NVIDIA 是否将 Gated DeltaNet-2 整合进其内部训练框架，判断其从论文到产品化的时间线。</li>
    <li>评估押注 Mamba/SSM 架构的初创标的，其技术路线是否会受 Gated DeltaNet-2 影响而需要更新。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Gated DeltaNet-2 在 delta-rule 框架基础上引入 channel-wise erase gate b_t 和 write gate w_t，分别控制对 key 侧旧内容的擦除幅度和 value 侧新内容的写入幅度，同时继承 KDA 的 channel-wise decay 自适应遗忘机制。作者推导了 fast-weight update 视角、含 channel-wise decay 的 chunkwise WY 算法及 gate-aware backward pass。1.3B 参数在 100B FineWeb-Edu token 上训练的综合性能超越 Mamba-2、Gated DeltaNet、KDA 和 Mamba-3，在长上下文 RULER 多键检索任务上提升最为显著。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：在 Gated DeltaNet 基础上引入独立的 channel-wise erase gate b_t 和 write gate w_t，将 Gated DeltaNet 与 KDA 统一为特例，解决标量门耦合问题；推导支持高效并行训练的 chunkwise WY 算法和 gate-aware backward pass。</li>
  <li><b>技术细节</b>：channel-wise decay 被吸收进非对称 erase 因子，保持矩阵运算块状结构不变，避免额外计算开销；gate-aware backward pass 显式追踪 b_t 和 w_t 的梯度路径，避免标量近似带来的梯度偏差。</li>
  <li><b>主结果</b>：1.3B 参数综合性能超越 Mamba-2、Gated DeltaNet、KDA、Mamba-3。</li>
  <li><b>关键结果点</b>：在长上下文 RULER needle-in-a-haystack 多键检索任务上提升最为显著；recurrent 和 hybrid 设置均保持强劲。</li>
</ul>

<h4>CLORE：内容级优化推理效率，删除冗余推理步骤</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22211">arxiv.org/abs/2605.22211</a></li>
  <li><b>信号源</b>：<b>Manling Li</b> · Yuyang Wu（共一）· Qiyao Xue（共一）· CMU Olexandr Isayev（通讯）</li>
</ul>
<callout emoji="💡">
  <p><b>Our Insights</b></p>
  <p>当前"高效推理"方法大多通过长度惩罚或 token 预算约束来压缩 chain-of-thought，但这会导致模型删去部分有效推理步骤。CLORE 直接对内容质量动刀：用外部增强模型删除重复段落、不可读内容和答案后多余探索，再以 reference-free DPO（直接偏好优化）辅助目标与 GRPO/DAPO 联合训练。关键在于编辑操作仅限正确轨迹的局部删除，off-policy mismatch 最小。</p>
  <p>对 LLM reasoning efficiency 赛道：CLORE 的差异化在于"内容质量 vs. 纯长度约束"，但其外部增强模型（负责内容编辑）的质量是瓶颈——需要一个能精准识别冗余推理的模型，这本身就依赖强大的 LLM 判断能力，存在循环依赖风险。更适合作为训练辅助工具而非独立推理压缩方案。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 CLORE 与 ThinkPrune 等长度压缩方法的组合效果，判断两类方法是否互补。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>CLORE 提出内容级推理优化框架，分两阶段：首先用外部增强模型对当前策略采样的正确 rollout 进行局部删除（去除重复片段、不可读内容、答案后冗余推理），生成增强版轨迹；然后将增强-原始对作为偏好数据，配合 reference-free DPO（直接偏好优化）辅助目标与 GRPO/DAPO 等主流 policy-gradient 方法联合优化。在 DeepSeek-R1-Distill-Qwen-7B 和 Qwen2.5-Math-7B 上跨五个数学推理基准验证有效性，与多种现有方法兼容。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：外部增强模型对正确 rollout 进行局部删除编辑，保留最终答案不变；增强-原始对通过 reference-free DPO 辅助目标与标准 policy-gradient 联合训练。</li>
  <li><b>技术细节</b>：增强操作仅限正确轨迹上的局部删除，保证编辑后样本仍落在策略分布附近，最小化 off-policy mismatch；DPO 目标无需参考模型，以原始 rollout 为隐式参考。</li>
  <li><b>主结果</b>：在五个数学推理基准上改善了准确率与效率的权衡。</li>
  <li><b>关键结果点</b>：显著减少重复推理、不可读内容和答案后探索行为；与 GRPO、DAPO、Training Efficient、ThinkPrune 均兼容。</li>
</ul>

<h4>ECPO：候选排序与可验证 evidence certificate 联合优化</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21993">arxiv.org/abs/2605.21993</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b> · Miaobo Hu（一作）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>企业级 RAG（检索增强生成）和决策支持系统的核心痛点之一是"模型给出答案但无法溯源"，ECPO 将候选排序与可独立核查的文本 span 证据联合优化，通过无标签确定性 verifier 自动计算证据有效性奖励，无需人工标注。CertNDCG 指标将排序质量与证据有效性合并度量，是一个更贴近实际部署需求的评估框架。</p>
  <p>对企业 AI 基础设施赛道：可解释性是企业买家的核心关切，ECPO 的"结论+证据联合输出"模式直接对接这一需求，比纯排序模型有更强的差异化卖点；但其在 MAVEN-ERE 和 RAMS（事件抽取数据集）上的验证范围较窄，泛化到其他企业场景（如法律、金融）的可行性需要进一步验证。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 ECPO 是否在法律/医疗等强监管行业的 RAG 应用场景中被采用，判断其可解释性差异化能否转化为商业价值。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>ECPO（Evidence-Coupled Policy Optimization）是一种 listwise policy optimization 框架，将候选排序与 evidence certificate 定义为联合 action 空间，使用三重耦合 reward（listwise ranking utility、span-level certificate validity、evidence-cycle reward）由无标签确定性 verifier 自动计算，并引入 CertNDCG 指标同时衡量排序质量与证据有效性。在 MAVEN-ERE 和 RAMS 固定上游抽取设置下，ECPO 相比 zero-shot、SFT、GRPO 及 RM-only 等多个基线表现更优。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：listwise policy optimization，action 空间为排序列表与 evidence certificate 的联合对象；三项耦合 reward 由 label-free 确定性 verifier 自动计算，无需人工标注。</li>
  <li><b>技术细节</b>：evidence-cycle reward 的关键是无需人工标签的确定性 verifier——将模型引用的文本 span 去除 claim 语义后尝试重建原始候选决策支持，重建成功则给予正向 reward，类似"自证"检验。</li>
  <li><b>主结果</b>：在 MAVEN-ERE 和 RAMS 上的 CertNDCG 指标超越全部对比方法。</li>
  <li><b>关键结果点</b>：实验涵盖 closed-roster、predicted-roster 和 hybrid-roster 三种场景；对比了多种竞争方法，ECPO 全面占优。</li>
</ul>

<h4>AttuneBench：基于真实多轮对话的 LLM 情感智能基准</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21739">arxiv.org/abs/2605.21739</a></li>
  <li><b>信号源</b>：<b>OpenAI Karina Nguyen</b>（资深合作）</li>
</ul>
<callout emoji="💡">
  <p><b>Our Insights</b></p>
  <p>现有 LLM 情感智能评测依赖合成提示或单轮对话，AttuneBench 用真实用户与匿名 LLM 交互的多轮对话数据构建评测，更贴近实际部署场景。最关键的发现是：情感智能的四个子能力（情绪识别、行为分类、偏好预测、响应质量）排名基本相互独立——这打破了"情感智能是单一能力"的隐含假设。</p>
  <p>对 AI 伴侣/心理健康/客服类产品赛道：偏好对齐比情绪标签准确率对模型的区分度更高，意味着"是否能精准预判特定用户在特定上下文需要什么类型的响应"才是产品差异化的核心能力，而非通用的情绪识别精度。这是一个对选型和评估框架有直接意义的洞察。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈 AI 伴侣/心理健康应用团队，了解其产品评测体系是否已超越情绪识别准确率，重点看偏好对齐能力的设计。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>AttuneBench 基于 200 段真实多轮人-模型对话构建 LLM 情感智能基准，参与者与匿名 LLM 对话后逐轮标注自身情绪状态、模型行为及偏好响应，覆盖情绪识别、行为分类、偏好预测和响应质量评判四个维度。对 11 个主流 LLM 的评测发现，四个子任务排名基本相互独立，情感智能由多个可分离子能力组成；偏好对齐与响应质量比情绪标签准确率对模型的区分度更高。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：基于 200 段真实多轮对话，参与者自我标注情绪状态和偏好响应（避免第三方标注偏差）；对 11 个 LLM 在四个子任务上独立评分后分析各维度排名相关性。</li>
  <li><b>技术细节</b>：多轮对话设计捕捉情绪状态随对话演进的动态变化；数据来自真实用户交互而非合成提示，提升生态效度。</li>
  <li><b>主结果</b>：11 个模型在四个情感智能子任务上排名基本相互独立，情感智能非单一能力。</li>
  <li><b>关键结果点</b>：偏好对齐和响应质量是区分模型能力最有效的指标；情感智能核心在于预测特定用户在上下文中想要什么类型的响应。</li>
</ul>

<h4>VPO：向量奖励策略优化，改善测试时搜索多样性</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22817">arxiv.org/abs/2605.22817</a></li>
  <li><b>信号源</b>：<b>Stanford Omar Khattab</b> · MIT Ryan Bahlous-Boldi（一作）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>标准 scalar RL post-training 的一个鲜少被提及的副作用是：它会让模型输出分布趋于低熵，导致 test-time search（测试时搜索）时 rollout 缺乏多样性，best@k 指标随 k 增大提升有限。VPO 将奖励保留为向量形式（如代码生成中每个测试用例的通过情况），鼓励不同 rollout 特化到向量奖励空间的不同 Pareto 方向，从而在搜索预算增大时获得更大收益。</p>
  <p>对 LLM inference-time scaling 赛道：test-time search 是当前提升模型能力最热门的方向之一，VPO 作为 GRPO 的 drop-in 替换，在 evolutionary search 场景中能解决 GRPO 完全无法解决的问题，说明训练阶段的奖励结构设计对 inference-time scaling 效率有直接影响——这是一个被低估的耦合关系。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 VPO 在更大规模模型（70B+）上的复现结果，判断其在实际产品级推理场景中的可行性。</li>
    <li>关注 Stanford Omar Khattab（DSPy 作者）团队的后续工作，其在 LLM 系统级优化上的研究方向持续值得跟踪。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>VPO（Vector Policy Optimization）作为 GRPO advantage estimator 的 drop-in 替换，将通常被聚合为标量的奖励保留为向量形式（如代码生成中各测试用例的通过/失败情况，或多用户偏好评分），训练策略模型生成多样化解集，其中每个解专注于向量奖励空间中不同的 Pareto 方向，从而在 test-time search 时获得更高覆盖度。在 4 个任务的 pass@k 和 best@k 指标上匹配或超越最强 scalar RL 基线，搜索预算越大优势越显著；在 evolutionary search 场景中更能解锁 GRPO 完全失败的难题。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：替换 GRPO 的 advantage estimator，将标量奖励扩展为向量；在向量奖励空间中为每个 rollout 分配方向权重，鼓励不同 rollout 在不同维度表现突出，类似多目标优化的 Pareto front 分解。</li>
  <li><b>技术细节</b>：advantage 估算不再对向量奖励求均值后归一化，而是为每个 rollout 分配向量方向权重，自然维持输出分布的高熵。</li>
  <li><b>主结果</b>：4 个任务上 pass@k 和 best@k 匹配或超越 GRPO 等 scalar RL 基线，搜索预算增大时优势更显著。</li>
  <li><b>关键结果点</b>：在 evolutionary search 任务中能解决 GRPO 完全失败的问题。</li>
</ul>

<h4>Self-play RL 稳定性：数据门控是关键，奖励设计是次要因素</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22217">arxiv.org/abs/2605.22217</a></li>
  <li><b>信号源</b>：Sophia Xiao Pu（一作，导师 William Wang）</li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>几乎所有 self-play RL 论文都把奖励设计（reward shaping）视为核心变量，这项研究用受控实验给出了反直觉的结论：<b>data-level gate 才是决定训练稳定性的关键机制</b>，严格的任务准入过滤即使搭配最简单的 self-consistency reward 也足以稳定训练，去掉 gate 后任何奖励设计都无法阻止崩溃。更反直觉的是"Grounded Proposer Paradox"——让出题方访问真实答案反而加速崩溃约 10 倍。</p>
  <p>这对 self-play RL 训练的实践意义直接且可行：在工程层面，data gate 的严格程度应被视为比奖励函数设计更优先的超参数。对于正在构建 self-play 训练基础设施的团队，这条结论可以直接指导工程决策，降低因奖励设计不当导致训练崩溃的风险。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>将此结论传递给正在使用 self-play RL 训练的投资组合公司，建议其优先审查 data gate 的设计而非奖励函数。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>研究通过两个可控实验环境（Python 输出预测任务和确定性 DSL 双任务），系统分析 data-level gate 严格程度与 reward 类型对 self-play RL 训练稳定性的独立与交互影响。结论表明：严格的 data gate 在所有 reward 变体下均可维持稳定性，而移除 gate 后任何 reward 均无法阻止崩溃；同时发现"Grounded Proposer Paradox"——出题方拥有 ground-truth 访问权时，反而比无 grounding 的出题方更快导致崩溃（加速约 10 倍）。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：2×2 因子实验，独立操控 data gate 严格程度（二值及连续 ε 参数）与 reward 类型（ground-truth / self-consistency），在两个受控环境中系统分析。</li>
  <li><b>技术细节</b>：data gate 基于可解率阈值过滤——只有 solver 在 N 次采样中解出率落在一定区间内的任务才进入训练池；连续 ε 参数揭示两阶段相变：训练指标在低 ε 时已解耦，验证精度在更高 ε 才开始下降。</li>
  <li><b>主结果</b>：严格 data gate 是 self-play RL 稳定性的核心约束，reward 类型是次要因素。</li>
  <li><b>关键结果点</b>：Grounded Proposer Paradox——出题方有 ground-truth 访问权反而加速崩溃约 10 倍。</li>
</ul>

<h4>SWE-Mutation：用变异测试评估 LLM 生成测试套件质量</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22175">arxiv.org/abs/2605.22175</a></li>
  <li><b>信号源</b>：<b>Kai Zhang</b> · Zhejiang Chinese Medical University Yuxuan Sun（一作）· USTC Zhenya Huang（通讯）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>SWE-bench 及其衍生 benchmark 主要评估 LLM 修复 bug 的能力，SWE-Mutation 转而评估"LLM 生成的测试套件能否发现故意写错的程序"。即使最强模型 DeepSeek-V3.1 检测率仅 <b>36.15%</b>，说明当前 LLM 生成的测试套件普遍缺乏足够的判别力——这是 AI 辅助软件工程工具链的一个重要短板。</p>
  <p>对 AI coding 工具赛道：测试生成质量是 agentic coding 工具（如 Codex、Cursor 等）进入安全关键场景（金融、医疗软件）的关键门槛。SWE-Mutation 提供了一个可量化的评测维度，可以直接用于区分不同编码 agent 的测试生成能力，是一个潜在的企业买家选型工具。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>将 SWE-Mutation 用于评估已投/关注的 AI coding 工具标的，检验其测试生成能力是否达到安全关键场景的要求。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>SWE-Mutation 以软件工程 benchmark 中的原始实例为基础，通过 language-agnostic 的 agentic mutation 框架自动生成语义上接近正确但存在细微缺陷的代码变体（mutants），用这些变体作为"测试探针"量化 LLM 生成测试套件的判别能力。benchmark 包含 2,636 个变异体（来自 800 个原始实例），覆盖 9 种编程语言，评估指标包括 verification rate 和 detection rate。在 7 个主流 LLM 上的实验表明，最强模型 DeepSeek-V3.1 检测率仅 36.15%。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：language-agnostic agentic mutation 框架让 LLM 扮演"对抗性程序员"，生成比传统语法规则 mutation 更难被简单测试识别的语义级变体；用 verification rate 和 detection rate 量化测试套件判别能力。</li>
  <li><b>技术细节</b>：agentic mutation 策略将平均检测率从 71.04% 降至 39.81%，比传统 mutation 方法更接近真实缺陷模式，说明生成的变体更具挑战性。</li>
  <li><b>主结果</b>：DeepSeek-V3.1 验证率仅 10.20%，检测率仅 36.15%，所有主流 LLM 均表现欠佳。</li>
  <li><b>关键结果点</b>：benchmark 覆盖 9 种编程语言的多语言子集，具备广泛适用性。</li>
</ul>

<h4>Deformba：自适应扫描策略 + 跨流融合，突破视觉状态空间模型固定扫描限制</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21308">arxiv.org/abs/2605.21308</a></li>
  <li><b>信号源</b>：<b>Jack Morris</b> · Hongyu Ke（一作）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>视觉 SSM（如 Mamba）的核心局限在于固定扫描顺序导致几何感知僵化，且无法支持跨信息流的查询式融合（如 BEV（鸟瞰图）感知需要多视角 cross-attention）。Deformba 通过可学习偏移量实现自适应采样（类似 Deformable Convolution 的思路），同时引入支持外部 query 流的 fusion 机制，在保持 SSM 线性复杂度的前提下同时解决两个问题。</p>
  <p>在 2D 分类/检测/分割与 3D BEV 感知均取得有竞争力的结果，说明方法的通用性有初步验证。对自动驾驶感知算法赛道：BEV 感知是当前端到端自驾的核心模块，能将 Mamba 类高效架构引入 BEV 场景的方法具有实际部署价值（推理延迟和内存效率是量产落地的关键指标）。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>评估自动驾驶感知算法赛道标的是否已将 SSM 类架构纳入 BEV 感知方案，Deformba 是否可作为其技术升级路径。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Deformba 在标准 SSM（Mamba）基础上引入两项改进：上下文自适应的空间位置增强模块，通过可学习偏移量对 patch 位置进行变形采样，替代固定扫描策略；以及跨流融合模块，允许不同信息流（如多视角图像）之间进行类似 cross-attention 的双向交互，支持 BEV 等查询驱动的 3D 感知任务。在 2D 图像分类、检测、分割及 3D BEV 感知多个基准上验证了通用性和有效性，同时维持 SSM 的线性时间复杂度。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：SSM 主干上引入可变形采样（deformable sampling）模块动态调整 patch 序列化的几何关系；设计 query-key-value 路径的跨流融合模块，让 SSM 隐状态可被外部 query 检索。</li>
  <li><b>技术细节</b>：可变形采样借鉴 Deformable Convolution 思路，通过可学习偏移量在 SSM 扫描路径上自适应选择空间上下文；fusion 模块类似给 RNN 补充可寻址的外部记忆接口。</li>
  <li><b>主结果</b>：在 2D 通用视觉任务（分类、检测、分割）及 3D BEV 感知任务上取得强竞争力的结果。</li>
  <li><b>关键结果点</b>：维持线性时间复杂度；在 BEV 感知等需要 cross-attention 的场景成功应用。</li>
</ul>

<h4>Flat-Pack Bench：家具组装视频细粒度时空理解基准，揭示 LVLM 的深层局限</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21625">arxiv.org/abs/2605.21625</a></li>
  <li><b>信号源</b>：<b>Cornell Noah Snavely</b> · Chetan Aditya（一作）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>当前视频理解基准大多测试粗粒度语义（"这段视频在干什么"），而具身智能和机器人操控需要的是细粒度时空推理（"第 3 步的零件 A 在哪里"、"这两个零件何时接触"）。Flat-Pack Bench 用 IKEA 式组装视频构建了这类细粒度任务，SOTA 模型表现显著差于粗粒度基准，暴露出视频理解模型在复杂操作场景下的核心短板。</p>

  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈专注机器人操控感知的团队，用 Flat-Pack Bench 类任务评估其 LVLM 的细粒度时空理解能力。</li>
  </ul>

<h5>摘要</h5>
<p>Flat-Pack Bench 以家具组装视频为核心，构建专门测试 LVLM（大型视觉语言模型）细粒度时空理解能力的基准，覆盖时序排序、状态定位、零件配合与追踪四类任务。每道题均为多选题，配有标注了相关零件的视觉提示图像。实验表明 SOTA LVLM 在细粒度时空推理任务上表现显著差于粗粒度任务，尤其在零件追踪和物理接触理解方面存在明显局限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：围绕四个子任务设计评测（时序排序、状态定位、零件配合、零件追踪），每道题配有视觉提示图像（高亮相关零件区域），支持细粒度问题的精准评估。</li>
  <li><b>技术细节</b>：视觉提示将空间理解与追踪难度从"找目标"与"理解交互"中解耦，精准探测模型时空推理短板。</li>
  <li><b>主结果</b>：SOTA LVLM 在细粒度时空推理任务中表现较差，尤其在零件追踪和物理接触理解方面。</li>
  <li><b>关键结果点</b>：揭示现有视频理解模型在复杂操作场景下的根本局限。</li>

<h4>Cambrian-P：相机位姿作为弱监督信号，视频 MLLM 空间推理提升 4.5-6.5%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22819">arxiv.org/abs/2605.22819</a></li>
  <li><b>信号源</b>：<b>NYU Saining Xie</b> · Jihan Yang（共一）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>video MLLM（视频多模态大模型）的核心短板是把视频帧当作孤立的 2D 快照，丢失帧间三维空间坐标信息。Cambrian-P 将相机位姿作为弱监督信号注入模型（凭借 per-frame learnable camera tokens + pose regression head），在 VSI-Bench（空间推理基准）上提升 <b>4.5-6.5%</b>，并在 ScanNet 流式位姿估计上达到 SOTA。</p>
  <p>更重要的是：使用野外视频伪标注位姿训练后，通用 video QA 性能也有提升，说明位姿信号的收益超出空间推理本身。这对具身 AI 和机器人操控赛道是直接利好——位姿信息是构建持久空间记忆（persistent spatial memory）的基础信号，而持久空间记忆是长时程操控任务的核心瓶颈之一。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 Saining Xie（NYU）团队的后续工作，Cambrian 系列在视频理解方向的进展值得持续跟踪。</li>
    <li>评估具身 AI 标的中 video MLLM 对位姿信号的利用程度，作为技术路线成熟度的判断维度。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Cambrian-P 在已有 video MLLM 骨架上为每帧附加一组可学习 camera tokens，并新增 pose regression head 联合预测相机位姿；通过精心设计的帧采样策略提供带位姿标注的训练信号，支持对野外视频使用伪标注位姿进行扩展训练。在 VSI-Bench 空间推理基准上提升 4.5-6.5%，在 8 个额外的空间及通用 video QA 基准上同步泛化，并在 ScanNet 流式位姿估计任务上达到 SOTA。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：per-frame learnable camera tokens 与视觉特征拼接后送入 LLM，pose regression head 以这些 tokens 为输入监督位姿回归损失；支持野外视频伪标注位姿的扩展训练。</li>
  <li><b>技术细节</b>：位姿头在训练时提供强几何约束，推理时无需显式位姿输入亦能激活模型的空间感知能力；类似在视频理解流水线中插入"隐式 GPS"模块。</li>
  <li><b>主结果</b>：VSI-Bench 空间推理基准提升 4.5-6.5%；ScanNet 流式位姿估计达到 SOTA。</li>
  <li><b>关键结果点</b>：使用野外视频伪标注位姿训练后，通用 video QA 性能也有提升，位姿信号收益超出空间推理范畴。</li>
</ul>

  <li><b>主结果</b>：1.8B 端侧版本综合表现超过 Microsoft、Doubao 等商业翻译 API；推理速度提升 1.5x。</li>

<h4>PDI-Net：光学先验融合红外重建与目标检测，推理时延降低 84%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21964">arxiv.org/abs/2605.21964</a></li>
  <li><b>信号源</b>：<b>Kai Zhang</b> · Tongji University Xinbin Cheng（通讯）</li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>PDI-Net 将红外计算成像（computational imaging）的重建网络与 YOLO 检测器深度融合，推理时跳过完整图像重建直接复用编码器特征，通过 PALS-Bridge 将场景相关 PSF（点扩散函数）先验注入多尺度卷积分支——在 M3FD 低信噪比条件下推理时延降低 <b>84%</b>，mAP@0.5:0.95 提升 <b>5.07%</b>。单镜头方案还减轻约 50% 系统重量，对无人机载荷和便携设备有实际价值。</p>

  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注红外感知（安防/防务/无人机）相关标的的算法路线，评估其是否已采用物理先验融合设计。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PDI-Net 采用训练/推理解耦策略：训练阶段使用完整监督 U-Net 进行图像重建；推理阶段仅保留 semi-U-Net 编码器与 YOLO-based 检测器共享特征，跳过解码器重建步骤。PALS-Bridge（physics-aware large-small bridge）模块以场景相关 PSF（点扩散函数）为先验，动态调制多尺度卷积分支权重，使网络感知光学模糊的空间变化特性。在 M3FD 低 SNR 条件下，推理时延比剪枝版 Rec+Det 基线降低 84.06%，mAP@0.5:0.95 提升 5.07%；单镜头部署方案较传统多镜头减轻约 50% 系统重量。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：训练/推理解耦：训练用完整 U-Net 提供充分监督，推理仅用 semi-U-Net 编码器与检测器共享特征，跳过解码器；PALS-Bridge 将 PSF 先验嵌入多尺度卷积调制。</li>
  <li><b>技术细节</b>：PALS-Bridge 让网络感知镜头在图像不同区域的成像质量差异，有针对性地修复和增强检测所需的语义特征；物理退化仿真 pipeline 用于训练数据生成。</li>
  <li><b>主结果</b>：推理时延降低 84.06%；mAP@0.5:0.95 提升 5.07%（M3FD 低 SNR 场景）。</li>
  <li><b>关键结果点</b>：单镜头方案减轻约 50% 系统重量；物理先验融合同时带来速度和精度双重收益。</li>
</ul>

<h4>PhysX-Omni：统一框架支持刚体/可变形体/关节体的仿真就绪物理 3D 生成</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21572">arxiv.org/abs/2605.21572</a></li>

<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>3D 生成工具的现有局限在于要么忽略物理属性，要么只支持单一物体类型，无法直接用于具身 AI 仿真训练。PhysX-Omni 将刚体、可变形体、关节体（articulated objects）三类物理 3D 生成统一在一个框架下，直接面向 simulation-ready 场景，凭借的是为 VLM（视觉语言模型）设计的高分辨率几何表示，避免了传统方法压缩几何特征导致物理属性预测失准的问题。</p>

  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 PhysXVerse 数据集和 PhysX-Bench 基准的开源进展，判断其对具身 AI 训练数据生成赛道的影响。</li>
    <li>评估具身 AI 仿真数据生成标的，了解其是否已将物理感知 3D 生成纳入数据流水线。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PhysX-Omni 以 VLM（视觉语言模型）为核心，设计了一种新型高分辨率几何表示，直接将 3D 结构编码输入 VLM 而不做压缩，在一个统一模型中同时处理刚体、可变形体和关节体的物理属性预测与 3D 生成。配套构建了首个通用 simulation-ready 3D 数据集 PhysXVerse（覆盖室内外多类别），以及涵盖 geometry、scale、material、affordance、kinematics、function description 六大维度的评测基准 PhysX-Bench。在常规指标和 PhysX-Bench 上均表现优越，并在 simulation-ready 场景生成和机器人策略学习扩展实验中展示了实际应用潜力。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：VLM 主干 + 新型高分辨率几何表示，直接编码 3D 结构而不压缩；数据侧构建 PhysXVerse，评估侧引入 PhysX-Bench 多维基准。</li>
  <li><b>技术细节</b>：高分辨率几何表示让 VLM 直接"看原图"而非"看缩略图"，保留细节以精准预测物理参数（材料、关节自由度等）；支持关节体（articulated objects）是与现有方法的关键差异。</li>
  <li><b>主结果</b>：在 PhysX-Bench 六维度评测上表现优越。</li>
  <li><b>关键结果点</b>：在机器人策略学习扩展实验中展示实际应用潜力；PhysXVerse 是首个通用 simulation-ready 3D 数据集。</li>
</ul>

<h4>DriveMA：单步语义决策标签替代自然语言推理链，Waymo 端到端挑战刷新 SOTA</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21273">arxiv.org/abs/2605.21273</a></li>
  <li><b>信号源</b>：Weicheng Zheng（一作）· Hang Zhao（通讯）</li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>驾驶 VLA（视觉-语言-动作模型）的自然语言推理链接口存在三个实际痛点：标注昂贵（高质量推理标注难以规模化获取）、推理慢（长链在实时场景中延迟不可接受）、小模型难以处理长推理链。DriveMA 用一步 meta-action（如"右转"、"跟车"等低熵语义决策标签）替代推理链，可从专家轨迹自动派生，无需额外标注。</p>
  <p>在 Waymo End-to-End Driving Challenge 上以 <b>2B</b> 和 <b>4B</b> 参数分别达到 RFS <b>8.060</b> 和 <b>8.079</b> 的新 SOTA，说明"简单接口 + 强化学习信用分配"组合在实际竞赛基准上已超越更复杂的方法。对自动驾驶 VLA 赛道：meta-action 接口大幅降低了 VLA 的标注门槛，是将语言模型推理能力低成本引入自驾决策层的可行路径。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 DriveMA 能否在更多驾驶基准（nuScenes / CARLA 等）上复现 SOTA，判断其泛化能力。</li>
    <li>评估自动驾驶 VLA 创业标的的接口设计，关注其是否已从复杂推理链转向低熵决策标签路线。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>DriveMA 以 one-step meta-action 作为 VLA（视觉-语言-动作模型）的语言接口，替代冗长的自然语言推理链。meta-action 是低熵的语义决策标签（如"直行"、"左转"），可从专家驾驶轨迹自动派生，无需额外高质量推理标注。训练采用 action-centric 监督学习与 turn-level credit-assignment 强化学习框架相结合，联合优化 meta-action 正确性、轨迹质量以及轨迹与 meta-action 的一致性三项目标。DriveMA 2B 模型在 Waymo End-to-End Driving Challenge 达到 RFS 8.060，4B 版本提升至 8.079，均为 SOTA。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：one-step meta-action 为语言接口；action-centric 监督 + turn-level credit-assignment RL 框架联合优化三项目标：meta-action 正确性、轨迹质量、轨迹与 meta-action 一致性。</li>
  <li><b>技术细节</b>：meta-action 从专家轨迹自动派生，支持可扩展监督；RL 框架逐决策步骤进行 credit assignment，避免端到端轨迹奖励的稀疏性问题。</li>
  <li><b>主结果</b>：Waymo End-to-End Driving Challenge：2B 达到 RFS 8.060，4B 提升至 8.079，均为 SOTA。</li>
  <li><b>关键结果点</b>：消融实验表明 meta-action 在表达力、可预测性和推理效率上优于自然语言推理链；NAVSIM 基准上也取得有竞争力的表现。</li>

<h4>stable-worldmodel (swm)：开源 world model 研究与评测统一平台</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21800">arxiv.org/abs/2605.21800</a></li>
  <li><b>信号源</b>：<b>Meta Yann LeCun</b> · Lucas Maes（共一）· Quentin Le Lidec（共一）</li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>world model 研究的碎片化问题（代码库分散、数据加载慢、缺乏标准评测）是阻碍该方向快速进展的结构性障碍。swm 通过统一数据层（Lance 格式高效视频加载）、标准化 baseline 和可控变因的泛化评测套件，为 world model 社区提供类似 Hugging Face 的基础设施——这件事由 <b>Meta Yann LeCun</b> 团队主导，信号权重较高。</p>
  <p>对 world model 赛道：标准化评测基础设施的建立往往是学术方向加速成熟的先导信号——当社区有了共同 benchmark，研究迭代速度会显著提升，商业化时间线也会相应提前。swm 的发布预示着 world model 从"各自为战"到"统一赛道"的转变可能加速。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 swm 在 world model 研究社区的采用率，判断其是否正在成为事实标准；若是，则 world model 相关赛道的商业化时间线需要提前考虑。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>stable-worldmodel（swm）将 world model 研究流程整合为三层统一框架：数据层（基于 Lance 列存格式实现高效视频数据加载，支持 MP4/HDF5/LeRobot 格式转换）、模型层（提供模块化 world model baseline 和 planning solver 规范化实现）、评测层（构建包含可控视觉/几何/物理变因的多环境评测套件，支持系统性分布外泛化评估）。当前论文以平台介绍为主，旨在解决 world model 研究碎片化、可复现性差的问题。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：三层统一框架：数据层（Lance 格式高性能视频加载）、模型层（模块化 baseline + planning solver）、评测层（可控变因的多环境 benchmark 套件）。</li>
  <li><b>技术细节</b>：Lance 格式支持随机访问大规模视频帧，显著降低 I/O 瓶颈；评测套件通过程序化控制外观/几何/物理参数构建系统性分布外测试。</li>
  <li><b>主结果</b>：提供端到端的 world model 研究支持框架；当前论文以平台介绍为主，尚未报告大规模性能对比数字。</li>
</ul>

<h4>Sensor2Sensor：将野外单目 dashcam 视频转换为自动驾驶多模态传感器数据</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22809">arxiv.org/abs/2605.22809</a></li>
  <li><b>信号源</b>：<b>GoogleDeepMind Songyou Peng</b> · Jiahao Wang（一作）</li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>自动驾驶数据采集成本极高，互联网 dashcam 视频海量但格式不匹配（单目、非标准视角、无 LiDAR）。Sensor2Sensor 用 4DGS（4D Gaussian Splatting，四维高斯溅射）重建真实 AV logs 并渲染 dashcam 风格视频构建配对训练数据，再用 diffusion 模型学习跨传感器生成映射——凭借的是 4DGS 动态场景建模的高保真渲染能力，以及 diffusion 模型强大的条件生成能力，两者结合才能解决无配对数据的核心难题。</p>
  <p>对自动驾驶数据赛道：若 Sensor2Sensor 的生成质量能通过下游策略训练的实际效果验证（而非只靠感知指标），将为 AV 开发解锁大规模外部 dashcam 数据来源，显著降低专用传感器数据采集的成本依赖——这直接影响以"专有高质量传感器数据"为护城河的 AV 数据公司的估值逻辑。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 Sensor2Sensor 生成数据用于 AV 策略训练的下游效果验证，判断其是否能替代部分真实传感器数据采集。</li>
    <li>评估以专有传感器数据为护城河的 AV 数据公司，其竞争壁垒是否因此类生成方法的成熟而减弱。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Sensor2Sensor 分两阶段：首先用 4D Gaussian Splatting（4DGS）对真实 AV logs 进行动态场景重建，通过新视角渲染生成对应的 dashcam 风格视频，构建配对训练数据；随后训练 diffusion 模型学习从 dashcam 单目视频到多模态传感器套件（多视角 RGB + LiDAR 点云）的生成映射。在保真度和真实性的量化评估中表现良好，成功将野外 dashcam 视频转换为符合 AV 训练格式的多模态数据。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：4DGS 重建阶段构建配对训练数据（dashcam 风格视频 ↔ AV 传感器套件）；diffusion 模型学习条件生成映射，联合生成多视角图像和 LiDAR 点云。</li>
  <li><b>技术细节</b>：4DGS 将 AV log 中的动态场景建模为时序高斯基元，支持任意视角和时刻渲染；diffusion 模型以 dashcam 帧为条件输入，联合生成空间对齐的多模态传感器数据。</li>
  <li><b>主结果</b>：在保真度和真实性量化评估中表现良好；成功将互联网和 dashcam 野外视频转换为 AV 训练格式的多模态数据。</li>
  <li><b>关键结果点</b>：为 AV 开发解锁大规模外部数据来源，降低专用传感器数据采集的成本依赖。</li>
</ul>

<h4>TacO：首个系统性触觉传感器基准，四种模态覆盖三类操控任务</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21976">arxiv.org/abs/2605.21976</a></li>
  <li><b>信号源</b>：<b>Xiaolong Wang</b>（UC San Diego）· Anya Zorin（一作）</li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>机器人触觉传感器领域长期缺乏跨模态的系统性对比——"触觉感知对操控有益"这一直觉从未被系统验证。TacO 通过训练独立策略控制变量，发现触觉信号的价值高度依赖于传感器模态、材料摩擦属性和任务类型，而非普遍有益。这对机器人硬件传感器选型是直接的可操作洞察：没有"最好的触觉传感器"，只有"最适合特定任务的触觉传感器"。</p>
  <p>对机器人触觉传感器赛道：TacO 的开源（传感器、代码、数据、硬件方案）将降低该方向的研究门槛，可能加速触觉感知在操控任务中的商业落地；但也意味着任何以"我们有更好的触觉传感器"为单一卖点的公司需要更精细地阐明其优势所在的具体任务场景。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>将 TacO 框架用于评估机器人触觉传感器赛道标的，要求其明确最适合的操控任务类型和材料场景，而非泛称"触觉感知能力"。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>TacO 为 visual/acoustic/magnetic/resistive 四种触觉传感器模态分别训练操控策略（manipulation policy），在 pick-and-place、reorientation、plug insertion 三类代表性 contact-rich 任务上进行对照评估，系统分析空间分辨率、shear sensing、触觉表征方式与材料摩擦系数对任务成功率的影响。研究发现触觉传感器的有效性强烈依赖于传感器模态、材料属性及具体任务，并非普遍有益；所有传感器、代码、数据和硬件方案将开源。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：为每种触觉传感器模态独立训练操控策略（learning from demonstrations），固定任务难度和机器人平台，仅变换传感器，以控制变量方式测量性能差异。</li>
  <li><b>技术细节</b>：四种模态（visual-based 如 GelSight 类、acoustic 如 STS、magnetic 如 uSkin、resistive 如压阻阵列）各自独立编码后输入策略网络；benchmark 覆盖三类 contact-rich 操控任务。</li>
  <li><b>主结果</b>：不同触觉传感器模态的适用场景存在显著差异，触觉信号并非普遍有益。</li>
  <li><b>关键结果点</b>：插拔类精细接触任务与抓取类任务对传感器属性的需求截然不同；所有数据和硬件方案开源。</li>
</ul>

<h4>PlexRL：集群级 LLM 服务复用框架，强化学习训练 GPU 成本降低 37.58%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20863">arxiv.org/abs/2605.20863</a></li>
  <li><b>信号源</b>：<b>Ziming Liu</b> · Peking Yiqi Zhang（一作）</li>
</ul>
<callout emoji="⚙️">
  <p><b>Our Insights</b></p>
  <p>RLVR（基于可验证奖励的强化学习）训练中 GPU 空闲是结构性问题——长尾 rollout 和工具调用阻塞在单任务内无法消除。PlexRL 发现不同 RLVR 任务的空闲期呈反相关分布，通过集群级时间片调度让多任务共享 LLM 服务实例，凭借 affinity 约束避免模型权重搬运，从而在不改变算法行为的前提下填补空闲——这是一个纯系统层面的优化，无需修改任何训练算法。</p>
  <p><b>37.58%</b> 的 GPU 成本降低对于训练成本动辄数百万美元的 RLVR 训练场景意义显著。对 LLM 训练基础设施赛道：这类集群调度优化的壁垒在于需要对大量训练任务有全局视野，更适合云厂商或大型 AI 实验室内部使用，而非作为独立商业化产品。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 PlexRL 是否被主流 RLVR 训练框架（如 veRL / OpenRLHF）集成，判断其从学术工具到工程标准的转化路径。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PlexRL 构建集群级统一 LLM 服务运行时，将各 RLVR 任务的 rollout inference 抽象为 serviceized 函数调用，由集群级调度器统一排队分发。调度器感知各任务的 rollout 空闲窗口，将其他任务请求填入；通过 affinity 约束（affinity constraints）保证模型状态一致性，全程无需重新加载或迁移模型权重。在集群有效容量和 GPU 小时成本上显著提升，最高降低 37.58%。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：集群级统一 LLM 服务运行时，集中管理模型 placement、状态转换和函数级调度；时间片复用多个 RLVR 任务的 LLM 执行，无需模型迁移。</li>
  <li><b>技术细节</b>：调度在函数粒度（function-level）执行；affinity 约束保证模型状态一致性，类似 CPU 时间片切换但作用于 GPU 上的 LLM 推理服务。</li>
  <li><b>主结果</b>：GPU 小时成本最高降低 37.58%。</li>
  <li><b>关键结果点</b>：对单任务的额外开销极小；算法行为不受影响；保持算法灵活性。</li>
</ul>

<h4>Cornell 量子增强 RL 框架用于化工过程综合，qubit 需求与问题规模解耦</h4>

  <li><b>信号源</b>：Cornell University</li>

<callout emoji="🧮">
  <p><b>Our Insights</b></p>
  <p>将化工流程设计（process synthesis）形式化为 MDP（马尔可夫决策过程）并用量子 RL 求解，核心贡献是 state encoding 打破了 qubit 数量随问题规模线性增长的扩展性瓶颈。在中等规模下，量子方法 per-parameter 效率优于经典 RL，但整体性能与经典方法持平——尚未在大规模问题上展示决定性优势。</p>
  <p>当前结论应被解读为"验证性探索"而非"实用化突破"。对量子 AI 赛道：量子 RL 在化工优化场景的落地还有相当距离，但 state encoding 的方法论本身具有迁移价值，可应用于其他组合优化类量子算法研究。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>持续追踪量子 RL 在化工/材料设计领域的进展，当前阶段作为趋势了解而非投资标的，需等待大规模优势的实际验证。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Cornell 将化工流程设计（process synthesis）正式建模为 MDP，开发量子增强 RL 算法，并以经典 RL 为基准进行对比评测。核心创新是 state encoding 算法——通过对 flowsheet 状态进行紧凑量子编码，使所需 qubit 数不再随设计空间大小线性扩展。在小规模问题上，量子与经典 RL 均能找到最优方案；在中等规模下，量子方法 per-parameter 效率优于经典 RL，但整体仍属验证性探索阶段。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：化工 process synthesis → MDP 形式化建模；量子增强 RL 算法 vs. 经典 RL 基准的受控对比评测，测试不同 unit count 规模的 flowsheet synthesis 问题。</li>
  <li><b>技术细节</b>：state encoding 将 flowsheet 状态映射到量子线路编码，使 qubit 需求与问题规模解耦；类似将稀疏矩阵压缩存储后再送入量子处理器。</li>
  <li><b>主结果</b>：小规模问题：量子与经典 RL 均能找到最优 flowsheet 设计；中等规模：量子方法 per-parameter 效率更优。</li>
  <li><b>关键结果点</b>：尚未在大规模问题上展示决定性优势；属于验证性探索阶段。</li>
</ul>

<h4>SciCore-Mol：三个可插拔认知模块增强 LLM 分子科学能力</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.22287">arxiv.org/abs/2605.22287</a></li>

<callout emoji="🧬">
  <p><b>Our Insights</b></p>
  <p>将 LLM 应用于分子科学的核心障碍是离散语言符号无法直接表达拓扑结构和连续化学空间。SciCore-Mol 为 LLM 配备三个专用化学插件（拓扑感知模块、latent diffusion 分子生成模块、反应推理模块），通过学习表示接口与主干 LLM 深度耦合，避免了传统 SMILES 字符串中转导致的信息损失。</p>
  <p><b>8B 参数开源系统</b>在部分子任务上超越专有大模型，说明模块化插拔设计在科学 AI 中的有效性已得到实质验证。对 AI for Drug Discovery / Materials Science 赛道：SciCore-Mol 的可插拔架构为构建垂直领域科学 AI 系统提供了一个可复用的设计范式；8B 规模可控的参数量使其具备实际部署可行性。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 SciCore-Mol 是否开源，若开源则评估其在药物发现/材料设计创业标的中的采用潜力。</li>
    <li>约谈专注分子生成/反应预测方向的 AI for Drug Discovery 团队，了解其对模块化 LLM 架构的技术路线判断。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>SciCore-Mol 由一个 LLM 主干和三个深度集成的可插拔模块构成：① 拓扑感知感知模块（处理分子图结构信息）；② 基于 latent diffusion 的分子生成模块（支持连续化学空间生成）；③ 反应感知推理模块（针对化学反应预测）。各模块通过学习表示接口与 LLM 交换信息，绕开纯文本中转带来的信息损失。8B 参数开源系统在分子理解、生成、反应预测和通用化学知识多项任务上综合表现具有竞争力，部分维度超越专有大模型。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：LLM 主干 + 三个可插拔认知模块；各模块通过学习表示接口（representation interfaces）与 LLM 交换高维嵌入，直接传递分子结构信息而非 SMILES 字符串。</li>
  <li><b>技术细节</b>：相比 SMILES 字符串中转，每个模块直接产出高维嵌入向量经接口传给 LLM，保留拓扑细节；latent diffusion 模块支持连续化学空间的分子生成。</li>
  <li><b>主结果</b>：8B 规模综合性能与闭源大模型相当，部分子任务超越。</li>
  <li><b>关键结果点</b>：模块化插拔设计有效；开源 8B 系统可控参数量具备部署可行性。</li>
</ul>
