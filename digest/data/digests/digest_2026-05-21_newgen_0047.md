<h3>AI 审稿人综合质量超越顶尖人工审稿人，但"撞车率"是人类 7 倍</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20668">Large-Scale Expert Annotation Study of AI vs. Human Peer Review</a></li>
  <li><b>信号源</b>：<b>Ohio State Xiang Yue</b></li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>过去两年，关于"AI 能否替代同行评审"的讨论几乎都停留在逸事层面——这项研究是迄今最系统的实证回答。结论是双面的：<b>GPT-5.2</b> 综合评分确实超越了每篇论文最高分的人工审稿人（<b>60.0% vs. 48.2%</b>，p=0.009），且能发现 <b>26%</b> 的人类未提出的独特问题。这对"AI 辅助审稿"类产品是直接的市场信号。</p>
  <p>但"撞车率"问题揭示了一个根本局限：三个 AI 审稿人之间重叠率高达 <b>21%</b>，而人类只有 <b>3%</b>。这意味着用多个 AI 模型做审稿不能像多位人类专家那样实现真正的覆盖面叠加——本质上是模型间的同质化，而非独立判断的集合。更重要的是，<b>16 类系统性弱点</b>（子领域知识不足、长上下文管理缺陷等）说明 AI 无法自我识别自己的盲区。</p>
  <p>投研含义：当前最合理的产品形态是"AI 作为初筛 + 人类深度审查"的混合审稿，而非纯 AI 替代。对于投在 AI for Science 或学术工具赛道的标的，需要关注其产品设计是否正确处理了这个覆盖面悖论——单纯堆 AI 模型数量无法解决集体同质化问题，<b>差异化的模型训练路径或专业分工机制</b>才是护城河。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>约谈 AI 辅助同行评审赛道创业团队（如 Wisdomdao、Consensus 等），重点问"如何解决 AI 审稿同质化问题"——若答案是"换更强的模型"则护城河不足。</li>
    <li>关注 Xiang Yue 团队后续是否发布 ArxivReviewBench 等配套评测集，若开源可作为同赛道标的能力对比基准。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>本研究招募 45 位物理、生物和健康科学领域专家，耗时共 469 小时，对 82 篇 Nature 系列论文的人工与 AI 生成评审进行大规模标注。标注对象为 2,960 条具体批评意见，评估维度包括正确性、重要性和证据充分性，被评估的 AI 审稿人涵盖 GPT-5.2、Gemini 3.0 Pro 和 Claude Opus 4.5。研究发现 AI 审稿人综合质量超越顶尖人工审稿人，但存在显著的同质化问题与 16 类系统性弱点。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：招募 45 位领域专家对 82 篇 Nature 论文的人工与 AI 评审进行三维度（正确性、重要性、证据充分性）标注，覆盖 2,960 条批评意见；被评估 AI 包括 GPT-5.2、Gemini 3.0 Pro、Claude Opus 4.5。</li>
  <li><b>技术细节</b>：每位专家对分配到的批评意见逐条打分并对所有审稿人相对排名；跨审稿人重叠率通过计算不同审稿人提出相同批评的比例衡量，类似检索系统"查全率"与"去重后查全率"之差。</li>
  <li><b>主结果</b>：GPT-5.2 驱动的审稿 agent 综合评分超过每篇论文最高分人工审稿人（<b>60.0% vs. 48.2%</b>，p=0.009）。</li>
  <li><b>关键结果点</b>：AI 审稿人能发现 26% 的人类未提出的独特问题，但三个 AI 审稿人之间重叠率高达 <b>21%</b>（人类 <b>3%</b>）；AI 存在 16 类系统性弱点，包括子领域知识不足、多文件长上下文管理缺陷、对次要问题过度批评等。</li>
</ul>

<h3>IXT：把后训练的"质量判断"注入预训练，计算效率最高提升 2.8x</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20285">Introspective Training (IXT): Injecting Post-Training Feedback into Pre-Training</a></li>
  <li><b>信号源</b>：<b>Allen AI Yejin Choi</b></li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>几乎所有大模型训练流程都假设 pre-training 和 post-training 是两个独立阶段——前者负责"喂数据"，后者负责"校正行为"。IXT 打破这个假设：凭借在预训练语料上叠加 thinking reward model 生成的自然语言批注作为前缀（prefix-conditioning），把质量信号从事后校正变为全程内嵌。</p>
  <p>这件事改变了什么判断：过去认为预训练数据质量提升只能靠"数据筛选（过滤低质语料）"，IXT 给出了另一条路——不筛掉数据，而是给数据贴条件标签让模型自行区分质量。在 <b>18T tokens</b> 规模上验证了 <b>2.8x</b> 计算效率提升，且 math/code 等专项领域超出普通训练可达上限，说明这条路径在大规模训练上是真实有效的。</p>
  <p>对投研的含义：这类方法若被主流训练基础设施（如 <b>Together AI</b>、<b>Coreweave</b> 等训练即服务平台）采纳，将直接压缩同等算力下的训练成本——对依赖"烧算力换性能"叙事的训练基础设施标的是潜在的<b>估值前提修正</b>信号。另外，thinking reward model 本身作为训练数据标注组件是一个值得关注的新品类。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>跟进 Allen AI 是否开源 IXT 工具链；若开源，评估 Together AI / Coreweave 等训练平台是否会集成，进而影响其差异化卖点。</li>
    <li>关注"thinking reward model 即服务"是否出现独立创业标的，该组件可能成为下一代数据标注基础设施的核心模块。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>IXT（Introspective Training）受 offline reward-conditioned RL（基于奖励条件的强化学习）启发，引入一个 thinking reward model 对所有训练数据自动生成自然语言 critique 形式的反馈标注，并以这些 feedback 作为 prefix 拼接到原始数据前进行条件化语言模型训练。方法在 7.5-12B 规模 dense LLM 上从头训练至 18T tokens，最高实现 2.8x 计算效率提升，并在 math、code 等领域超出普通训练可达性能上限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：IXT 受 offline reward-conditioned RL 启发，thinking reward model 对全部训练语料（含 pre-training 数据）生成自然语言 critique，以 feedback 作为 prefix 拼接到原始数据前，进行条件化语言模型训练，使模型从 pre-training 阶段就感知数据质量差异。</li>
  <li><b>技术细节</b>：thinking reward model 对每条样本生成批评段落，prefix 拼接后训练时模型同时建模 feedback 和原始内容；类似于给每篇文章贴上"高质量/低质量+原因"标签，让 next-token prediction 从一开始就是有条件的，而非等权对待所有 token。</li>
  <li><b>主结果</b>：在 7.5-12B dense LLM 上从头训练至 18T tokens，IXT 将 scaling curve 向下弯折，整体计算效率最高提升 <b>2.8x</b>。</li>
  <li><b>关键结果点</b>：在 math 和 code 等专项领域，IXT 训练的模型达到了普通训练方式无法企及的性能水平；方法在 pre-training 全程持续有效，非仅在特定阶段发挥作用。</li>
</ul>

<callout emoji="⚙️">
  <p><b>Our Insights</b></p>

    <li>梳理 kernel 自动优化赛道（如 Modular、Baseten、KernelBench 等）是否已将 agentic LLM 纳入竞争对手分析，更新投研判断。</li>
  </ul>
</callout>

<p>Alibaba Cloud 在 Qwen Conference 2026 期间展示 Qwen3.7-Max 自主优化 Extend Attention Kernel 的案例：模型在无人工干预下持续运行 35 小时，执行 432 次 kernel 评估、1158 次工具调用，最终相比 Triton 参考实现取得 10.0x 几何平均加速，体现了 Qwen3.7-Max 在 agentic 长时程工程任务上的实际能力。</p>

<h3>OpenAI 推理模型推翻 Erdős 猜想：通用模型跨越几何与代数数论领域鸿沟</h3>
<ul>
  <li><b>信号源</b>：<b>Rohan Paul</b>（独立评论，<a href="https://x.com/rohanpaul_ai/status/2057354887185481889">原推文</a>）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p><b>OpenAI</b> 通用推理模型（非专用定理证明器）在平面单位距离问题上找到多项式量级改进的无限构造族，推翻了 Erdős 1946 年猜想，证明已由外部数学家独立验证。凭借的是 test-time compute（推理时算力）的大幅增加，而非专项数学训练。</p>
  <p>这件事让"AI 数学能力的瓶颈在哪里"这个问题有了新答案：不是模型容量，而是<b>给多少时间思考</b>。Erdős 猜想横跨初等几何与 Golod–Shafarevich 代数数论，正是这类需要跨领域知识整合的问题，过去被认为远超 LLM 能力边界。若此突破可复现，将使"通用推理模型用于科研前沿"的时间节点明显提前。</p>
  <p>注意：此信号标注为 needs_human_review，建议在正式引用前核实原始论文/技术报告。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 OpenAI 是否发布对应技术报告或论文；若有，重点关注 test-time compute 规模与推理策略细节。</li>
    <li>重新评估 AI for Mathematics 赛道（Lean 形式化证明、数学定理发现工具）的竞争格局——通用推理模型的突破是否使专用数学 AI 标的的差异化前提弱化。</li>
  </ul>
</callout>
<h5>信号内容</h5>
<p>据 Rohan Paul 评论，OpenAI 通用推理模型在平面单位距离问题（Erdős 1946 年猜想）上取得突破：找到了多项式量级改进的无限构造族（infinite class field towers、Golod–Shafarevich theory），而非仅针对特定有限情形的改进，证明已由外部数学家验证为有效。此成果被 OpenAI 联合创始人 Greg Brockman 定性为"AI 生成新知识的里程碑"。</p>

<h3>Google DeepMind Antigravity Science Skills：30+ 生命科学数据源接入 AI 研究工作流</h3>
<ul>
  <li><b>信号源</b>：<b>Google DeepMind</b> 官方（<a href="https://x.com/GoogleDeepMind/status/2057256257153884161">原推文</a>）</li>
</ul>
<callout emoji="🧬">
  <p><b>Our Insights</b></p>
  <p><b>Google DeepMind</b> 将 UniProt、AlphaFold Database 等 30+ 个主流生命科学数据库整合进 Antigravity，凭借的是 DeepMind 自身对这些数据源的深度运营关系（AlphaFold DB 本身就是 DeepMind 资产）——这构成了一条竞争对手难以快速复制的数据接入护城河。</p>
  <p>对 ai4s 赛道的影响：Antigravity 若将 Science Skills 做深，将直接挤压独立生命科学 AI 工具（如 Benchling AI、Geneious 等）的市场空间。核心问题是 Google 是否会将 Antigravity 定位为平台型工具（开放 API）还是封闭产品——前者对第三方开发者是机会，后者是竞争威胁。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>调研 Antigravity 的定价与 API 开放策略，判断其对生命科学 AI SaaS 创业（如 Benchling、Labgenie 等）的实际威胁程度。</li>
    <li>关注 UniProt / PDB 等数据源是否与 Google 签署独家或优先接入协议——若有，对其他平台是结构性障碍。</li>
  </ul>
</callout>
<h5>信号内容</h5>
<p>Google DeepMind 宣布为旗下 AI 研究工具 Antigravity 推出 Science Skills 功能，整合 UniProt、AlphaFold Database 在内的 30 余个生命科学数据源，支持研究者通过自然语言交互完成蛋白质查询、结构检索等日常科研任务，旨在加速从假设提出到数据分析的全流程科研工作流。</p>

<h4>IXT：把后训练的"质量判断"注入预训练，计算效率最高提升 2.8x</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20285">Introspective Training (IXT): Injecting Post-Training Feedback into Pre-Training</a></li>
  <li><b>信号源</b>：<b>Allen AI Yejin Choi</b></li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>几乎所有大模型训练流程都假设 pre-training 和 post-training 是两个独立阶段——前者负责"喂数据"，后者负责"校正行为"。IXT 打破这个假设：凭借在预训练语料上叠加 thinking reward model 生成的自然语言批注作为前缀（prefix-conditioning），把质量信号从事后校正变为全程内嵌。</p>
  <p>这件事改变了什么判断：过去认为预训练数据质量提升只能靠"数据筛选"，IXT 给出了另一条路——不筛掉数据，而是给数据贴条件标签让模型自行区分质量。在 <b>18T tokens</b> 规模上验证了 <b>2.8x</b> 计算效率提升，且 math/code 等专项领域超出普通训练可达上限，说明这条路径在大规模训练上是真实有效的。</p>
  <p>对投研的含义：这类方法若被主流训练基础设施（如 <b>Together AI</b>、<b>Coreweave</b> 等训练即服务平台）采纳，将直接压缩同等算力下的训练成本——对依赖"烧算力换性能"叙事的训练基础设施标的是潜在的<b>估值前提修正</b>信号。另外，thinking reward model 本身作为训练数据标注组件是一个值得关注的新品类。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>跟进 Allen AI 是否开源 IXT 工具链；若开源，评估 Together AI / Coreweave 等训练平台是否会集成，进而影响其差异化卖点。</li>
    <li>关注"thinking reward model 即服务"是否出现独立创业标的，该组件可能成为下一代数据标注基础设施的核心模块。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>IXT（Introspective Training）受 offline reward-conditioned RL 启发，引入一个 thinking reward model 对所有训练数据自动生成自然语言 critique 形式的反馈标注，并以这些 feedback 作为 prefix 拼接到原始数据前进行条件化语言模型训练。方法在 7.5-12B 规模 dense LLM 上从头训练至 18T tokens，最高实现 2.8x 计算效率提升，并在 math、code 等领域超出普通训练可达性能上限。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：IXT 受 offline reward-conditioned RL 启发，thinking reward model 对全部训练语料生成自然语言 critique，以 feedback 作为 prefix 拼接到原始数据前，进行条件化语言模型训练，使模型从 pre-training 阶段就感知数据质量差异。</li>
  <li><b>技术细节</b>：thinking reward model 对每条样本生成批评段落，prefix 拼接后训练时模型同时建模 feedback 和原始内容；让 next-token prediction 从一开始就是有条件的，而非等权对待所有 token。</li>
  <li><b>主结果</b>：在 7.5-12B dense LLM 上从头训练至 18T tokens，IXT 将 scaling curve 向下弯折，整体计算效率最高提升 <b>2.8x</b>。</li>
  <li><b>关键结果点</b>：在 math 和 code 等专项领域，IXT 训练的模型达到了普通训练方式无法企及的性能水平；方法在 pre-training 全程持续有效。</li>
</ul>

<h4>AIR：用"不对称正则化"让 LLM 安全行为不随 prompt 表述改变</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20994">Anchor Invariance Regularization for Context-Invariant Safety Alignment</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b>（arXiv）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>LLM 安全对齐的一个核心痛点：同样语义的请求，用对抗性表述（改写、绕话、提示注入）就能绕过安全约束。AIR（Anchor Invariance Regularization）通过"不对称正则化"解决这个问题——以可验证的 anchor prompt 为固定基准，单向驱动开放式 prompt 向 anchor 靠拢，而不是让两端互相妥协。</p>
  <p>这件事的重要性在于：分布外（OOD）一致性提升 <b>33.49%</b> 是一个实质性的数字，意味着对于生产环境中"无法预料的用户表述变体"有实际鲁棒性提升。对于做企业级 LLM 安全合规的标的（红队测试、安全护栏 API），这类方法既是潜在技术武器，也是竞争压力——若大模型厂商原生集成 AIR 类机制，第三方安全护栏产品的附加价值将下降。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>与已投或在看的 LLM 安全合规创业（如 Lakera、Guardrails AI 等）讨论：若主流模型原生支持 AIR 类 context-invariant alignment（上下文不变对齐），其产品护城河是否仍然成立。</li>
    <li>关注该方法与 GRPO 等 group-based preference optimization 的组合实验结果，判断其能否成为标准训练流程的通用组件。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>AIR（Anchor Invariance Regularization）将训练 prompt 分为可验证的 anchor prompt（如多选题）和依赖噪声 reward model 的开放式 prompt，通过 stop-gradient 机制以 anchor 端输出为目标单向正则化开放式端行为。方法以辅助损失形式与 GRPO 联合训练，在 Safety、Moral Reasoning、Math 三类任务上，组内分布准确率提升 12.71%，分布外一致性提升 33.49%。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：AIR 将 prompt 分为 anchor（有可验证反馈，如多选题）和开放式两类；stop-gradient 截断 anchor 端梯度，仅驱动开放式端向 anchor 靠拢；以辅助 loss 形式配合 heterogeneous prompt grouping 与 GRPO 联合训练。</li>
  <li><b>技术细节</b>：不对称正则化是核心——只有一方作为基准（anchor 端固定），校准方向明确可靠，避免对称正则化导致两端互相拉扯而退化的问题。</li>
  <li><b>主结果</b>：在安全、道德推理和数学任务上，组内分布准确率提升 <b>12.71%</b>，分布外一致性提升 <b>33.49%</b>。</li>
  <li><b>关键结果点</b>：AIR 作为即插即用辅助 loss，与 GRPO 等 group-based preference optimization 方法兼容，无需修改基础训练架构。</li>
</ul>

<h4>AGPO：自适应控制 RL 训练的 clip 幅度与采样温度，无需额外 critic 网络</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20722">AGPO: Adaptive Group Policy Optimization for LLM Reasoning</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b>（arXiv）</li>
</ul>
<callout emoji="🎯">
  <p><b>Our Insights</b></p>
  <p>GRPO（一种主流的基于组的偏好优化，Group Relative Policy Optimization）训练中，超参数（clip 边界、采样温度）固定不变是个已知缺陷——同一套参数在训练早期和晚期表现迥异。AGPO 的思路是让训练过程"自我感知"：用 reward 分布的离散度、偏斜度、熵等组级别统计量动态调节 trust region 和温度，不引入额外 critic 网络，计算开销接近原始 GRPO。</p>
  <p>关键数字：在 9 个数学/STEM 基准上，Qwen2.5-14B 的 GSM8K 达 <b>67.3%</b>、MATH 达 <b>40.5%</b>，超越 PPO 和 GRPO 基线，且跨 Llama-3-8B、Gemma-2-9B 均可复现。这是一个方法改进工作，主要含义是：后训练（post-training）优化仍有较大工程空间，专注于训练效率的基础设施标的（如 <b>Anyscale Ray</b>、<b>Verl</b> 等训练框架）若集成此类自适应机制将提升产品竞争力。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>跟踪 AGPO 是否被 open-source 后训练框架（如 OpenRLHF、veRL）采纳；若有，用于评估当前在看的后训练基础设施标的的技术领先性。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>AGPO 在 GRPO 基础上增加两个自适应控制器：adaptive clipping（利用 reward 离散度、偏斜度、vote entropy、policy entropy 和 KL drift 动态设置 trust region 大小）和 bidirectional adaptive temperature sampling（以 running baseline 为参考动态调节 decoding temperature），共享同一组 probe 统计状态，不引入额外 critic 网络。在 9 个数学/STEM 基准上持续优于 PPO 和 GRPO，跨三个模型系列可复现。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：AGPO 在 GRPO 基础上增加 adaptive clipping 和 bidirectional adaptive temperature sampling 两个控制器，共享 probe 统计状态，无需独立 value head。</li>
  <li><b>技术细节</b>：每次更新前用少量 probe rollout 估计 group 级别统计量（reward 均值、方差、偏斜等），统计量既决定本步 PPO clip 边界，也决定下一轮采样温度；计算开销接近原始 GRPO。</li>
  <li><b>主结果</b>：Qwen2.5-14B 在 GSM8K 达 <b>67.3%</b>、MATH 达 <b>40.5%</b>，在相同 token 预算下超越 PPO/GRPO 基线。</li>
  <li><b>关键结果点</b>：性能提升可迁移至 Llama-3-8B 和 Gemma-2-9B，跨 9 个英文/中文数学与 STEM 基准均有收益；ablation 验证 adaptive clipping 与 bidirectional adaptive temperature 各自贡献互补。</li>
</ul>

<h4>PF+BV：在完全形式化证明与自然语言之间架桥，模块化验证数学证明</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20531">Pseudo-Formalization and Block Verification for Mathematical Proof Verification</a></li>
  <li><b>信号源</b>：<b>Stanford Tengyu Ma</b></li>
</ul>
<callout emoji="🧠">
  <p><b>Our Insights</b></p>
  <p>数学证明验证领域存在两个极端：完全形式化（Lean/Coq）精确但门槛极高；LLM-as-judge（以 LLM 作为评判者）灵活但不可靠。PF+BV 在中间地带找到了一个工程可行的方案——把自然语言证明重构为带明确前提/结论的自包含模块，再逐模块独立验证。</p>
  <p>在 precision 和 recall 双指标上 Pareto 优于 LLM-as-judge 基线，同时配套发布 ArxivMathGradingBench。这件事对 AI for Mathematics 赛道的含义：如果证明验证能以自然语言为主要接口（而非 Lean），将大幅降低数学 AI 工具的普及门槛，对面向研究者的数学助手类产品（如 Morph、Wolfram Alpha Next 等）是直接的产品设计参考。<b>Lean/形式化证明工具</b>的替代压力将进一步上升。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>追踪 ArxivMathGradingBench 的社区采用情况，若成为标准基准，可用于比较不同数学 AI 工具的验证能力。</li>
    <li>评估 PF+BV 能否直接集成到 AI 写作辅助或科学论文审阅产品中，作为数学段落验证模块。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PF+BV（Pseudo-Formalization + Block Verification）分两步：首先由 LLM 将非结构化自然语言证明翻译为 Pseudo-Formal 格式（每个模块含 premises、conclusion、natural language proof），然后对每个模块独立调用 LLM 进行正确性验证，模块间依赖关系通过前提声明显式传递，避免上下文污染。在涵盖奥林匹克和研究级数学的两个 benchmark 上，precision 和 recall 双指标 Pareto 优于 LLM-as-judge 基线，并配套发布 ArxivMathGradingBench 评测集。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：PF+BV 分翻译和验证两步：LLM 将自然语言证明切割为含前提/结论/证明三部分的自包含模块（Pseudo-Formal 格式），再对每模块独立验证；模块间依赖通过前提声明显式传递。</li>
  <li><b>技术细节</b>：翻译步骤要求 LLM 识别证明逻辑边界，每块明确声明所依赖的上游前提，使验证时每块可独立送入 LLM 而无需完整上下文——类似编译器将程序分解为函数，每个函数有明确输入输出签名，可单独类型检查。</li>
  <li><b>主结果</b>：在竞赛和研究级数学两个 benchmark 上，PF+BV 在错误发现的 precision 和 recall 双指标上均 Pareto 优于 LLM-as-judge 基线。</li>
  <li><b>关键结果点</b>：配套发布 ArxivMathGradingBench，覆盖奥林匹克及前沿数学题目，支持后续研究标准化对比。</li>
</ul>

<h3>多模态智能 (Multimodal Intelligence)</h3>

<h4>Deformba：给视觉状态空间模型装上"可移动眼球"，突破固定扫描限制</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21308">Deformba: Deformable State Space Models for Visual Understanding</a></li>
  <li><b>信号源</b>：<b>Jack Morris</b></li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>视觉 SSM（状态空间模型，如 Mamba）过去两年在图像任务上被广泛尝试，但固定扫描顺序（行优先/列优先）导致其在需要灵活空间感知的任务（如 3D BEV 感知）上存在结构性短板。Deformba 通过借鉴可变形卷积（Deformable Convolution）思路，在 SSM 扫描路径上引入可学习偏移量，同时设计适配 SSM 的跨模态 fusion 模块。</p>
  <p>这件事对多模态视觉赛道的含义：若 SSM 的线性复杂度优势能与 Transformer 的空间灵活性相结合，将使 SSM 在边缘部署和长序列视觉任务上更具竞争力。对于依赖 Transformer-based 视觉主干的自动驾驶感知创业（BEV perception 是核心场景），Deformba 是一个需要纳入技术路径对比的候选方案。但当前仅在标准 benchmark 上验证，尚缺生产环境数据。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 Deformba 在 nuScenes/Waymo 等自动驾驶 benchmark 上的后续结果；若超过主流 Transformer BEV 方案，对 BEV 感知创业标的是强信号。</li>
    <li>评估边缘部署场景（如 NVIDIA Orin 等车载芯片）下 SSM vs. Transformer 的实际推理效率差异，这是线性复杂度优势能否转化为商业价值的关键。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>Deformba 以 SSM（状态空间模型）为主干，在序列扫描阶段引入可变形的上下文自适应采样机制，动态调整图像 patch 的空间关系建模；同时设计了适配 SSM 特性的跨模态 fusion 模块，使其可以接受外部 query 流（如多视角特征）进行交互。在 2D 图像分类、目标检测、分割及 3D BEV 感知多个 benchmark 上验证了通用性和有效性，同时维持线性时间复杂度。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：Deformba 在 SSM 扫描路径上引入可学习偏移量（借鉴可变形卷积思路），实现上下文自适应的空间关系建模；同时设计跨模态 fusion 模块，将外部信息流作为 key/value 注入 SSM 隐状态更新过程。</li>

<h4>SAVER：社交媒体多图信息抽取的"按需看图"框架</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20713">SAVER: Selective Visual Evidence Retrieval for Multimodal Information Extraction</a></li>
  <li><b>信号源</b>：<b>Xin Wang</b></li>
</ul>
<callout emoji="🔬">
  <p><b>Our Insights</b></p>
  <p>多模态信息抽取（MNER、MRE 等任务）的通常做法是"always-on"——无论是否需要，都融合视觉信息。SAVER 用统计置信门控（Conformal Groundability Gate，CGG）判断每个候选实体是否真的需要视觉支持，再用 submodular 选择器从多图中挑最小可信子集。</p>
  <p>对投研的含义偏向技术路径层面：这类"按需多模态融合"机制若成为业界标准，将使多模态推理（inference）的计算成本大幅下降，对面向高吞吐社交媒体分析的 AI 基础设施标的是正向信号。同时，CGG 的统计校准思路（用 Clopper–Pearson 置信上界控制激活率）是一种可迁移到其他"按需调用"场景的通用设计模式。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>评估 SAVER 的 CGG 门控机制能否迁移到 RAG（检索增强生成）系统中，作为"是否需要检索"的置信判断组件。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>SAVER 针对社交媒体多图多模态命名实体识别（MNER）和关系抽取（MRE）场景，通过 Conformal Groundability Gate（CGG）在 span/pair 级别判断是否需要激活视觉分支，再由 submodular 选择器从多张图像中筛选兼顾相关性与多样性的最小证据子集，经 Set Transformer 聚合后与文本融合完成分类。在 F1、AURC、FLOPs 和 P90 延迟上均优于纯文本及 always-on 多模态基线。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：SAVER 分三阶段：(1) CGG 门控——估计视觉可接地性，决定是否激活视觉分支；(2) submodular 证据选择——从多图中选出相关性/多样性最优子集；(3) Set Transformer 聚合 + energy-inspired 评分头完成最终分类。</li>
  <li><b>技术细节</b>：CGG 以 Clopper–Pearson 置信上界为锚点在 held-out split 上校准激活阈值，确保在给定风险水平下激活覆盖率可控；submodular 选择器通过边际收益递减保证所选图像集合不重复且覆盖全面。</li>
  <li><b>主结果</b>：SAVER 在 MNER 和 MRE 任务上持续提升 F1，同时降低 AURC、减少 FLOPs 和 P90 推理延迟。</li>
  <li><b>关键结果点</b>：按需视觉融合在准确性与效率上均优于 always-on 策略，CGG 的统计校准机制提供了可控的激活覆盖率保证。</li>
</ul>

<h4>DriveMA：用一步语义决策标签替代自然语言推理链，端到端驾驶刷新 Waymo SOTA</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.21273">DriveMA: Meta-Action Driven Vision-Language-Action Model for Autonomous Driving</a></li>
  <li><b>信号源</b>：<b>Hang Zhao</b></li>
</ul>
<callout emoji="🤖">
  <p><b>Our Insights</b></p>
  <p>驾驶 VLA（视觉-语言-动作模型）的主流接口是自然语言推理链——这在大模型上听起来自然，但在实际训练中面临标注成本高、推理慢、小模型难以驾驭等问题。DriveMA 用一步 meta-action（如"右转"、"跟车"等低熵语义标签）替代冗长推理链，既保留语义决策依据，又可从专家轨迹自动派生，无需人工标注推理链。</p>
  <p><b>2B 模型取得 Waymo End-to-End Driving Challenge RFS 8.060 的 SOTA</b>，这个数字意义在于：参数量极小（2B）却超越更大模型，说明接口设计比模型规模更关键。对自动驾驶 AI 创业的含义：若 meta-action 这类低熵接口成为驾驶 VLA 的标准，依赖高质量自然语言推理标注数据的数据供应商的壁垒将下降，而能从驾驶数据中自动提取语义标签的工具链将升值。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>跟进 DriveMA 是否在 nuScenes 或 NAVSIM 等其他基准上复现优势；若跨 benchmark 稳定，考虑约谈 Waymo / 端到端驾驶赛道创业讨论接口设计选择。</li>
    <li>评估 meta-action 自动派生工具链是否存在独立商业价值（如驾驶数据标注 SaaS），若有则是一个新的细分投研方向。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>DriveMA 以 one-step meta-actions（一步语义决策标签）作为驾驶 VLA 的语言接口，替代冗长的自然语言推理链。Meta-actions 可从专家驾驶轨迹自动派生，作为中间接口将规划意图与具体轨迹解耦。训练侧采用 action-centric 监督学习结合 turn-level credit-assignment（逐轮信用分配）强化学习框架，联合优化 meta-action 正确性、轨迹质量及一致性。在 Waymo End-to-End Driving Challenge 上，2B 和 4B 参数量均刷新 SOTA（RFS 分别为 8.060 和 8.079）。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：DriveMA 以 one-step meta-actions 作为语言接口；训练采用 action-centric 监督学习 + turn-level credit-assignment RL，联合优化三项目标：meta-action 正确性、轨迹质量、轨迹与 meta-action 一致性。</li>
  <li><b>技术细节</b>：Meta-actions 是低熵语义决策标签，可从专家轨迹自动推导，无需人工标注；RL 部分逐 turn 信用分配避免稀疏奖励，使长序列规划获得有效梯度信号。</li>
  <li><b>主结果</b>：DriveMA 2B 在 Waymo End-to-End Driving Challenge 达到 RFS <b>8.060</b>，4B 版进一步提升至 <b>8.079</b>，均为 SOTA。</li>
  <li><b>关键结果点</b>：消融实验表明 meta-actions 在表达力、可预测性与推理效率间的权衡优于自然语言 reasoning 和细粒度动作序列；在 NAVSIM 上亦取得有竞争力的表现。</li>
</ul>

<h4>PlexRL：集群级 LLM 服务时间片复用，最高降低 37.58% GPU 成本</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.20863">PlexRL: Cluster-Level LLM Service Multiplexing for RLVR Training</a></li>
  <li><b>信号源</b>：<b>Ziming Liu</b>（arXiv）</li>
</ul>
<callout emoji="⚙️">
  <p><b>Our Insights</b></p>
  <p>RLVR（基于可验证奖励的强化学习训练，Reinforcement Learning with Verifiable Rewards）训练中的 idle 时间是结构性问题——长尾 rollout、工具调用阻塞等在单任务层面无法消除。PlexRL 发现不同任务的 idle 窗口往往反相关，凭借集群级统一调度让多个 RLVR 任务共享 GPU 资源，无需昂贵的模型权重迁移。</p>
  <p><b>37.58%</b> 的 GPU 成本降低是一个实质性数字。对 AI infra 赛道的含义：这类集群级调度优化若被主流训练框架（如 <b>veRL</b>、<b>OpenRLHF</b>）或云平台原生集成，将直接影响 RLVR 训练的总拥有成本，使得当前按 GPU 小时计费的训练服务商面临定价压力。对于做训练即服务（Training-as-a-Service）的标的，这是需要纳入技术路径评估的信号。</p>
  <p><b>Potential To-dos</b></p>
  <ul>
    <li>关注 PlexRL 是否开源或被 veRL/OpenRLHF 社区采纳；若有，评估对 Coreweave、Lambda Labs 等 GPU 云服务商的成本竞争格局影响。</li>
    <li>与已投的训练基础设施标的讨论：当前调度层是否已有类似 PlexRL 的多任务复用机制，若无，差距有多大。</li>
  </ul>
</callout>
<h5>摘要</h5>
<p>PlexRL 是一个集群级 runtime，将 LLM 推理和训练功能抽象为统一服务（serviceized execution），集中管理模型 placement、状态转换和函数级调度，在严格 affinity 约束下对多个 RLVR 任务进行 LLM 执行时间片复用，无需昂贵的模型迁移。通过集中式调度器追踪各任务实时状态，在检测到某任务进入 idle 期时将其时间片分配给其他有计算需求的任务，最高将 GPU 小时成本降低 37.58%。</p>
<h5>方法与结果</h5>
<ul>
  <li><b>方法框架</b>：PlexRL 是集群级 runtime，将 LLM 推理和训练抽象为统一服务，集中管理 placement、状态转换和函数级调度；在 affinity 约束下对多个 RLVR 任务进行 LLM 执行时间片复用，无需模型权重迁移。</li>
  <li><b>技术细节</b>：集中式调度器追踪各 RLVR 任务实时状态（rollout、训练、等待），检测到 idle 期时将时间片分配给其他任务；调度在函数粒度执行，affinity 约束保证模型状态一致性，避免显存间权重搬运开销。</li>
  <li><b>主结果</b>：PlexRL 最高将 GPU 小时成本降低 <b>37.58%</b>，集群有效容量显著提升。</li>
  <li><b>关键结果点</b>：对单任务的额外开销极小，算法行为不受影响；利用的是不同 RLVR 任务 idle 窗口反相关这一结构性规律，在单任务层面无法等效实现。</li>
</ul>

  <p><b>Potential To-dos</b></p>

  </ul>
</callout>
<h5>摘要</h5>
<p>Cornell 将化工流程合成（process synthesis）正式建模为马尔可夫决策过程（Markov decision process，MDP），设计量子增强 RL 算法变体，并通过 state encoding 算法使 qubit 需求与 unit 数量解耦，解决此前量子方案扩展性差的瓶颈。在小规模设计空间中量子 RL 可找到最优解，在中等规模下 per-parameter 效率优于经典 RL baseline，但尚未在大规模问题上展示决定性优势。</p>

  <li><b>关键结果点</b>：尚未在大规模问题上展示决定性优势；state encoding 解耦 qubit 与问题规模是可扩展性的关键先决条件。</li>
