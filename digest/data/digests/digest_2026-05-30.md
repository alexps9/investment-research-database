<h3>Stanford ViewSuite：VLM 三维视角规划评测 benchmark + 视点连接图蒸馏，Qwen2.5-VL-7B 从 2.5% 升至 47.8% 反超 GPT-5.4 Pro</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.29563">ViewSuite: Benchmarking and Improving VLMs for Interactive 3D View Planning</a></li>
  <li><b>信号源</b>：<b>Stanford Jiajun Wu</b> · <b>Northwestern Manling Li</b> · Kangrui Wang（一作，导师 Bill Freeman）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文表明：让通用 VLM "看懂"真实 3D 场景并规划下一步视角，不需要从头设计一套专门的三维感知系统——关键在于训练时用视点连接图（view graph）把所有成功和失败探索轨迹汇聚起来再蒸馏为监督信号，绕开强化学习稀疏奖励的困难。<b>这意味着 3D 主动感知原本作为独立技术方向的门槛正在被通用 VLM 的训练范式内化</b>——Qwen2.5-VL-7B 微调后达到 47.8%，超越 GPT-5.4 Pro 的 18.5%，说明开放小模型在足够的结构化数据下能反超闭源大模型。</p>
</callout>

<p>　　视觉语言模型在静态图像问答和单步感知上已有较强表现，但主动规划——即在真实三维空间中根据当前视角决定下一步应该看哪里——一直是公认的短板。现有工作要么假设视角位置已知，要么仅评测单步决策，缺乏系统性的多步规划评测。</p>
<p>　　ViewSuite 在 ScanNet 真实 3D 点云场景上构建 benchmark，测试 13 个前沿 VLM，发现所有模型在多步视角规划上均大幅失败，且随视点距离增加性能下滑更明显。论文提出的解决方案是让模型先在场景中自由探索，把所有探索轨迹（无论成功与否）汇聚成结构化视点连接图，再将图中丰富的视点转换关系蒸馏为分类、预测等监督信号，最终将 Qwen2.5-VL-7B 从 2.5% 提升至 <b>47.8%</b>，超过 GPT-5.4 Pro（18.5%）和 Gemini 3.1 Pro（21.4%）。</p>

<h3>Anthropic 用稀疏自编码器从 Claude 3 Sonnet 拆出 3400 万个可读特征，验证"欺骗"等安全相关概念可因果控制模型输出</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.29358">Scaling and evaluating sparse autoencoders</a></li>
  <li><b>信号源</b>：<b>Anthropic Trenton Bricken</b> · Adly Templeton（共一）· Tom Conerly（共一）· Tom Henighan（通讯）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这项工作是机械可解释性（mechanistic interpretability）首次在真正的生产级模型上完成规模验证——不是玩具网络，是 Claude 3 Sonnet。稀疏自编码器（SAE）在中间层 residual stream 上学出 3400 万个语义方向，其中包括"欺骗"、"权力寻求"等可以被激活引导（activation steering）因果控制的开关。<b>这意味着"AI 对齐靠黑盒监控"这一工作假设开始出现替代路径：如果 SAE 特征能被系统性审计并在推理时实施干预，监管机构和大模型开发者之间的技术接口将从"行为评测"升级为"机制级合规工具"</b>。</p>
</callout>

<p>　　过去几年，可解释性研究大多停留在小规模合成任务上，在真实生产级大模型上能否找到语义清晰的表示方向，一直是未解问题。Anthropic 的研究路线是：先在小模型上验证稀疏自编码器的有效性，再沿 scaling laws 扩展到 Claude 3 Sonnet 的完整推理管线。</p>
<p>　　这次工作在 Claude 3 Sonnet 中间层 residual stream 上训练了包含最多 <b>3400 万</b>个方向的过完备字典，每个字典元素对应一个可解释特征。提取的特征涵盖多语言泛化、跨模态响应（纯文本训练但能响应图像），以及安全相关概念如 deception（欺骗）、power-seeking（权力寻求）、sycophancy（谄媚）。通过激活引导对这些特征施加扰动，可因果改变模型输出，证明这些方向不是相关性噪声而是真实的功能节点。作者同时指出特征集仍不完整，且目前缺乏对特征质量的严格定量评估方法。</p>

<p><em>* 注：论文为 Anthropic 内部自披露，特征覆盖率和激活引导的可靠性尚未经过外部独立复现验证。</em></p>

<h3>Stanford NeuROK：无需预设物理模型、数据驱动学习物体 4D 动态形变的运动学参数化框架</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.30347">NeuROK: Neural Object Kinematics for 4D Dynamic Generation</a></li>
  <li><b>信号源</b>：<b>Stanford Jiajun Wu</b> · Geng Chen（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文的核心转变是：物理仿真不再需要预先知道"这是布料""这是果冻"，也不再需要手工写出对应的微分方程——模型在大规模 4D 数据上训练，自己学出物体所有可能形变状态的 latent space，再把物理动力学问题压缩到低维空间里用 Lagrangian 力学框架求解。<b>这意味着类别特定物理模型假设作为 4D 仿真的必要前提正在被数据驱动路径替代，能否构建大规模高质量 4D 标注数据集，将成为这条路线能走多远的关键卡点</b>，而非继续细化每类物体的物理建模精度。</p>
</callout>

<p>　　此前，物体动态仿真的主流路径是类别专用的物理参数估计：布料有布料的方程、流体有流体的方程，系统辨识（system identification）需要为每类物体单独标定物理参数，泛化成本高。NeuROK 转向数据驱动路线，不再假设物体类别或物理方程形式。</p>
<p>　　NeuROK 由 transformer-based encoder-decoder 构成：encoder 将物体任意形变状态压缩为低维 latent 向量，decoder 将采样得到的 latent 还原为三维变形几何。仿真时，动力学问题被转化为在低维 latent 空间内求解 Lagrangian 力学方程，避开了高维几何空间的复杂计算。框架在大规模 4D 数据集上训练，在多种动态物体类型（软体、关节体等）上均展现出优于先前类别专用方法的效果和泛化能力。</p>

<p><em>* 注：结果来自自披露的预印本，跨类别泛化能力的边界条件和量产规模数据集构建细节尚未经过独立评测。</em></p>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>传统数据合成要求工程师手写 prompt 描述"目标领域是什么"，DOMINO 把这个步骤省掉了——只需给几个领域参考样本，模型通过对比解耦（contrastive disentanglement）自动提炼跨样本稳定的领域特征，再用它生成合成训练数据。<b>这意味着领域自适应数据合成的门槛从"需要领域专家写描述"降到"只需几个样本"，LLM mid-training 的可扩展性出现新路径</b>——但目前只在编程领域验证，向其他垂直领域的泛化能力仍需观察。</p>
</callout>

<p>　　LLM 领域微调的核心障碍之一是高质量领域数据的稀缺性，而人工编写 prompt 描述目标领域既耗时又难以覆盖细粒度特征，特别是当领域边界模糊或隐式时。</p>
<p>　　DOMINO 采用归纳范式：输入一组领域参考样本，通过 prompt tuning 学习 soft domain token，同时引入对比解耦目标函数，将领域级共享模式与样本特异噪声分离，得到能代表该领域的最小充分表示（minimal sufficient domain representation），再以此引导 LLM 生成领域对齐的合成训练数据。在编程 benchmark 上，DOMINO 合成数据微调后 Pass@1 较强 instruction-tuned backbone 最高提升 <b>4.63%</b>，并超越基于显式 prompt 描述的数据合成方法。</p>

<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.30288">MIRA: Source-Aware Data Selection for LLM Mid-training</a></li>
  <li><b>信号源</b>：<b>Stanford Jiajun Wu</b>（合作）· Haowen Wang（共一，导师 Zhuowen Tu）· Yaxin Du（共一，导师 Siheng Chen）· Xianglong Liu（通讯）</li>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>MIRA 解决的是 LLM mid-training 阶段"用什么数据"的问题，但它的方法是先让强模型对每个数据来源自动归纳出"应该评估什么维度"（rubric discovery），再把这些判断蒸馏成轻量 student scorer 来完成大规模过滤。<b>核心信号在于"评分标准本身应该是数据驱动的"——统一评分表在 21 个不同代码来源上效果不如来源自适应标准</b>，这对当前依赖人工设计质量过滤器的数据工程流程是个直接挑战。</p>
</callout>

<p>　　LLM 的 mid-training 阶段面临的数据选择难题是：不同来源的代码质量维度各异，统一的规则过滤器难以适配所有来源，而直接用强模型打分成本极高。</p>

<h4>DSRD：统一时序记忆与结构传播的动态图框架，14 个 benchmark 上链接预测和节点分类达到 SOTA</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.29453">Dual-Scale Retentive Dynamics for Dynamic Graph Learning</a></li>
  <li><b>信号源</b>：<b>Yi Zhang</b>（资深合作）· Qian Chang（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>动态图学习的长期困境是时序记忆和图结构传播总是分开建模——时序模型不懂拓扑，图神经网络（GNN）不懂时序演化。DSRD 用单一 recurrent 状态同时编码两者，并引入可学习衰减核（adaptive decay kernel）自动适配不同交互频率的节点。<b>双重编码的泛化能力在 14 个真实世界 benchmark 上得到验证，说明这个方向的统一化建模已经走通了，不再是理论可行</b>。往下游看，这个框架对实时风控图谱、推荐系统时序 embedding 等需要同时处理时序和拓扑的场景有直接迁移价值。</p>
</callout>

<p>　　现有动态图方法要么固定 temporal decay 参数，要么只做有限跳图传播，在高低频混合的真实图上泛化能力有限，时序遗忘速率和结构传播深度往往需要人工为不同图类型分别调参。</p>
<p>　　DSRD 维护一个统一的 retentive representation state，通过 dual-scale retentive state 在单一循环公式内同时捕获时序动态与结构传播；adaptive decay kernel 利用可学习的 time-sensitivity 参数，根据节点的交互频率自动切换短期响应与长期记忆的权重。框架从理论上建立了 event-wise 并行聚合与高效循环状态更新之间的等价关系，训练可并行，推理可循环。在 14 个真实世界动态图 benchmark 上，链接预测（link prediction）和节点分类（node classification）任务均在 transductive 和 inductive 两种设置下达到当前最优。</p>

<h4>LAR：权重谱与激活谱重叠度量训练时泛化诊断，3B 参数语言模型预训练中有效追踪泛化转变</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.28975">Log-Alignment Ratio: A Training-Time Generalization Diagnostic for Neural Networks</a></li>
  <li><b>信号源</b>：<b>Ashish Vaswani</b>（资深合作，Google）· Ali Shehper（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>训练时泛化诊断的传统路径是保留验证集、持续评测，LAR（对数对齐比，log-alignment ratio）提供了一个不需要额外验证集的替代方案——通过比较权重矩阵的奇异值分布和输入激活在奇异方向上的投影分布，当两者高度对齐时意味着模型在记忆而非泛化。<b>这个指标仅用前向传播中已有的计算量即可获得，计算开销可忽略，在 grokking 场景和 3B 参数语言模型预训练中均能有效追踪泛化转变</b>，说明它作为通用训练监控工具的潜力是真实存在的，而非理论构造。</p>
</callout>

<p>　　大规模模型预训练中判断何时出现过拟合，通常需要一个独立验证集持续评测，但这既有数据成本，也引入了泛化估计的分布偏差。LAR 将泛化诊断问题转化为权重谱和激活谱的对齐度测量，无需额外数据。</p>
<p>　　具体而言，对矩阵做奇异值分解（SVD）后，得到归一化奇异值平方分布 p 和输入激活在各奇异向量方向上的归一化投影平方分布 q；两者的 KL 散度反映了参数与激活的对齐程度，对齐度高意味着模型在记忆而非在泛化。实验表明，在 grokking 任务中 LAR 能预测学习函数有效维度（k ≈ n^{2(1-LAR)}），在 3B 参数语言模型预训练中，LAR 偏离基线的速率加快能预示过拟合临近，且不产生额外计算开销。</p>

<h4>OccamToken：无需训练的视觉 token 剪枝框架，register token 作噪声基线，1.4% token 保留超 93% 精度</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.29657">OccamToken: Training-Free Visual Token Pruning via Register-Anchored Relative Evidence Testing</a></li>
  <li><b>信号源</b>：<b>Ting Chen</b>（合作，前 DeepMind）· Li-Sheng Geng（共一）· Guohao Chen（共一，导师 Bernard Ghanem）· Jianfei Yang（通讯，Nanyang Technological University）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>VLM 推理成本居高不下的重要原因是视觉 token 数量庞大，但现有剪枝方法依赖"哪些 token 全局重要"的绝对排名，attention sink 效应会系统性扭曲这个分数。OccamToken 改用 register token 作为天然噪声基线做相对比较，不需要训练任何额外参数。<b>在 LLaVA-NeXT 上压缩到仅保留 1.4% token 仍保持超 93% 精度，说明 VLM 视觉 token 的实际信息密度远低于当前部署中使用的数量</b>，无需重新训练模型即可大幅降低推理成本的路径已经打通。</p>
</callout>

<p>　　OccamToken 利用 register token 在 Transformer 中自然积累低信息 attention 模式的特性，将其作为"噪声基线"：对每个视觉 token 评估其是否提供了超出 register 参考的额外信息（相对证据测试），动态阈值由 register attention 分布决定，实现图像自适应和查询自适应的双重剪枝，无需任何参数训练。在 LLaVA-NeXT 上将 2880 个视觉 token 压缩至约 40 个（保留率 <b>1.4%</b>），仍保留超过 <b>93%</b> 的原始精度，在 LLaVA-NeXT、LLaVA-v1.5 和 Qwen3-VL 上均验证有效。</p>

<h4>RoboWits：双臂机器人推理导向 benchmark，30 个 seed tasks + 208 个变体任务揭示 VLA 策略适应能力短板</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.30326">RoboWits: Benchmarking Cognitive Reasoning and Creative Tool Use for Bi-Manual Robot Manipulation</a></li>
  <li><b>信号源</b>：<b>Yejin Choi</b>（合作）· <b>UMass Amherst Chuang Gan</b>（资深合作，MIT-IBM Watson AI Lab 首席研究员）· Chunru Lin（共一）· Hongxin Zhang（共一）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>RoboWits 的核心发现直接挑战了当前机器人策略研究的一个隐含假设——"在原始任务上做好 fine-tuning 就足够了"。预训练视觉语言动作模型（VLA）在 seed task 上 fine-tuning 后可以初步成功，但面对几何、材料或装配关系改变的变体任务时性能大幅下滑。<b>这意味着当前 VLA 的泛化能力瓶颈不在于感知或动作执行，而在于推理和策略适应——这个短板在过去的评测框架里是被系统性低估的</b>，RoboWits 的变体任务生成机制使得这个差距第一次被可量化地暴露出来。</p>
</callout>

<p>　　双臂操控机器人的评测此前以能否完成标准任务为主要指标，缺乏对推理泛化能力的系统测试——在课本例题上能拿满分，并不意味着能处理变式题。</p>
<p>　　RoboWits 通过 multi-agent 协作流水线构建推理导向的异常场景任务：由种子任务生成、评估指标生成、场景生成和任务变异（mutation）四类 agent 协作，在 30 个 seed tasks 基础上引入几何、材料和装配三类维度的变化，生成 208 个变体任务。评测对象包括 robot policies、预训练 VLA 和 oracle-state planners 三类基线。结果表明，所有测试的预训练 VLA 在 seed tasks 上经 single-task fine-tuning 后有初步成功，但在变体任务上性能显著下滑，证明当前模型在策略适应、创造性工具使用和鲁棒推理方面仍有明显差距。</p>

<h4>VLA-Trace：三层证据链诊断 VLA 内部机制，揭示 π0.5 与 OpenVLA 多模态路由策略的根本性差异</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.30117">VLA-Trace: A Unified Framework for Diagnosing Vision-Language-Action Models</a></li>
  <li><b>信号源</b>：<b>Yi Zhang</b>（合作）· Haoyuan Shi（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>VLA-Trace 提供的不是一个更好的 VLA 模型，而是一套能让研究者看清"VLA 模型内部发生了什么"的诊断工具，从表征动态追踪、attention 因果干预到行为探测三层构建完整证据链。<b>π0.5 和 OpenVLA 在 fine-tuning 动态和 action decoding 上存在根本性机制差异这个发现，说明当前对 VLA 的评测和改进方案可能是建立在不同内部机制的基础上——这对下一代 VLA 的架构设计和迁移学习策略是个直接提示</b>。</p>
</callout>

<p>　　VLA 模型如何将视觉语言知识转化为具身控制动作，此前基本上是一个黑盒，研究者只能通过最终任务成功率间接推断模型的内部工作原理，难以定位失败的具体原因。</p>
<p>　　VLA-Trace 构建三层统一证据链：表征层使用跨模态与 checkpoint 漂移的居中核对齐（centered kernel alignment，CKA）追踪表征演化；因果干预层通过 attention knockout（注意力剔除）屏蔽特定模态的注意力头，定位哪些层真正决定动作输出；行为探测层在实际 rollout 中注入语义扰动，测试模型是否真正理解指令。实验表明 π0.5 和 OpenVLA 在 fine-tuning 期间表现出截然不同的模态特定适应动态，依赖不同的多模态路由策略完成 action decoding，且两者在细粒度语义跟随上均有明显短板。</p>

<h4>ICC：将循环一致性约束从 slot 空间迁移到重建流形，解决视频物体中心学习中的特征坍塌问题</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.30211">Implicit Cycle Consistency for Video Object-Centric Learning</a></li>
  <li><b>信号源</b>：<b>Zhiyuan Li</b>（合作）· Rongzhen Zhao（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>循环一致性（cycle consistency）在多目标跟踪（MOT）中是成熟工具，但直接施加到视频物体中心学习（OCL）的 slot 空间会触发特征坍塌（feature collapse）——因为同一场景的 slot 表示本质上是随机非唯一的，强制点对点对齐等于强迫多种合法答案收敛成一种。ICC 把约束转到重建流形（reconstruction manifold）上，让 slots 在"最终重建出什么"这件事上达成软共识而不是在"每个 slot 是什么"上刚性对齐。<b>这个转移说明 OCL 中的监督设计需要考虑 slot 表示的内在歧义性，未来的训练策略设计可以此为参考</b>。</p>
</callout>

<p>　　视频物体中心学习旨在从视频中无监督地分离出独立物体的 latent slot 表示，循环一致性约束被用来强化跨帧的 slot 时序关联，但 slot 空间的随机性和非唯一性使得刚性的点对点循环映射会坍塌到单一模式。</p>
<p>　　ICC 在自监督视频 OCL 框架下，将循环一致性约束从 slot 空间迁移到重建空间（像素或特征重建流形），通过重建层面的一致性间接约束 slot 的时序关联，允许 slots 以集体方式重建出一致的视觉场景，而不要求每个 slot 跨帧刚性对齐。在复杂视频 OCL benchmark 上，ICC 优于显式循环一致性（ECC）基线，同时有效规避了特征坍塌。</p>

<h3>ai4s (AI for Science)</h3>

<h4>iLoRA：首个贝叶斯图条件 LoRA 框架，推断微生物交互 latent graph 用于 IBD 多队列诊断</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2605.30179">iLoRA: Bayesian Graph-Conditioned LoRA for Microbiome Disease Diagnosis</a></li>
  <li><b>信号源</b>：<b>Song Yang</b>（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>LoRA 微调通常用静态低秩更新适配下游任务，对样本间的潜在交互结构视而不见。iLoRA 把图结构推断和预测联合训练，让每个样本的低秩更新都依赖于它推断出的微生物交互拓扑（interaction topology）。<b>更关键的是恢复的 latent graph 与人工标注和队列级微生物组关联一致——这说明模型不只是在拟合标签，而是真正学到了生物学上有意义的结构，这类可解释图结构对药物靶点发现和队列层面的因果推断有直接价值</b>。</p>
</callout>

<p>　　炎症性肠病（IBD）的微生物组诊断面临样本量小、队列间分布偏移大和微生物间相互作用未被建模三大挑战，传统 LoRA 微调忽略了不同患者样本之间微生物的交互关系。</p>
<p>　　iLoRA 包含两个联合训练的分支：图推断分支从物种丰度等输入特征推断贝叶斯（Bayesian）latent interaction graph，对图结构引入先验以校准不确定性；图条件 LoRA 分支将推断图的结构信息作为条件信号调制 LoRA 的 A/B 矩阵，使每个样本的低秩更新依赖于其推断出的微生物交互拓扑。在多队列 IBD 诊断和交互式问答两项任务上均优于强 LoRA 和贝叶斯自适应基线，恢复的 latent graph 与领域知识吻合，且额外计算开销适中。</p>

<h4>OOD-GraphLLM：首个将图 LLM 用于分布外泛化药物协同预测，检索增强指令微调对齐分子拓扑与语义</h4>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>

</callout>
