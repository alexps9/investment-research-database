<h3>NVIDIA 发布 Cosmos 3：单一专家混合架构统一五种模态，机器人策略、视频生成双线获最佳开源评级</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02800">Cosmos 3: An Omnimodal World Foundation Model</a></li>
  <li><b>信号源</b>：<b>NVIDIA Wenliang Dai</b> · <b>NVIDIA Jim Fan</b> · <b>MIT Song Han</b> · <b>Haoxiang Wang</b> · <b>Yuke Zhu</b></li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>NVIDIA 正在用 <b>Cosmos 3</b> 争夺一个定义权——把"Physical AI backbone"从"多个专用模型的流水线"换成"单一全模态框架"。这件事的意义不在于某项指标提升，而在于：<b>如果通用全模态预训练能持续保持各垂直方向的 SOTA，那么为视频生成、机器人策略、VLM 分别维护独立模型栈的逻辑就会被动摇</b>——专用模型赛道作为独立产品线的存在意义会被系统性地向下压缩。Cosmos 3 同时在 Artificial Analysis 和 RoboArena 两个独立榜单获评最佳开源，说明这不只是自我宣称。</p>
</callout>

<p>　　具身智能领域长期面临的问题是：视觉理解、视频生成、机器人策略控制分别由不同的专用架构承担，模态之间的信息流动需要复杂的工程拼接。Cosmos 3 采用专家混合变换器（Mixture-of-Transformers，MoT）架构，为语言、图像、视频、音频与动作序列各自路由专用的 transformer expert，同时共享整体调度——这避免了单一密集变换器（dense transformer）对所有模态使用相同参数的低效问题，也无需维护五套独立模型。</p>

<p>　　实验结果显示，Cosmos 3 post-trained 模型在 Artificial Analysis 排名中获 Text-to-Image 和 Image-to-Video 双项最佳开源模型，同时在 RoboArena 上获评最佳开源策略模型，验证了全模态统一训练在理解与生成任务上均可达到前沿水平。完整代码、模型权重及合成数据集以 OpenMDW-1.1 许可证开放发布。</p>

<h3>LEAP：agentic 框架让通用 LLM 形式化定理证明一次解题率从 10% 升至 70%，Putnam 竞赛全题通过</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03303">LEAP: A Generalist Agentic Framework for Formal Theorem Proving</a></li>
  <li><b>信号源</b>：<b>Google Quoc V. Le</b> · Po-Nien Kung（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文表明：通用 LLM 在形式化数学证明（formal theorem proving）上达到专用系统水平，不需要专门训练，只需要一套把"自然语言推理"与"Lean 编译器反馈"循环接通的 agentic 框架。<b>这意味着原本专门为形式化证明设计的系统，作为通用 LLM 之外的独立技术方向，其存在价值正在被快速压缩</b>——LEAP 在 Lean-IMO-Bench 上以 70% 解题率超过专用金牌级 IMO 系统的 48%，差距已经翻转。Putnam 竞赛全题通过提供了一个干净的外部验证锚点。</p>
</callout>

<p>　　形式化数学证明长期是专用系统的主场：Lean、Isabelle 等形式化语言系统需要逐步可验证的证明步骤，通用 LLM 因为无法精确跟踪 Lean 编译器的报错反馈，一次解题率极低。LEAP 的核心思路是将复杂证明问题拆分为较小的子目标，用 LLM 生成非正式推理蓝图，再通过与 Lean 编译器的迭代交互（自我精炼，self-refinement）把蓝图逐步转化为机器可验证的形式化证明——每次编译失败后，模型根据报错修订对应的证明片段，类似程序员在 IDE 中循环调试直到编译通过。</p>

<p>　　评测结果：在自建 Lean-IMO-Bench 上，LEAP 将通用 LLM 的一次性形式化解题率（one-shot formal solve rate）从低于 10% 提升至 70%，超过专用金牌级 IMO 系统的 48% 基准；在 2025 Putnam 竞赛中解决全部 12 道题，与前沿专用形式化数学模型持平；还自主完成了 Knuth Hamiltonian 分解问题关键子问题的形式化证明，展示了研究级实用性。</p>

<p><em>* 注：Lean-IMO-Bench 为论文团队自建评测集，结论尚待独立第三方复现验证。</em></p>

<h3>Humanoid-GPT：20 亿帧 mocap 预训练，人形机器人全身运动 zero-shot 泛化跟踪</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03985">Humanoid-GPT: A GPT-Style Transformer for Whole-Body Motion Tracking</a></li>
  <li><b>信号源</b>：<b>王鹤 He Wang</b> · Zekun Qi（共一）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文把"大规模预训练赋予 zero-shot 泛化"的逻辑从语言直接搬到了全身运动控制（whole-body control）上，用 <b>20 亿帧</b> mocap 数据训练因果注意力变换器（causal attention Transformer）。<b>这意味着浅层多层感知机（MLP）tracker 长期依赖的"针对每套动作专项训练"路线，在泛化覆盖上正在被预训练方案超越</b>——scaling 实验进一步验证数据量与模型容量共同驱动性能，这个边际收益仍在增加。</p>
</callout>

<p>　　人形机器人全身运动控制此前依赖浅层 MLP tracker，这类方案在高动态动作和泛化之间存在明显的取舍：专注高动态的 tracker 往往在没见过的动作上失效，反之亦然。Humanoid-GPT 将不同来源的 mocap 数据统一 retarget 到同一骨架，以 GPT 风格的因果自注意力进行序列自回归建模，学习从运动参考序列到控制输出的映射，推理时直接 zero-shot 跟踪未见过的运动参考。</p>

<p>　　2B 帧预训练语料整合了 AMASS 等主流公开数据集与大规模内部录制，覆盖多样化人体运动模态。实验结果在 zero-shot 泛化和高动态跟踪两个维度同时建立了新性能前沿，打破了此前 MLP tracker 的敏捷性和泛化性之间的固有取舍。scaling 分析验证了数据量和模型容量共同驱动 zero-shot 运动跟踪性能的提升。</p>

<p><em>* 注：2B 帧语料中含大量内部录制数据，外部复现依赖是否能获取等价规模的 mocap 数据。</em></p>

<h3>Google 发布 Gemma 4 12B：去掉视觉 encoder，16GB VRAM 本地运行，Apache 2.0 开源</h3>
<ul>
  <li><b>论文 / 公告</b>：<a href="https://x.com/googleaidevs/status/2062204432658386950">Google AI Developers 官方公告</a></li>
  <li><b>信号源</b>：<b>Google AI Developers</b></li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p><b>Gemma 4 12B</b> 抛弃独立视觉和音频编码器（encoder），让视觉和音频 token 直接进入大语言模型主干（backbone）。这套无编码器（encoder-free）设计使模型在 16GB 消费级显卡上即可运行，同时保持接近 26B 规模专家混合（MoE）模型的性能。<b>端侧多模态推理门槛的进一步压缩，意味着依赖"云端 API 代理多模态能力"的应用层方案面临更直接的替代压力</b>；Apache 2.0 开源将这种压力同步传导到整个开发者生态。</p>
</callout>

<p>　　传统多模态模型将视觉和音频各自接入独立的编码器进行特征抽取，再对接语言模型主干。这类架构在端侧部署时往往因参数量和内存占用较大而受限。Gemma 4 12B 的无编码器架构让视觉和音频 token 直接参与语言模型的注意力计算，简化了架构的同时显著降低了整体参数量。</p>

<p>　　Google DeepMind CEO Demis Hassabis 同步披露 Gemma 4 系列累计下载量已超 1.5 亿次，显示该系列在开发者社区中获得了广泛部署。模型以 Apache 2.0 许可证开源，权重已上传至 Kaggle 和 HuggingFace，并与 llama.cpp、MLX、vLLM、Ollama、SGLang 等主流推理框架兼容。</p>

<h4>ALAR：双模式 LLM Agent 推理框架，按需切换隐性推理与显式思维链，token 消耗最多降 84.6%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02871">ALAR: Adaptive Latent-to-Articulate Reasoning for LLM Agents</a></li>
  <li><b>信号源</b>：<b>Yi Zhang</b> · Dongwon Jung（一作，导师 Muhao Chen）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>现有 LLM agent 框架在每个决策步骤都强制生成完整的显式推理链（chain-of-thought，CoT），而 ALAR 表明：大多数 agent 决策步骤根本不需要这些文字——用隐性推理（latent reasoning）快速过就够了，只有在真正复杂的步骤才展开显式 CoT。<b>这意味着"token 消耗 = 推理质量"的假设需要被修正</b>：在工具调用任务上最多降低 84.6% 的 token 消耗同时保持性能，说明当前 agentic inference 有大量冗余。往供应链方向看，inference serving 成本优化的空间不只在硬件层——框架层的推理模式选择可能是更直接的杠杆。</p>
</callout>

<p>　　多轮 agentic 任务的推理成本长期被视为难以压缩，因为去掉 CoT 通常意味着性能下降。ALAR 的核心贡献是将推理解耦为两种模式：常规决策步骤使用紧凑的 latent reasoning，仅在判断为复杂步骤时才切换为显式 CoT。框架以 agent 的动作序列而非推理链作为监督锚点训练 latent reasoning，通过优化目标引导模型学会在 latent reasoning 足够时抑制显式 CoT 生成。</p>

<p>　　评测在 agentic search 和 tool-use 两类 benchmark 上进行，ALAR 在保持与全量 CoT 基线相当或更优准确率的同时，将生成 token 数分别最多压缩 43.6% 和 84.6%。这种模式选择策略通过额外优化目标实现，推理开销随任务难度自适应。</p>

<h4>EvoDS：自进化数据科学 Agent，自主技能合成与自适应上下文压缩，平均超 SOTA 开源 agent 28.9%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03841">EvoDS: A Self-Evolving Autonomous Data Science Agent</a></li>
  <li><b>信号源</b>：<b>Hao Liu</b></li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>EvoDS 同时解决了 data science agent 的两个结构性瓶颈：静态工具集（不能扩展新技能）和上下文爆炸（长流程中 token 耗尽导致任务失败）。用自主技能合成（Autonomous Skill Acquisition，ASA）让 agent 把成功的代码片段自动变成可复用的技能；用自适应上下文压缩（Adaptive Context Compression，ACC）把"保留哪些历史"建模为强化学习控制问题。<b>这意味着 data science agent 的核心差异化正在从"用哪个基础模型"转向"工具和上下文管理的架构设计"</b>，彻底消除 out-of-token 失败是一个可被独立验证的干净信号。</p>
</callout>

<p>　　现有数据科学 agent 通常使用固定的工具集和暴力截断上下文来处理长任务，前者限制了 agent 处理新任务类型的能力，后者则在复杂多步骤流程中频繁导致失败。EvoDS 采用两阶段多智能体训练框架：第一阶段 ASA 让 agent 在执行任务时将成功的代码片段抽象为带签名的可调用技能，写入技能库后续可检索复用；第二阶段 ACC 将"保留哪些历史上下文"视为学习到的策略，agent 根据当前任务阶段动态决定压缩粒度。理论分析表明，分层设计可降低工具选择误差。</p>

<p>　　在四个多样化数据科学 benchmark 上，EvoDS 平均性能领先现有开源 agent 28.9%，并通过 ACC 策略彻底消除了长流程中的 out-of-token 失败问题。</p>

<h4>SkillDAG：将 LLM Agent 技能库建模为有向图，结构化检索在 ALFWorld 上超 Graph-of-Skills 基线 +12.8 分</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03056">SkillDAG: A Typed Directed Graph for Structured Skill Retrieval in LLM Agents</a></li>
  <li><b>信号源</b>：<b>Yang You</b> · Tong Bai（共一，导师 Yang You）· Wan ZhengLin（共一，导师 Yang You）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>SkillDAG 把技能选择从向量相似度匹配升级为结构化图检索，让依赖关系、冲突关系、特化关系都被显式建模为图的边类型。<b>更关键的是，这个图在 agent 执行过程中持续自演化——无需离线重建，这让技能库可以随实际使用不断扩展和修正</b>。在技能池扩大 10 倍时仍保持检索质量，说明这套架构对规模化更鲁棒。对 agentic 产品而言，这条路线意味着"让 agent 通过使用来积累和优化技能"是可以工程实现的，不只是概念。</p>
</callout>

<p>　　当前 LLM agent 技能检索普遍依赖向量相似度，缺乏对技能之间结构关系的建模，导致检索结果常常聚堆在同一语义区域，同时无法感知技能之间的依赖和冲突。SkillDAG 将技能库表示为带类型有向图，边类型包括依赖、冲突、特化、重复等。推理时 agent 通过结构化检索接口同时获取向量匹配结果、类型边邻居和冲突信号，形成多源候选集。采用提议后提交（propose-then-commit）协议，agent 可在执行后提交新边，使图结构跨 episode 累积演化。集合单调在线编辑（set-monotone online edits）保证新增边只扩大正确技能的召回，不驱逐已有命中。</p>

<p>　　在 ALFWorld 上成功率达 67.1%，比最强 Graph-of-Skills 基线高 +12.8 点；SkillsBench 奖励达 27.3%，超越基线 +8.6 分，检索精确率（Ret@K）从 65.5 提升至 78.2。技能池扩大 10 倍时候选排名保持鲁棒。</p>

<h4>Activation Oracle 改进与 AObench：四项训练优化提升 LLM 残差流解读质量，发布首个系统性评测基准</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02609">Improving Activation Oracles for Mechanistic Interpretability</a></li>
  <li><b>信号源</b>：<b>Neel Nanda</b>（资深合作）· Jan Bauer（共一）· Celeste De Schamphelaere（共一）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>激活解读器（Activation Oracle，AO）是机制可解释性（mechanistic interpretability）研究中把 LLM 内部残差流状态转化为人类可读文字的一类方法。<b>这篇论文的关键信号不是 AO 能力有多大提升，而是它发布了第一个系统性评测基准 AObench</b>——这意味着该方向开始有了可被跨团队引用和比较的评测标准，是一个方向从"各自探索"走向"规范化竞争"的起点。同时对 text-inversion confounds（文本反转干扰）的识别，说明现有评测存在系统性偏差，后续数据值得关注。</p>
</callout>

<p>　　激活解读器试图将 LLM 残差流中的激活值转化为人类可读的语义描述，是机制可解释性研究的重要工具。本文从四个维度对 AO 训练流程进行改进：引入在线策略展开（on-policy rollout）使训练分布贴近推理时的真实激活分布；改进对话数据集质量；扩展输入到多层残差流以捕捉更丰富上下文；改进激活注入公式以减少文本反转（text-inversion）干扰。</p>

<p>　　四项改进在可读性和幻觉减少等质量层面有显著提升，但核心能力指标改善有限。更重要的贡献是发布 AObench，作为首个系统性 AO 评测套件，为后续研究提供统一的评测标准，并明确识别了 text-inversion confounds 对评测可信度的干扰。</p>

<h4>神经网络群论特征学习理论：两层网络在 group composition 任务中神经元收敛到单一不可约表示</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02993">Feature Learning in Neural Networks via Group Composition: A Representation-Theoretic Analysis</a></li>
  <li><b>信号源</b>：<b>Zhuoran Yang</b>（资深合作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文把"神经网络学到了什么"这一问题在群结构（group composition）任务上给出了严格的数学答案：每个神经元几乎必然收敛到一个单一的不可约表示（irreducible representation），梯度流等价于黎曼梯度上升（Riemannian gradient ascent）。<b>更值得注意的是，它揭示了一种跨层傅里叶系数的秩一对齐（rank-one alignment）现象</b>——这可能为解释 grokking（泛化突然跳跃）等训练动态现象提供新的理论框架。对在训练理论方向布局的团队，这是一个值得追踪的理论工作。</p>
</callout>

<p>　　神经网络特征学习的理论分析长期缺乏精确的收敛描述。该工作将投影梯度流（projected gradient flow）提升到傅里叶（Fourier）域，把训练动态重新表述为定义在矩阵值群表示空间上的黎曼梯度上升，以表示论能量函数为核心分析对象。跨层系数的秩一对齐类似矩阵的幂迭代（power iteration）——梯度流自然过滤掉低能量表示，使权重矩阵坍缩为主导不可约表示的秩一结构。</p>

<p>　　对 Abelian 群，论文给出了完整的 population-level 描述：随机初始化促进跨非平凡表示的均匀多样化，并诱导 Haar-uniform 相位，通过多数投票（majority-vote）机制近似指示函数。相位对齐和表示竞争均以指数速率收敛。理论结果对 Abelian 群完整，对一般有限群分析了神经元级别的收敛行为。</p>

  <p>Demo2Tutorial 把录屏变成结构化教程，同时让两类受众受益：人类学得更快，GUI agent 的规划能力得到提升。<b>这件事真正有意思的地方在于：同一套生成的教程既是人类可读文档，也是 agent 的规划知识来源</b>——这说明人类工作示范数据的价值不只是供人类消费，可以被直接转化为 agent 训练素材，降低了 GUI agent 数据采集的门槛。往下游看，企业软件培训内容的大规模生产方式可能因此改变。</p>
</callout>

<p>　　软件教程的生成长期依赖人工撰写，成本高且难以保持与软件版本同步。Demo2Tutorial 由四个模块组成：录制器采集操作录屏；多模态动作解析器（Action Parser）融合视觉截图与交互日志，重建每个操作的语义意图；步骤规划器（Step Planner）构建包含目标与子步骤的层级任务图；教程合成器（Tutorial Composer）生成结构化图文教程。</p>

<p>　　评测基于官方软件文档衍生的 benchmark，Demo2Tutorial 生成的多模态教程质量超越人工撰写教程，并显著优于 baseline 方法。用生成教程训练 GUI agent 后，规划能力和泛化能力均得到提升；真实用户测试显示教程也能加快人类的任务完成速度。</p>

<h4>STS：两阶段视觉 token 剪枝，先排斥采样保空间多样性，再指令感知注意力过滤无关 token</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03569">Structure-to-Semantics: A Two-Stage Visual Token Pruning for VLMs</a></li>
  <li><b>信号源</b>：<b>Kai Zhang</b>（共一）· Jiahui Wang（共一）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>现有视觉语言模型（VLM）的视觉 token 剪枝大多依赖注意力分数（attention score）单一指标，导致保留的 token 聚堆在同一语义区域、丢失空间覆盖。STS 将剪枝拆成两步：先用排斥采样（repulsion-based sampling）强制 token 在空间上相互推开，再用指令感知的交叉注意力（instruction-aware cross-attention）做语义筛选。<b>这意味着 VLM 推理效率优化从单一指标转向"结构多样性加语义相关性"的联合目标</b>，细粒度任务的对齐质量有实质提升空间，下游的本地部署和边缘推理方案可能从中直接获益。</p>
</callout>

<p>　　视觉 token 剪枝是降低 VLM 推理开销的重要手段，但直接依赖注意力分数剪枝会导致高分 token 集中于语义相似区域，关键的空间上下文细节被丢弃。STS 的排斥采样第一阶段类似泊松圆盘采样，强制被保留的 token 在特征空间中相互推开，防止选择结果聚堆；第二阶段以文本 embedding 为查询（query）、视觉 token 为键值（key/value），计算指令相关性分数进行二次筛选，使最终保留的 token 兼顾空间分布与语义相关性。</p>

<p>　　在多个 benchmark 上，与仅依赖注意力分数的剪枝方法相比，STS 既提升了保留 token 的结构多样性，也改善了细粒度任务的指令对齐质量。</p>

<h4>VSTAT：视频视觉状态追踪 benchmark，SOTA 多模态大模型表现仅略优于先验基线，远低于人类</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03920">VSTAT: A Benchmark for Visual State Tracking in Videos</a></li>
  <li><b>信号源</b>：<b>Saining Xie</b>（资深合作）· Sihyun Yu（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>VSTAT 揭示了一个让人不安的结论：当前多模态大模型（MLLM）在视频理解上的表现仅略优于"直接猜答案分布"的先验基线。更具体的诊断是，<b>模型"会想"但"看不见"——推理链本身逻辑正确，但触发推理所需的视觉事件根本没被感知到</b>。这意味着当前 MLLM 视频理解产品的性能上限，不在于推理链质量，而在于视觉感知的连续追踪能力；这个瓶颈目前连 agentic 方案也无法绕过。</p>
</callout>

<p>　　现有视频理解评测（如问答任务）往往可以从单帧信息推断答案，无法真正衡量模型对跨帧状态变化的持续追踪能力。VSTAT 包含 834 个合成和真实视频片段以及 1500 道必须整合整段视频时序信息才能回答的问题，专门针对实体状态和事件的连续追踪。</p>

<p>　　评测发现，SOTA MLLM 在 VSTAT 上的表现与 answer-prior 基线（直接猜概率最高答案）差距极小，与人类表现差距显著。通过将模型的推理链与对应视频帧逐一对比，研究者定位到失败主要发生在"视觉感知"阶段而非"语言推理"阶段：模型的文字推断逻辑本身是对的，但它感知不到触发推断所需的视觉事件。包括 MLLM-based video agent 和 coding agent 在内的 agentic 方案也均未能有效弥合此差距。</p>

<h4>ConTrack：基于 RL 约束优化的灵巧手操控轨迹跟踪，dual-variable 自适应调节任务与风格权衡</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.03177">ConTrack: Constrained Reference Tracking for Dexterous Manipulation via RL</a></li>
  <li><b>信号源</b>：<b>Xiaolong Wang</b>（资深合作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>ConTrack 把灵巧手操控中"物体轨迹跟踪"和"动作风格保真"之间的权衡，从手工调参的超参数变成了一个自适应学习问题。Lagrange 对偶变量（dual-variable）自动调节这两者的权重，配合从中间状态重置（mid-trajectory reset）来稳定长序列强化学习训练。<b>这件事的意义在于：接触丰富的长序列灵巧操控的泛化问题，此前被视为 RL 训练稳定性的硬瓶颈，ConTrack 提供了一个可在真实机器人上验证的工程方案</b>。往下游看，手术机器人、精密装配等对动作保真度要求高的场景，这条路线的成熟度在上升。</p>
</callout>

<p>　　灵巧手操控的强化学习训练长期面临两个难题：一是任务目标（物体跟踪精度）和风格目标（关节动作与示范保真）难以同时优化，手工调参费时且不鲁棒；二是长序列任务中 agent 因早期失误无法到达后半段，导致训练样本不均匀。ConTrack 将物体目标轨迹跟踪设为硬约束，策略在满足该约束的前提下最大化动作保真度；约束满足程度通过 Lagrangian 对偶变量动态调节。自适应中间轨迹重置库（adaptive mid-trajectory reset library）则存储策略可达的中间仿真状态，用于课程式重置，覆盖训练序列的后半段。</p>

<p>　　在仿真和真实机器人实验中，物体位姿准确率和任务成功率均显著优于现有方法，同时保持关节与接触时序的保真度。</p>
