<h3>微软 MSR Jianfeng Gao 团队发布 OpenWebRL：4B 视觉网页 Agent 靠少量示范 + 在线 RL 媲美 OpenAI CUA，网页 Agent 训练摆脱大规模人工标注依赖</h3>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02031">OpenWebRL: Training Visual Web Agents via Online Multi-Turn RL on Real Websites</a></li>
  <li><b>信号源</b>：<b>Microsoft Research Jianfeng Gao</b> · Rui Yang（一作，导师 Tong Zhang）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>这篇论文表明：训练视觉网页 agent 不必依赖大规模人工标注轨迹——只用 0.4K 初始化示范数据和 2.2K 在线任务，一个 4B 模型就能通过稀疏的轨迹级成功信号（trajectory-level reward）在真实浏览器里自学，最终在 Online-Mind2Web 上达到 <b>67.0% 成功率</b>，直接对标 OpenAI CUA。<b>这意味着"标注数据是 web agent 训练瓶颈"的前提正在被拆除</b>——开源小模型走在线强化学习（online RL）这条路，已经能和专有闭源系统站在同一条起跑线上。</p>
</callout>

<p>　　网页 agent 的主流训练路径长期依赖大量人工演示数据，从 Mind2Web 到 WebArena，几乎所有基准都预设了高质量离线轨迹作为监督信号来源。4B 以下开源模型在 live-web 任务上被认为远落后于 GPT-4o 级别的专有系统，数据标注成本和闭源 API 优势构成双重壁垒。</p>
<p>　　OpenWebRL 提出把问题转换为在线强化学习：先用 0.4K 精选轨迹做监督初始化，再让 agent 在真实网站上反复尝试，只用轨迹成功与否作为稀疏奖励信号，multimodal context 管理模块负责处理长截图序列。OpenWebRL-4B 在 Online-Mind2Web 达到 67.0%、DeepShop 达到 64.0%，同等规模开源 agent 中 SOTA，与 OpenAI CUA 和 Gemini CUA 保持竞争力。研究同时系统分析了 online RL 在 agentic reasoning 上改善的关键设计选择（奖励粒度、探索策略、context 截断），为后续工作提供了清晰的复现路径。</p>

<h4>Stanford Diyi Yang 团队提出 SkillHarm：Agent Skill 供应链攻击成功率达 86.3%、现有防御全面失效，agentic 安全架构假设需重估</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02540">SkillHarm: Evaluating Skill-Based Attacks on AI Agents</a></li>
  <li><b>信号源</b>：<b>Stanford Diyi Yang</b> · Yuting Ning（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>AI agent 安全研究的注意力长期集中在 prompt injection 和越狱（jailbreak），但 SkillHarm 指出了一个更结构性的攻击面：<b>第三方 skill 包本身就是持久化的攻击载体</b>——固定载荷投毒（Fixed-Payload Poisoning，FPP）调用即触发（攻击成功率 86.3%），自我变异投毒（Self-Mutating Poisoning，SMP）则在首次良性执行后静默篡改 skill 内容、等待下次复用时引爆（成功率 69.3%）。现有防御措施的表面失败率被系统性低估，因为大量"防御成功"实为 agent 根本未接触到中毒文件。如果 agentic 产品的 skill marketplace 模式继续扩展，这套攻击路径的威胁会随 skill 生态规模线性放大。</p>
</callout>

<p>　　当前 agent 安全研究大多假设威胁来自用户侧或 LLM 输出侧，对 tool/skill 这一中间层关注不足。随着 OpenAI Codex plugins、Anthropic 工具调用生态快速扩张，第三方 skill 包的安全边界问题越来越紧迫，但此前缺乏系统性评估框架。</p>
<p>　　SkillHarm 构建了覆盖 skill 使用全生命周期的评测基准，879 个样本横跨 71 项技能，12 种风险类型按 agent 工作流组件分类（数据管道、系统环境、agent 自主决策）。通过 AutoSkillHarm 流水线（coding agent 自动生成毒化 skill 包），FPP 和 SMP 两类场景均实现规模化测试。实验表明，当前主流 agent 在 FPP 下攻击成功率 86.3%、SMP 下 69.3%；更关键的发现是，现有防御的表观失败率因"agent 未接触中毒文件"而被系统低估，并非真正的防御能力。</p>

<h4>HKUST / Ke Tang 团队提出 PaW：世界建模辅助监督融入在线 RL 协同训练，推理零额外开销提升 language agent</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02388">Policy and World Model Co-Training for Language Agents</a></li>
  <li><b>信号源</b>：<b>Xin Wang（合作）</b> · Ning Lu（共一）· Baijiong Lin（共一，HKUST）· Ke Tang（通讯）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>PaW 的核心发现是：on-policy RL rollout 本身已经隐含了训练世界模型（World Model）所需的（action, next observation）信号，不必引入独立仿真器或额外推理步骤。<b>这意味着"世界建模需要独立模块"的假设在 language agent 训练场景下可能并不成立</b>——把 WM 辅助损失直接附加到策略梯度训练上，既不增加推理开销，又能在三个 agentic benchmark 上对多种模型和 RL 算法取得一致性提升。这套思路对当前主流的"先训 world model、再训 policy"两阶段管线构成方法论上的挑战。</p>
</callout>

<p>　　在线强化学习训练 language agent 时，一个常见的直觉是：如果模型能预测环境的下一个状态，它的探索策略会更高效。但实现这一点通常需要一个独立的世界模型组件，带来额外推理成本和训练复杂度。PaW 指出了一个更简洁的解法：on-policy rollout 的每一条 transition（action → next observation）天然就是世界模型的训练数据，只需在策略训练时同时施加 WM 预测损失即可。</p>
<p>　　具体实现上，PaW 引入三个稳定训练的组件：基于 action entropy 的 WM 数据选择（高熵 transition 含更丰富探索信息，优先选入）、noise-tolerant WM 损失（减少错误观测对梯度的干扰）、以及 reward-adaptive 损失平衡（根据当前 reward 信号动态调节 WM 损失权重）。在三个 agentic task benchmark 上，PaW 对多种模型规模和 RL 算法均取得一致性提升，验证了该框架的泛化性。</p>

<h4>Neel Nanda / Ari Holtzman 两团队再审 Subliminal Learning：现象更可能是 LoRA 微调的脆弱工件，此前研究高估其威胁</h4>

<p><b>论文 1（Neel Nanda 团队）：Steering Vector Distillation 解释 Subliminal Learning</b></p>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.00995">Subliminal Learning via Steering Vector Distillation</a></li>
  <li><b>信号源</b>：<b>Neel Nanda（资深合作）</b> · Camila Hartmann Blank（共一，Stanford）· Agam Bhatia（共一，Stanford）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>Nanda 团队把"学生模型微调教师输出时习得教师隐性行为"这件事归结为一个可解释的机制：教师的 system prompt 被近似为一个方向向量（steering vector），微调过程中 Adam 等自适应优化器能积累这些微小但方向一致的梯度信号，最终让学生模型学到同方向的 vector。<b>这意味着 subliminal learning 不是神秘的"行为传染"，而是自适应优化器（adaptive optimizer）放大弱信号这一已知机制的直接产物</b>——可被 steering vector 近似的 system prompt 会被学到，不能近似的则不会。这为 fine-tuning 安全研究提供了一个新的攻击和防御分析框架。</p>
</callout>

<p>　　模型蒸馏和微调的安全隐患研究近年快速增长，但"为什么学生模型会从语义无关数据中习得教师偏好"一直缺乏机制层面的解释。Nanda 团队在两个开源模型上系统追踪了激活空间的变化方向，发现 steered 数据上的激活梯度含有微小但一致的 steering 方向分量。</p>
<p>　　关键实验：(1) 将 steering vector 加入教师模型激活生成训练数据，让学生微调该数据，投影梯度后发现方向对齐；(2) 随机方向 vector 不会被学到，验证机制特异性；(3) 用非自适应优化器（SGD）时现象消失，因为 outlier 梯度噪声淹没了弱信号——Adam 是 subliminal learning 的必要条件，而非充分条件。</p>

<p><b>论文 2（Ari Holtzman 团队）：Subliminal Learning 是 LoRA 微调的脆弱副产物</b></p>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.00831">Subliminal Learning is a Fragile Artifact of LoRA Fine-Tuning</a></li>
  <li><b>信号源</b>：<b>Ari Holtzman（资深合作，UBC）</b> · Todd Nief（一作，导师 Ari Holtzman）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>Holtzman 团队的结论与 Nanda 团队形成有趣互补：subliminal learning 的行为传播强度与 LoRA rank 呈倒 U 型，full fine-tuning 下现象完全消失；同时高度依赖 fine-tuning 和 evaluation 时的上下文（如 system prompt）一致性，条件不一致时传播失效。<b>这意味着此前研究夸大了 subliminal learning 的稳健性和威胁程度——它更像一个 LoRA 超参数敏感的工件（artifact），而非语言模型间可靠的隐性行为传播机制。</b> 两篇论文合读，对 fine-tuning 安全评估方法的设计有直接影响：测试条件必须严格控制 LoRA 配置和上下文，否则会系统性高估或低估风险。</p>
</callout>

<p>　　Holtzman 团队通过消融实验系统测试了 LoRA rank 变化、full fine-tuning 对比、以及不同 system prompt 组合对 subliminal learning 传播强度的影响。核心发现：传播效果在中等 rank 时最强（倒 U 型曲线），rank 过大或过小都会减弱，full fine-tuning 时归零。</p>
<p>　　进一步，研究者将隐性行为激活定位到 fine-tuning 和 evaluation 阶段共享的特定 token 位置（如 default system prompt、chat template tokens）——只有这些 token 共现时现象才出现，说明 subliminal learning 的"传染"本质是特定上下文条件下的统计伪相关，而非模型间真正的知识或行为传递。</p>

<h4>Manling Li 团队提出 BAGEN：前沿 Agent 普遍缺预算感知，SFT+RL 省 28–64% token，揭示 agent 计费与信任缺口</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.00198">BAGEN: Budget-Aware Agent Evaluation Framework</a></li>
  <li><b>信号源</b>：<b>Manling Li（资深合作）</b> · Yuxiang Lin（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>BAGEN 揭示了一个被忽视的 agent 能力维度：任务性能强的 agent 不一定具备预算感知能力，两者相关系数仅 r=0.35。前沿模型普遍倾向在无望成功的任务上持续消耗资源而不主动告警。<b>这意味着 agentic 产品的计费模型和用户信任构建存在一个系统性缺口——用户无法预测 agent 会在什么时候"认赔"停止，而不是继续无效消耗。</b> SFT+RL 训练可在失败轨迹上节省 28–64% token，说明预算感知是可训练的能力，但区间精确校准（interval coverage 上限仅 47%）仍是未解难题，预算预测的可靠性离真正生产可用还有距离。</p>
</callout>

<p>　　当前 LLM agent 的评估体系几乎完全聚焦于任务完成率，对计算开销的合理性几乎没有系统性考量。在实际部署中，agent 在注定失败的任务上持续运行不仅浪费计算资源，还会带来不可预测的计费成本，这对企业级部署是一个直接的摩擦点。</p>
<p>　　BAGEN 将预算分为内部预算（agent 计算开销）和外部预算（agent 动作产生的资源消耗），将预算感知形式化为渐进式区间估计：agent 在执行每一步时需预测剩余预算的上下界，并在大概率无法完成时主动告警。评测采用 rollout-replay 协议，在四个环境和五个前沿 agent 上系统评测。SFT+RL 训练后，early stop 机制在失败轨迹上最多节省 64% token，但区间精确校准的 interval coverage 上限仅 47%，说明模型在"知道自己何时该停"和"准确估计还需多少资源"之间，前者可训练、后者仍困难。</p>

<h4>Yang Liu 团队提出 CPT 认知配对训练：提升 LLM 元认知，14B 模型数学 +2.2 分、弃权准确率 +5.2 分</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.00869">Cognitive Pairwise Training for LLM Metacognition</a></li>
  <li><b>信号源</b>：<b>Xiaolong Wang（合作）</b> · Yang Liu（通讯，导师 Frédo Durand）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>CPT 在标准监督微调（SFT）和强化学习（RLVR）之间插入一个认知中间对齐阶段：通过对同一问题的多条推理轨迹做配对比较（可信 vs 有缺陷），让模型内化推理质量的判别边界，而不是直接记忆"何时说不知道"的表面模式。<b>在 14B 规模下，CPT+RL 相比 SFT+RL 数学平均分提升 +2.2、弃权准确率（abstention-F1）提升 +5.2，且在 5 种模型规模和 3 个模型家族上保持一致性——这表明"推理质量的自我判断"是一个可系统训练的独立能力维度，而不仅仅是更大更强模型的副产品。</b></p>
</callout>

<p>　　让 LLM 知道自己"什么时候不确定"是对齐研究的核心难题之一。现有方法或通过 RLHF 隐式训练，或通过 SFT 直接教模型输出"我不知道"，但这两种方式都容易导致模型学到表面模式而非真正的推理自我评估能力。</p>
<p>　　CPT 把推理轨迹的质量判断转化为配对排序问题：收集模型在同一问题上的多条推理轨迹，构造可信推理与有缺陷推理的对比样本，用对比训练信号让模型在推理层面学会区分高低质量轨迹，形成内化的质量判别边界，再与 RLVR 结合进行最终训练。实验在 14B 模型上验证：CPT+RL 相比 SFT+RL 数学平均分提升 +2.2，弃权准确率提升 +5.2，且在多模型家族和多评估设置下保持一致性，说明 CPT 的对齐效果具有泛化性而非过拟合。</p>

  <p>这篇论文针对多 agent 协作维护共享知识库时的治理难题提出了一套结构化协议：commit-reveal 投票（先提交承诺再揭示，防止策略性模仿）+ Beta Reputation/EigenTrust 加权 + 渐进式制裁。在中等对抗场景下 precision 达 0.826，优于多数投票的 0.791；压力场景下差距扩大（0.807 vs 0.740），且性能退化速度约为多数投票的三分之一。<b>关键发现是 commit-reveal 机制单独贡献了 8.2–8.6pp 的 precision 提升，超过声誉加权和审议机制的组合效果</b>——这对正在构建 multi-agent 知识协作系统的团队有直接的设计参考价值。不过渐进式制裁模块在仿真中未被触发，实际对抗场景下的有效性仍待验证。</p>
</callout>

<p>　　多 agent 系统中，当多个 AI 协作维护共享知识库时，面临三类核心威胁：agent 无状态性导致威慑机制失效、模型同质化破坏群体智慧独立性、以及奉承行为（sycophancy）瓦解审议共识。现有方案大多采用简单多数投票，缺乏系统性的治理框架。</p>
<p>　　协议分三层：将知识 artifact 生命周期形式化为带标签的转移系统；将 Beta Reputation 与 EigenTrust（基于图传播放大高信誉节点影响力）结合进行声誉加权审议投票；针对无状态 agent 设计渐进式制裁机制，区分故障与对抗行为。通过含 100 个 agent、7 种行为原型的模拟（30 随机种子）在两种对抗场景下评估，commit-reveal 机制的防策略性模仿效果在所有组件中贡献最大。</p>

<h4>Yingchun Wang / Ling Feng 团队提出 SentGuard：句子级流式安全审查，两句内覆盖 90.5% 不安全案例、误报率仅 7.41%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02041">SentGuard: Sentence-Level Streaming Safety Guardrails for LLMs</a></li>
  <li><b>信号源</b>：<b>Xin Wang（共一）</b> · Jiaqi Yu（共一，导师 Ling Feng）· Yingchun Wang（通讯）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>现有 LLM 安全审查方案面临"等全文生成再判断（太慢）"和"逐 token 判断（语义不完整、误报高）"的两难。SentGuard 以句子为最小审查单元，配合轻量 waiting buffer 与目标 LLM 并行运行，在 LLM 生成第 N+1 句时同步审查第 N 句。<b>在 5 个安全 benchmark 上，SentGuard 两句话内检测出 90.5% 不安全案例，流式误报率（streaming false-positive rate）仅 7.41%，优于 token 级方法</b>——这套方案对生产级流式 LLM 服务的安全审查层设计有直接参考价值，同时附带的 StreamSafe benchmark 为该方向提供了标准化评测基础。</p>
</callout>

<p>　　LLM 流式输出已成为产品侧标配体验，但现有 guardrail 方案大多针对完整响应设计，无法在流式场景下提供实时干预能力。token 级方法虽然延迟低，但单个 token 的语义不完整导致高误报率，实用性受限。</p>
<p>　　SentGuard 由轻量 waiting buffer 和句子级 guardrail 模型组成，与目标 LLM 并行运行。Buffer 将流式 token 聚合为句子块，只有通过安全验证的句子块才释放给用户，引入小幅延迟换取审查窗口。模型使用 coarse-to-fine 训练目标：先粗粒度判断整体有害意图，再细粒度定位具体句子。若某句被判定为不安全，系统可在用户看到下一句之前中断输出。此外，随论文发布的 StreamSafe benchmark 含 8 类危害类别的逐句结构化标注，覆盖推理段与响应段的安全风险演变，为后续研究提供了标准化评测框架。</p>

<h4>浙江中医药大学 Hao Liu 提出 Adaptive PT-CBF：允许多机器人临时断连并限时重连的控制屏障函数框架</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.01526">Adaptive Prescribed-Time Control Barrier Functions for Multi-Robot Network Reconnection</a></li>
  <li><b>信号源</b>：Hao Liu（一作，Zhejiang Chinese Medical University）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>传统多机器人系统要求持续保持通信连通，代价高且限制灵活性。这篇工作用 adaptive prescribed-time 控制屏障函数（PT-CBF）允许机器人先断开连接，再在可调的规定时间内重连，并引入联合考虑任务执行效率和重连紧迫度的触发机制。<b>这意味着多机器人系统的"连通性约束"不再必须是实时硬约束，而可以转化为时间有界的软约束</b>——这对仓储调度、搜救等需要灵活队形变换的场景有实际价值，但目前仅在仿真中验证，真实机器人集群的通信延迟和噪声影响仍待考察。</p>
</callout>

<p>　　多机器人编队任务中，连通性保障通常以"所有机器人始终在通信范围内"为前提，这显著限制了队形变换的灵活性，在复杂环境中尤为突出。控制屏障函数（Control Barrier Function，CBF）是一种将安全约束编码为李雅普诺夫类条件的形式化方法，此前主要用于碰撞避免等实时安全约束。</p>
<p>　　Adaptive PT-CBF 在标准 CBF 基础上引入时变增益，使安全约束的收敛速率可随时间调节，保证机器人在任意可行的规定时间内回到通信范围。触发机制以任务状态和重连紧迫度为双输入，决定何时激活重连行为——任务执行效率高时推迟重连，重连紧迫度超阈值时强制触发。理论上证明了在规定有限时间内收敛，实验验证了任务效率提升且重连满足约束。</p>

<h3>世界模型 (World Model)</h3>

<h4>MIT Song Han 团队提出 LongLive-RAG：把自回归视频生成历史帧变为可检索记忆库，VBench-Long 最佳平均排名</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.02553">LongLive-RAG: Retrieval-Augmented Long Video Generation</a></li>
  <li><b>信号源</b>：<b>MIT Song Han（合作）</b></li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>自回归视频生成中，滑动窗口注意力（sliding-window attention）导致误差随序列长度不断累积——这是长视频生成质量随时间退化的根本原因。LongLive-RAG 把已生成的历史帧（latents）变成一个可检索的记忆库，每生成新块时主动检索相关历史帧而不只依赖最近窗口内容。<b>这意味着"长视频生成的质量上限受滑动窗口约束"这一认知需要被更新</b>——通过检索增强生成（RAG）这一语言模型领域的成熟工具，同样可以有效缓解视频生成中的误差累积问题，且可作为即插即用模块与多种自回归视频扩散 backbone 结合。在 VBench-Long 基准上，LongLive-RAG 在多个 backbone 上取得最佳平均排名，代码已开源。</p>
</callout>

<p>　　长视频生成中的关键技术挑战之一是"漂移（identity drift）"——随着生成长度增加，角色外观、场景背景等元素会逐渐偏离初始状态，根源在于滑动窗口只能看到最近帧，而最近帧可能已经发生了误差累积。传统解法是扩大窗口大小（计算成本高）或引入独立的长程注意力（架构复杂）。</p>
<p>　　LongLive-RAG 是一个即插即用的检索增强框架：每生成一个新视频块时，用 query embedding 从动态历史 latents 库中检索最相关的历史帧作为非局部上下文条件输入。为提升检索判别性，引入 Window Temporal Delta Loss（时序差分损失）——惩罚同窗口内相邻帧 embedding 的过度相似，强迫检索模块关注"发生了什么变化"而非"当前是什么状态"。在 VBench-Long 基准上跨多个 backbone 验证了有效性，检索开销相对于生成步骤较小。</p>

<h4>NYU Kyunghyun Cho / Cambridge Miles Cranmer 团队提出 Walrus 物理基础模型：≤3 样本微调后 Zero-Shot 跨越百年模拟-实验差距</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.01470">Bridging the Simulation-Experiment Gap in Rayleigh-Taylor Instability with a Foundation Model</a></li>
  <li><b>信号源</b>：<b>NYU Kyunghyun Cho</b> · <b>Cambridge Miles Cranmer</b> · Payel Mukhopadhyay（一作）</li>
</ul>

<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p>物理仿真到真实实验的迁移鸿沟，长期被认为是数值模型（DNS）边界条件理想化的固有缺陷，不同社区已争论近百年。Walrus 这项工作把问题重新框定：<b>用预训练于连续介质动力学的基础模型，只需 ≤3 个 DNS 样本微调，就能 zero-shot 预测真实滑动隔板实验室数据，混合增长率 α 自然落入观测区间（0.06–0.07）而非停留在 DNS 理想值（0.02）</b>。更重要的是，模型同时提供了数据驱动证据：初始条件差异是导致差距的主因，而非数值方案本身。如果这套少样本物理迁移框架能推广到更多流体力学场景，科学仿真与实验之间的"验证黑箱"可能会被系统性地打开。</p>
</callout>

<p>　　Rayleigh-Taylor 不稳定性（RTI）是重流体在轻流体之上时界面失稳并形成混合层的经典问题，广泛出现在惯性约束聚变、超新星爆炸和海洋混合等场景中。过去几十年，直接数值模拟（DNS，Direct Numerical Simulation）得到的混合增长率 α 约为 0.02，而实验室测量值约为 0.06–0.07，两者之间的差距是该领域长期未解的"百年悬案"，传统解释包括初始扰动差异、三维效应和可压缩性等多种假说。</p>
<p>　　Walrus 以连续介质动力学仿真数据预训练，编码了流体物理的底层结构。研究者用不超过 3 个 RTI DNS 样本做少样本微调，随后直接将模型应用于真实滑动隔板实验室数据（完全 zero-shot，无实验标签）。预测的 α 自然落入 0.06–0.07 观测区间，同时模型 zero-shot 泛化至训练中从未出现的稳定分层浮力区，正确预测混合层增长放缓。这为"初始条件差异是主因"这一假说提供了独立数据驱动佐证。</p>

<p><em>* 注：上述结果来自单篇预印本，混合增长率数值和泛化结论尚待同行评审及独立实验室的交叉验证。</em></p>
