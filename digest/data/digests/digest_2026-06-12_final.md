<h3>Bezos 旗下 Prometheus 完成 120 亿美元融资，估值 410 亿——Physical AI 赛道迎来史上最大单笔押注</h3>
<ul>
  <li><b>信号源</b>：<b>TechCrunch</b>（<a href="https://techcrunch.com/2026/06/11/jeff-bezoss-prometheus-raises-12b-to-build-an-artificial-general-engineer-for-the-physical-world/">TechCrunch 报道</a>）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p text-indent="1">这笔融资的意义不止于规模——<b>120 亿美元让 Prometheus 以单笔融资额直接对齐 OpenAI 量级，标志着 Physical AI 赛道从"早期探索"进入"大资本卡位"阶段</b>。Bezos 选择将重型工程与药物设计并列为核心场景，指向一个有别于人形机器人路线的判断：物理世界最有价值的 AI 替代目标不是体力劳动者，而是专业工程师与科学家——即高认知 × 高物理耦合的复合型工作。这条路线与 Google DeepMind 的 AlphaFold / AlphaCode 路线有交叉，但 Prometheus 明确向"通用工程师"而非"专项工具"定位，意味着它要整合跨领域物理建模、仿真与设计闭环。当前阶段更多是战略卡位，后续需关注：核心团队背景（硬件 / 仿真 / 化学各方向的比例）、与亚马逊供应链生态的协同关系，以及是否会走垂直领域先深后广的路径。</p>
</callout>
<p text-indent="1"><b>Prometheus</b> 由 <b>Jeff Bezos</b> 创立，定位为面向物理世界的"人工通用工程师"（Artificial General Engineer），目标场景覆盖重型工程设计与药物设计。本轮融资金额 <b>120 亿美元</b>，融资后估值达 <b>410 亿美元</b>。</p>
<p text-indent="1">融资规模在 Physical AI 初创公司中前所未见，超越此前 Physical Intelligence（pi）、Figure AI 等同赛道公司的单轮记录。这一量级的融资通常意味着投资方对长周期、重研发路径的认可，也反映了当前 AI 资本市场对"将通用 AI 能力嵌入物理设计闭环"方向的高度预期。</p>
<p text-indent="1">从产业传导角度看，本次融资将加剧对物理仿真、多物理场建模、高保真仿真器（如 Isaac Sim、Mujoco 生态）等基础设施层的需求争夺，并可能推动药物设计 AI（如分子动力学仿真与生成）与重工程 AI 的技术栈向更统一的"物理世界基础模型"方向收敛。</p>
<p text-indent="1"><em>* 注：融资金额与估值来自 TechCrunch 报道，非 Prometheus 官方披露，以官方公告为准。</em></p>

<h3>Anthropic 与 TCS 战略合作，专属业务单元落地——AI 模型厂商借道全球 IT 服务商攻企业市场</h3>
<ul>
  <li><b>信号源</b>：<b>Anthropic / TCS</b>（<a href="https://techcrunch.com/2026/06/11/anthropic-taps-tcs-to-scale-its-enterprise-ai-deployments/">TechCrunch 报道</a>）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p text-indent="1">这笔合作揭示了当前 AI 企业市场的一条关键扩张路径：<b>模型厂商自身不具备大规模企业交付能力（实施、集成、合规、本地化），而 TCS 等顶级 IT 服务商恰好拥有数千家全球企业的深度关系网络</b>。Anthropic 为 TCS 成立"专属业务单元"背书，本质上是将企业落地外包给渠道，换取更快的客户覆盖速度。这与微软将 OpenAI 能力内化到 Azure、Copilot 的路线不同——Anthropic 选择通过独立的 SI（系统集成商）渠道保持模型中立性，同时借助 TCS 进入金融、制造、医疗等传统行业的高壁垒客户。风险在于：TCS 同时服务多家 AI 厂商，专属单元的"排他性"深度值得关注；而 Anthropic 品牌在终端企业客户中的感知度，将在很大程度上取决于 TCS 的交付质量。</p>
</callout>
<p text-indent="1"><b>Anthropic</b> 与全球最大 IT 服务商之一 <b>TCS（塔塔咨询服务）</b>达成战略合作协议。TCS 将为此成立专属业务部门，专门负责向其企业客户群体部署 Anthropic 旗下 Claude 系列模型。</p>
<p text-indent="1">TCS 在全球拥有逾 60 万名员工和覆盖金融、制造、零售、医疗等行业的大型企业客户群，此次合作实质上为 Anthropic 打通了一条不依赖自建销售团队的企业交付渠道。此前 Anthropic 已通过 Amazon Bedrock、Google Cloud Vertex AI 等云平台提供 API 访问，TCS 合作则代表其向端到端企业实施层渗透的进一步布局。</p>

<h3>AI 自己做 AI 研究拿出首批硬成果——Recursive 系统在三项基准上超越人类与数百个 AI agent 数月积累的最优方案</h3>
<ul>
  <li><b>信号源</b>：<b>Recursive</b> 官方（<a href="https://www.recursive.com/articles/first-steps-toward-automated-ai-research">技术博客</a>）· <b>Jeff Clune</b>（<a href="https://twitter.com/jeffclune/status/2065063979765166123">X 推文</a>）· Techmeme 多源证实</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p text-indent="1">这条信号的分量在于"超越的对象是谁"：不是某个研究者的单次结果，而是 Karpathy 公开基准之上、几十位人类研究者带着数百个 AI agent 协作数月沉淀的社区最优解。<b>递归自我改进（RSI）——AI 改进 AI 自身——由此从理论叙事进入了可检验的实证阶段</b>，与本周"OpenAI 若 RSI 突破将推迟 IPO"的报道互为印证：头部机构在为同一件事做准备。对投研更直接的含义是成本：该系统瞄准的三个杠杆——更好的训练算法、更快的训练、更高的硬件利用率——恰好是 AI 产业最大成本项的全部构成，自动研究系统每一次发现都可直接折算为算力开支的节约。<b>若这类系统的搜索效率持续超过人类社区，AI 研发本身将成为新的"算力杠杆"赛道</b>；制约项是基准的窄域性与 reward hack 风险，需观察向大模型尺度的泛化。</p>
</callout>
<p text-indent="1"><b>成果要点：</b><b>Recursive</b> 于 6 月 11 日发布其自动化 AI 研究系统的早期结果，三项基准全部刷新最优：固定算力预算的小模型训练（NanoChat Autoresearch）验证损失 0.9372 → <b>0.9109 BPB</b>（等效 <b>1.3 倍</b>提速），且超越的是 autoresearch@home 社区（数十位人类 + 数百个 agent）的集体最优；NanoGPT Speedrun 训练时间 79.7 秒 → <b>77.5 秒</b>；GPU kernel 优化基准 SOL-ExecBench 上 235 个 kernel 的平均得分 0.699 → <b>0.754</b>，向硬件理论极限的差距缩小 <b>18%</b>。相关产物已<a href="https://github.com/recursive-org/first-steps-toward-automated-ai-research">开源</a>供外部检验。</p>
<p text-indent="1"><b>系统机制：</b>系统把研究循环完整自动化——提出想法、写代码实现、跑实验、验证结果、再根据所学选择下一个实验；多条研究线程长程并行、保留跨实验上下文、合并有效分支，并对每个"性能提升"做防作弊（reward hack）与方差校验后才认作真实进展。</p>
<p text-indent="1"><b>技术发现样例：</b>最大单项增益来自一种更丰富的短上下文记忆机制——把二元/三元词组哈希进固定尺寸查找表、经可学习门控混入注意力的 value 通路，思路与 DeepSeek Engram 的哈希表稀疏化方向相通，但作者团队表示不知道此精确变体的先例——<b>即这一改进由系统自己"研究"出来，而非复述人类已知方案</b>。</p>
<p text-indent="1"><b>路线脉络（related work）：</b>这条"AI 自动做研究"路线的节点：<b>Sakana AI</b> 的 AI Scientist（2024，自动生成科研论文，验证流程可自动化）→ <b>Google DeepMind</b> 的 AlphaEvolve（2025，进化式编码 agent 自动发现算法、曾为 Google 数据中心调度回收约 0.7% 算力——验证发现可落地成钱）→ <b>Karpathy</b> 的 autoresearch 与社区化的 autoresearch@home（2026，提供人机协作的公开基准与集体基线）→ <b>Recursive</b> 本次结果首次展示自动系统在公开社区基准上系统性超越人类 + agent 的集体积累。</p>
<p text-indent="1"><em>* 注：全部数字来自 Recursive 官方博客自述（对照基线为 autoresearch@home 公开数据，且官方称已剔除此前最优解中的轻微 reward hack 后重测）；开源产物可复核，但尚无独立第三方复现；基准均为小模型窄域场景，结论向大模型尺度的泛化待验证。</em></p>

<h4>Coinbase 上线 MCP 工具包：AI agent 可自主下单交易并通过 x402 协议付费访问数据</h4>
<ul>
  <li><b>信号源</b>：<b>Coinbase</b>（<a href="https://techcrunch.com/2026/06/11/coinbase-debuts-mcp-for-agent-trading/">TechCrunch 报道</a>）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p text-indent="1">MCP（Model Context Protocol，即让 AI 模型与外部工具/数据源通信的标准协议）与支付协议的结合，是 AI agent 从"信息助手"向"自主执行者"演进的关键一步。<b>Coinbase 的 x402 支付层让 agent 可以自主决定为哪些数据付费，标志着"AI agent 有自己的预算与支付能力"这一方向从概念落地为产品</b>。风险层面，自主交易 agent 的权限边界与安全审计是重要问题。</p>
</callout>
<p text-indent="1"><b>Coinbase</b> 推出面向开发者的 MCP 工具包，允许 AI agent 在 Coinbase 平台上自主执行加密货币交易操作。配合 <b>x402 协议</b>（一种允许 HTTP 请求附带微支付的协议），agent 还可以按需付费访问高级数据和 API，无需人工授权每笔支付。这是主流 Web3 平台首次将 MCP 标准与支付能力直接结合，为 agent 自主经济活动提供了基础设施雏形。

<p>{anchor:card_3}</p>
<h4>MLJailDe：11 语言越狱检测框架——跨语言攻击的防御盲区被大幅压缩至 F1 97%</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.11202">MLJailDe: Multilingual Jailbreak Detection via Cross-Lingual Representation Alignment</a>（Shuyu Jiang（一作）· <b>Yi Zhang</b>（合作））</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p text-indent="1">LLM 安全对齐长期以英语为主要战场，而攻击者很早就发现"换一门小语种"往往能绕过防护。<b>MLJailDe 证明可以用表示空间对齐的方式在不同语言之间共享"威胁识别"能力，从已知语言泛化到未见语言的 F1 仅下降 1.4 个百分点</b>，在实用性上有直接部署价值。</p>
</callout>
<p text-indent="1">越狱（jailbreak）指通过精心设计的提示词绕过 LLM 的安全限制，让模型输出有害内容。<b>MLJailDe</b> 框架通过三步解决跨语言越狱检测问题：① 利用回译（back-translation）构造覆盖 11 种语言、语义一致的训练数据集（含 2,232 条正常样本与 1,239 条越狱样本）；② 用相对距离约束（relative-distance constraint）将不同语言但意图相同的恶意 prompt 在模型内部表示空间中"拉近"；③ 引入类别不均衡感知损失函数提升决策边界质量。</p>
<p text-indent="1"><b>核心结果</b>：已见语言 F1 达 <b>98.5%</b>，未见语言零样本泛化 F1 达 <b>97.1%</b>，超越当前最强基线，且无需为每种新语言单独标注数据。</p>

<h4>Bebop：用 TV loss 修复 RL 训练中"投机猜词"失效问题，Qwen 系列端到端加速 1.8×</h4>
<ul>
  <li><b>论文</b>：<a href="https://arxiv.org/abs/2606.12370">Bebop: Accelerating RL Rollouts via MTP and TV Loss</a>（Yucheng Li（共一）· Huiqiang Jiang（共一）· <b>Yi Zhang</b>（合作）· <b>Junyang Lin</b>（合作）· Jingren Zhou（通讯）· Alibaba Group）</li>
</ul>
<callout emoji="💡" background-color="light-orange" border-color="orange">
  <p><b>Insights</b></p>
  <p text-indent="1">投机解码（speculative decoding）是目前加速大模型推理的主流工程手段——用小模型"猜"后续词，大模型批量验证，猜对了就直接用，猜错了再重来。在正常推理时这很有效，<b>但 RL 训练过程中模型的"不确定性"（熵值）会剧烈波动，导致猜词命中率大幅下降，加速效果几乎消失</b>。Bebop 揭示了熵值与命中率之间的线性负相关规律，并设计 TV loss 直接以"提高命中率"为优化目标——相当于给投机猜词机制打了一个专门对抗 RL 训练扰动的补丁。这一方向对所有基于 RL 后训练的模型（如数学推理、代码生成、agent 训练）都有直接工程价值。</p>
</callout>
<p text-indent="1">在大语言模型的强化学习（RL）训练中，rollout 阶段（即让模型大量生成文本以采样训练数据）是主要速度瓶颈之一。<b>Bebop</b> 的核心发现：多 token 预测（Multi-Token Prediction，即一次预测多个后续词以加速生成）的"命中率"与 RL 阶段模型熵值呈负线性相关——熵越高，猜词越不准。</p>
<p text-indent="1">解决方案分两层：① 用概率性拒绝采样（probabilistic rejection sampling）替代贪心草稿生成，缓解熵扰动；② 提出端到端 <b>TV loss（Total Variation loss，即直接优化词分布差异的损失函数）</b>替代传统 cross-entropy/KL 目标，使模型直接以提高多步猜词命中率为优化目标。配合 pre-RL 阶段的多 token 预测训练，避免了昂贵的在线更新开销。</p>
<p text-indent="1"><b>核心结果</b>：多 token 预测命中率最高达 <b>95%</b>，推理吞吐额外提升 <b>25%</b>；在 <b>Qwen3.5/3.6/3.7</b> 异步 RL 训练中端到端加速最高达 <b>1.8×</b>，覆盖数学推理、代码生成和 agent 任务。</p>
