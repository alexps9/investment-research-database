## 1. Goal 三联（决定后续所有 prompt 设计）

### 1.1 TL;DR

| 维度 | 定义 |
|---|---|
| **读者** | 20+ 人，跨赛道 PE/VC 投研，本科 CS/AI 基础 |
| **何时读** | 12:00 推送后，3-5 分钟扫读 |
| **为什么读** | 决定"今天这份日报值不值得点进去看头条 + 跟同事 sync 时能提到的概览" |
| **状态变化** | 30 秒后能复述"今天日报涵盖哪些赛道 + 最重要事件是什么 + 为什么重要" |
| **三联 goal** | ① **看懂** — 不涉及技术细节也能秒懂 / ② **觉得重要** — 含潜在影响判断的钩子 / ③ **吸引点开** — 留信息差让读者想看头条 |
| **失败标准** | ① 含未解释的技术黑话（让跨赛道读者卡住）/ ② 只复述事件，没有"为什么重要"暗示 / ③ 5 条都是同一机构周边（缺多样性）|

### 1.2 论文 Insights（200-280 字长卡）

| 维度 | 定义 |
|---|---|
| **读者** | 关注该赛道的 PM / Analyst，本科 CS/AI 基础 |
| **何时读** | TL;DR 之后点进去深读，每条 1-2 分钟 |
| **为什么读** | 修正自己对该赛道的判断 / 跟 partner 讨论时有谈资 / 触发 Potential To-dos |
| **状态变化** | 读完能用一句话说："这条信号让我对 X 的判断变了：原来 Y，现在 Z" |
| **5 层次** | 命中 ≥ 1 层即合格：机制 why / 格局 shift / 认知 reframe / 节奏 timing / 标的 actionable |
| **失败标准** | ① 读完只能复述事件，不能复述判断变化 / ② 80% 字数在 why 技术细节 / ③ 投研意涵是空标签（"对 AI agent 有利"）/ ④ 与摘要段重复 |

### 1.3 行业应用 Insights（60-100 字短卡）

| 维度 | 定义 |
|---|---|
| **读者** | 同论文 Insights，特别是关注产品 / 估值动态的人 |
| **何时读** | 1 分钟内扫完 |
| **为什么读** | 快速识别"这件事对赛道格局意味什么 + 谁该警觉" |
| **优先级** | 论文 Insights 重，行业可略浅（命中 1 层即可）|
| **失败标准** | ① 复述推文内容 / ② 宏大叙事（"AGI 又近一步"）/ ③ 没指向具体标的类型 |

### 1.4 Potential To-dos

| 维度 | 定义 |
|---|---|
| **本质** | **思维提示，打开读者某个注意窗口**（不是强制行动建议）|
| **读者** | 该赛道 owner |
| **何时执行** | 当天下午 / 第二天 |
| **状态变化** | 读完知道"接下来值得关注的方向 / 信号" |
| **失败标准** | ① "继续观察 / 拭目以待" / ② 抽象动作（"加强对 X 赛道关注"）/ ③ 跟 Insights 重复无增量 |

## 2. 共同设计原则

### 2.1 投研意涵粒度（混合策略）

| 场景 | 写法 |
|---|---|
| 原文明确提到公司 / 公开报道 | 点名具体公司（"NVIDIA / Dell / SMCI"）|
| 原文未明确提到 / 推测 | 退到类标的（"inference-as-a-service 创业公司"）|

### 2.2 批判性观点（选择性）

| 场景 | 是否允许批判 |
|---|---|
| 单个信号 | ❌ 不下 "overhyped / narrative 走不远" 判断 |
| 跨信号交叉验证（同赛道 ≥ 3 信号佐证）| ✅ 允许扶批 |

### 2.3 共识锚点

软要求 · 读者已在赛道中可忽略 · 开头风格多样化（每天 7 篇头条中至多 1-2 篇用"业界主流认为"句式）

## 3. Prompt 片段（v6.8 草稿，可直接替换 v6.7 对应段）

### 3.1 TL;DR Prompt 段

```markdown
## TL;DR 写法（v6.8）

### Goal 三联（每条 TL;DR 完成后必须自查）

1. **看懂**：跨赛道读者（消费 / 制造 / 医药背景 PE）读完能秒懂这件事是什么——不涉及技术细节也行
2. **觉得重要**：含潜在影响判断的钩子——告诉读者"这件事为什么值得点开"，但不展开判断
3. **吸引点开**：留信息差——读者读完想看头条详情

### 硬约束

- 总条数 3-5 条，每条 `<li>` 独立
- **赛道标签前置**：`【基础模型】` / `【多模态智能】` / `【世界模型】` / `【AI infra】` / `【ai4s】` / `【商业进展】` / `【行业应用】`
- 末尾加锚点占位：`（<a href="https://anchor.placeholder/headline_N">↗</a>）`
- 缩写规则：`中文释义（英文缩写）` 格式 — **中文在前**

### 软约束

- 数字：原文有数字 → 加粗（`<b>+30%</b>`），原文没数字 → 不勉强
- 机构名：必加粗
- 赛道分布：5 条以 ≥ 3 个 track 中心，但如果某赛道信号集中可 4-5 条同赛道（优先体现 trend）

### 反向自查清单（写完每条后必查，命中任一项 → 重写）

❌ 含未解释的技术黑话（SPL / DPO / GRPO / NAS / GEMM / KL散度 等）
❌ 只复述事件，没暗示"为什么重要"
❌ 5 条都是同一机构周边（如 4 条都是 OpenAI / Anthropic 周边）
❌ 钩子前置但读完不知道发生了什么
❌ 跨赛道读者读完无法用一句话复述

### 反例 vs 正例

❌ "【AI infra】Microsoft 提出 PEEK 方法，用 1024 token persistent context map 提升 SWE-Bench Verified 34%"
   问题：persistent context map / SWE-Bench Verified 跨赛道读者卡住；只有事件，无"为什么重要"暗示

✅ "【AI infra】**Microsoft** 提出新方法，用持久化"上下文地图"让大模型不再每次重新读完全部代码——编程任务准确率提 **34%**、成本最高降 **5.8x**。如果稳定可复现，会重新定义 AI 编程产品的成本结构。"
   修复：① 给"上下文地图"加比喻 + 隐去"persistent context map" 这种黑话；
        ② 末尾加一句潜在影响判断（"重新定义...成本结构"）；
        ③ 数字保留加粗
```

### 3.2 论文 Insights — **A 版（why 段高深度）**

```markdown
## 论文 Insights 写法（v6.8 · A 版）

### 整体框架

整体长度 200-280 字。5 要素按自然顺序串成 2-3 段：

1. 共识锚点（**软要求**，1-2 句以内）
2. 研究动作 + 关键结果（数字带参照系）
3. **why 段（高深度，35-50 字）— 读者要能复述方法**
   - ✅ "凭 KV cache 跨 72 卡共享 + Vera CPU stack 整合 + W4A4 量化贯穿训练-推理 三件事"
   - ✅ "用 PagedAttention 把 KV cache 切成固定 block + block table 间接寻址 + 跨请求 prefix sharing"
4. 认知改变（独立 `<p>` 块）："原本被认为 X，现在变成了 Y"
5. **投研意涵（混合粒度，占 ≥ 1/2 篇幅）**：
   - 事实明确 → 点名公司（"对 **Dell / HPE / SMCI** 这类 OEM 整合商不利"）
   - 事实不明确 → 退类标的（"对 inference-as-a-service 创业公司护城河需重估"）
6. Potential To-dos（思维提示，1-2 条 `<ul><li>`）：打开注意窗口（"等待某 paper 复现"、"关注某方向团队"），**不写强制行动**

### 加粗规则

- 数字 / 公司 / 实验室名必加粗
- 核心判断词选择性加粗（"贬值"、"利好"、"窗口期"、"洗牌"）

### 反向自查清单（写完后必查，命中任一项 → 重写）

❌ 读完只能复述事件，不能复述判断变化
❌ 80% 字数在 why 技术细节（why 应 35-50 字内）
❌ 投研意涵是空标签（"对 AI agent 有利"、"AGI 又近一步"）
❌ 与摘要段重复
❌ 单信号下了 "overhyped / narrative 走不远" 这种价值判断（除非跨信号交叉验证）
❌ Potential To-dos 写了 "继续观察 / 拭目以待"
```

### 3.3 论文 Insights — **B 版（why 段中深度）**

```markdown
## 论文 Insights 写法（v6.8 · B 版）

### 整体框架（与 A 版唯一差异：why 段深度）

整体长度 200-280 字。

3. **why 段（中深度，15-25 字）— 读者只需知道方法属于哪一类技术**
   - ✅ "凭 KV cache 优化（同 vLLM 思路）"
   - ✅ "凭 stack 整合（GPU + 自研 CPU + 量化训练）"
   - ✅ "凭 long context 路线上的一个新点（不是从头训练）"
   - ❌ 不要写到"复述方法"级别

其他 5 个要素（共识锚点 / 研究动作+结果 / 认知改变 / 投研意涵 / Potential To-dos）与 A 版完全一致。

### why 段对比（同一论文）

| A 版 | B 版 |
|---|---|
| "1/10 inference 成本来自 KV cache 跨 72 卡共享 + Vera CPU stack 整合 + W4A4 量化贯穿训练-推理 三件事的协同" | "1/10 inference 成本主要凭 stack 整合（NVL72 + Vera CPU + 量化路径）" |
| 50 字 | 18 字 |

剩余篇幅释放给 shift / actionable 两层。
```

### 3.4 行业应用 Insights（60-100 字短锋利）

```markdown
## 行业应用 Insights 写法（v6.8）

### 结构（按顺序，不必标段落标签）

1. 这件事在做什么 + 1 句 why（含具体公司 + 动作 + 卡位 / 机制解释）
2. 为什么是信号（必须指向赛道格局变化 / 卡点变动 / 重要先例）
3. 谁该关心（点名标的类型 / 已投赛道 / 竞争对手）

### 长度 60-100 字（论文版的 1/3）

### 反向自查清单

❌ 复述推文 / 公告内容（"DeepSeek 宣布降价 75%"）
❌ 宏大叙事（"AGI 又近一步"）
❌ 没指向具体标的类型
❌ "还需观察"废话钩子
```

### 3.5 Potential To-dos

```markdown
## Potential To-dos 写法（v6.8）

### 本质

**思维提示**，打开读者某个注意窗口 — 不是强制行动建议。

### 三种正确形式

1. **关注窗口**："看 Anthropic 后续 release 是否跟进这种安全测试" / "关注 X 团队下篇 paper"
2. **复现 trigger**："等 3-6 个月看 benchmark 是否在第三方复现"
3. **判断 trigger**："如果 Y 复现，inference-as-a-service 估值前提需要调整"

### 反向自查清单

❌ "继续观察"
❌ "拭目以待"
❌ "若可复现就 X，否则 Y"（这是双假设废话）
❌ 抽象动作（"加强对 X 赛道关注"）
❌ 跟 Insights 内容重复
❌ 强制行动建议（"约 X 公司"、"重评估 Y 标的"）— **本版本不要**
```

## 4. A/B 实验设计

### 4.1 实验对象

**今日（5.25）v2 日报中的 13 篇前沿研究论文** + **5 篇头条论文**（合 17 篇 Insights 候选）：

实际跑：抽 5 篇代表性论文（不同赛道、不同 novelty）

候选样本（按 5.25 v2 顺序）：
1. `2605.22819` Saining Xie · Cambrian-P（多模态 + 头条）
2. `2605.21572` Ziwei Liu · PhysX-Omni（世界模型 + 头条）
3. `2605.22817` Omar Khattab · VPO（基础模型 + 头条）
4. `2605.22791` Yejin Choi · Gated DeltaNet-2（基础模型 + 普通）
5. `2605.22287` Zhiyuan Liu · SciCore-Mol（ai4s + 普通）

### 4.2 跑法

对每篇论文：
- **A 版 prompt** 跑 Opus 4.6 → 输出 200-280 字 Insights（why 段 35-50 字深）
- **B 版 prompt** 跑 Opus 4.6 → 输出 200-280 字 Insights（why 段 15-25 字浅）
- 输入：相同 paper signal + RM 数据

5 篇 × 2 版 = 10 个 Insights → 飞书 doc 并排展示

### 4.3 评估维度（人工 review）

每条 Insights 评分 1-5：

| 维度 | 评分标准 |
|---|---|
| **读完判断变化清晰度** | 5 = 一句话能复述判断 / 1 = 只能复述事件 |
| **why 段帮助度** | 5 = 让我能跟创业者讨论 / 3 = 让我大致理解类别 / 1 = 占字数无收益 |
| **shift / actionable 篇幅** | 5 = 占 ≥ 2/3 / 1 = 占 ≤ 1/3 |
| **投研意涵粒度** | 5 = 点名 + 类标的混合得当 / 1 = 全部空标签 |
| **整体偏好** | 5 = "周一会上想 share 这条" / 1 = "跳过算了" |

### 4.4 决策规则

- A 版总分 > B 版 + 2 → 采纳 A 版（高深度路线）
- B 版 ≥ A 版 → 采纳 B 版（中深度路线）
- 二者各有所长 → 拆分：头条用 A 版 / 普通论文卡片用 B 版
