## 3. 目标与动机层

### 阅读漏斗 3 层（**用户原话**）

| 层级 | 作用 | 期待时长 |
|---|---|---|
| **TLDR** | **决定要不要继续看** | 30s-1min |
| **头条** | **最有价值的信息提取** | 2-3 条 × 1-2 min |
| **Insights**（详） | **详细沉淀了解** | 弹性 |

### 动机

| 动机 | 具体 |
|---|---|
| **跟上 frontier** | X / 公众号 / Newsletter 已看过的事件，希望日报加投研视角 + 赛道认知调整 |
| **调整赛道认知** | "原来 X 必须 Y，现在 X 用 Z 也行" 型 reframe（谢赛宁式）|
| **创业公司洞察** | 某类一级市场创业公司 涌出 / 被压缩 — 类标的判断，**不点名现有 portfolio** |
| **跟同事谈资** | 看完能跟 partner / 团队 sync 时引用"今天看到一个 X" |

### 不是动机（容易误解）

❌ 二级市场 deal sourcing
❌ Portfolio company review（不点名现有 portfolio）
❌ 学术研究 / 复现论文方法

## 4. 痛点与障碍层

### 痛点 priority（用户原话）

> "优先级：TLDR > 头条 insights + 标题等 > 其他的 insights"

| Priority | 痛点 | 具体表现 |
|---|---|---|
| **P1 紧急** | TLDR 不锐利 | 看完不知道哪条最值得深读 · 没让人觉得"必须看头条" |
| **P2 关键** | 头条 + 头条 Insights + 标题 | 多是事件复述（缺 reframe）· 头条标题不够 hook · Insights 太学术 |
| **P3 次要** | 普通论文卡片 Insights | 即使深度差也能忍 · 用户不会每篇都精读 |

### 障碍

⚠️ **当前障碍 1 · "易懂"标准被低估**

> **用户原话**："可学习位姿好像更听不懂吧"

我之前以为 "易懂" = 给英文术语加中文翻译。**真正的标准**是：
- **TLDR + 头条标题 完全去英文**（不出现 VSI-Bench / Cambrian-P / token / KV cache）
- **直说认知路径调整**（"不必依赖 X，靠 Y 同样有效"型 framing）
- **不解释技术细节**（不绕弯做比喻 / 翻译 / 场景化）

⚠️ **当前障碍 2 · 头条 Insights 投研意涵悬浮**

读完只能复述事件，不能复述判断变化。

⚠️ **当前障碍 3 · 不知道"对我有什么关系"**

应该明确指向：**某类一级市场创业公司 涌出 / 被压缩**（类标的层），**不点名现有 portfolio**

## 6. 信息渠道与决策路径

### 团队已有的信息渠道（**用户确认 · 最活跃的 3 个**）

1. **X / Twitter 原生刷**
2. **中文 tech 公众号 / 知乎**（机器之心 / 量子位 / RTAI）
3. **英文 Newsletter**（The Information / Stratechery / Import AI）

### 日报对比这些渠道的差异化价值（**用户原话**）

| 渠道 | 日报相对它的差异化 |
|---|---|
| X 原生 | **抓 X 的 frontier 信号 + 加投研视角 + 赛道认知调整** |
| 中文公众号 | **更 frontier**（公众号延迟 2-7 天）+ **更投研而非科普** |
| 英文 Newsletter | **中文化 + 一级市场视角**（Stratechery 偏商业，Import AI 偏技术，没有投研维度）|

### 关键差异化（用户原话扩展）

> "日报能针对我们的用户画像，针对论文中较为枯燥晦涩的话，使用更加简单易懂的语言让我听懂在讲什么问题，有哪些重要的进展，并告诉我对我有什么关系等"

3 个具体价值点：
1. **听懂在讲什么问题** — 完全去英文术语 · plain 中文路径词
2. **有哪些重要进展** — 头条最有价值信息提取
3. **对我有什么关系** — 某类一级市场创业公司 涌出 / 被压缩

## 9. 由画像导出的 v6.8 prompt 必改项（vs 现状 v6.7）

| 改动项 | 原因（追溯画像）|
|---|---|
| **TL;DR 完全去英文术语** | "易懂"标准被低估 · 用户原话否定 "可学习位姿" |
| **TL;DR 必含"路径/前提"型表达** | 谢赛宁式 reframe 是核心价值 |
| **头条标题**也走 plain 中文 · 同 TLDR 标准 | 头条标题与 TLDR 同属漏斗顶端 |
| **头条 Insights** 保留英文（深度阅读场景，画像知识储备能处理）| 漏斗第 2 层，可保留专业术语 |
| **"对我有什么关系"必明确指向**：某类一级市场创业公司 | 不点名 portfolio · 类标的判断 |
| **Insights priority** 头条 > 普通卡片 · LLM 应在头条 Insights 上花更多 token | priority 排序 |
| **不强制每日 N 条 reframe** | 自然抽取 · 有就有 |
