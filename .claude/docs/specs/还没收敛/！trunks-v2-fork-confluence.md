# Trunks V2：分叉与合流视图

## Problem

当前 trunks 视图的局限：
- 跨 lane 的 S 曲线看不出关系类型（继承/竞争/借鉴长一样）
- 固定 4 条平行线太自上而下，跨 lane 论文没有好位置
- 新论文进来不知道挂哪里

需要一个视图让用户一眼看出：一条路线在哪里分裂成两派，在哪里又合回来。

## 目标

展示技术路径的**分叉与合流**：
1. 一条主干在某时间点分成两条分支（= 竞争/路线之争）
2. 两条不同路径在某个节点汇合（= 融合/新范式诞生）
3. 跨路径的借鉴关系（= idea 传播）

## 视觉设计

### 核心隐喻：河流分叉合流 + 贝塞尔曲线

参考 `prototypes/topic-view-exploration.html` #1 紧凑弧线风格。

### 布局
- 4 条水平主干（RL/Imagination、Diffusion Video、JEPA/Latent、Policy/VLA）
- Lane 间距足够大，给跨 lane 弧线留空间
- X 轴 = 时间
- 只挂"路标"节点（非所有论文）

### 节点筛选标准（什么论文该挂上去）
- 路径起点（开创新方法论）
- 路径转折点（解决前一代瓶颈，后续工作都跟它）
- 分叉点（从同一祖先衍生出互相竞争的方向）
- 合流点（inherits ≥ 2 条不同路径）
- 量化兜底：impact_score ≥ 50

### 三种关系的视觉编码
- **inherits**：主干延续线 + 分叉弧（实线，lane 颜色）。结构本身表达继承。
- **competes**：同一分叉点出发的两条分支。分叉 = 竞争，不需要额外线。
- **borrows**：跨 lane 虚线弧 + 文字标注（如 "borrows diffusion training"）

### 跨 Lane 论文处理
- 合流节点用菱形/星标
- 放在 inherits 权重更大的那条 lane 上
- 从另一条 lane 画弧线汇入菱形，标注来源

### 分叉的视觉表达
- 从分叉点以贝塞尔曲线向上或向下剥离
- 分支线宽可随时间/活跃度变化（粗=主流，细=边缘化）

### 合流的视觉表达
- 两条路径的线汇入一个菱形节点
- 菱形后继续延伸为新的路径

## 新论文挂载流程

算法推荐 + 人确认：
1. 新论文标注了 `inherits: X` → 推荐挂在 X 所在路径的后续
2. 新论文标注了 `competes: X` → 推荐在 X 的分叉点后新建分支
3. 新论文标注了 `inherits: [A, B]`（A、B 不同路径）→ 推荐为合流点
4. 新论文标注了 `borrows: Y`（Y 在另一条路径）→ 画跨 lane 虚线弧
5. 人确认或调整位置

## 约束

- X 轴 = 时间（保持可读性）
- 不预设主干结构，主干从 inherits 链中涌现
- 保持现有 lane 颜色体系
- 路标节点数量控制在每条 lane 6-10 个（太多就乱了）

## 验收标准

- [ ] 能清晰看出至少 3 个分叉事件和 2 个合流事件
- [ ] 跨 lane 借鉴关系有明确视觉标注
- [ ] 新论文只需标注关系即可推荐位置
- [ ] 不超出屏幕宽度（或可横向滚动）
