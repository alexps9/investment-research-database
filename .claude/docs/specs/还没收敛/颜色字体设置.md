## 节点大小

`impact_score` → 连续映射：

```
base = 3 + (score / 100) * 11    // 范围 3px - 14px
radius = isFoundation ? base + 2 : base
```

- Foundation（shape ≠ 'square'）：始终有 label，+2px 加成
- Adaptation（shape = 'square'）：impact > 35 时显示标题，否则纯小点

## 动态效果

| 状态 | 视觉表现 |
|------|----------|
| Rising（is_rising=true） | opacity 脉冲动画 0.85↔0.5（2s 循环）+ 高斯模糊发光 |
| Weak signal（is_weak_signal=true） | 虚线边框 stroke-dasharray: 3 2 |
| Hover Foundation | 半径 +2px，黑色 stroke；其 adaptation 群保持可见，其他节点淡化至 12% opacity |
| Hover Adaptation | 半径 +2px，黑色 stroke；不触发聚集效果 |

### Hover 聚集逻辑

当 hover 一个 Foundation 节点时：
1. 该 Foundation 自身正常高亮
2. `builds_on` 包含该 Foundation id 的 adaptation 保持 opacity: 1
3. 其他所有 Foundation 和 adaptation → opacity: 0.12
4. transition: opacity 0.2s

## Impact 算法

权重：citation 45% + venue 30% + institution 25%

- Citation: log(cited_by_count+1) → 全局 min-max 归一化到 0-100
- Venue: T1(best paper)=100, T2(顶会)=75, T3(好会)=50, T4(workshop)=25, T5(preprint)=10
- Institution: T1(顶级AI lab)=100, T2(一流学校)=75, T3(强校/大厂)=50, T4(其他)=25
- Rising: 近 2 年论文中 citation rate (cit/month) top 20%
- Weak signal: institution_tier ≤ 2 + cited < 20 + age ≤ 6 months
- 如 citation_pct > 70 且 venue < 50，则 venue 分向上修正（高引用补偿低 venue）
