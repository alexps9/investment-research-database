一眼总览

做得好的：你和 Claude 形成了强大的迭代式设计合作关系——把它当成可视化方案的讨论伙伴，对不满意的方案果断拒绝（比如从双 tab 方案转向密度编码方案）。你的爬虫工作展示了出色的问题解决能力，而从代码库中直接提取面试叙述的做法也非常聪明。

阻碍因素：Claude 这边：经常走错方向（编辑错误文件、误解概念、提出未落地的架构），需要多轮纠正。你这边：可视化会话缺少具体前置锚点——没有明确验收标准或像素级参考时，Claude 会"解读"而非"实现"，导致每次 3+ 次布局失败。API 不稳定（连接中断、余额不足）加剧了两个问题。

快速改进：创建自定义 Skill（/command）用于可视化迭代循环——每次渲染后自动截图、等待确认再继续。爬虫管道可用 Headless Mode 无人值守运行。

高级玩法：未来可以并行启动多个 Agent，每个用不同方案实现同一可视化，截图对比选择。更强的是：让 Claude 从 spec 中提取验收标准，生成自动化测试，然后自主迭代直到全部通过。
