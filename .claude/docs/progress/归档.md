**Generated Content** (phase3-integration-docker-deployment-spec.md, ~25KB):

**1. Docker Architecture**:
- **Backend Dockerfile**: Multi-stage build (builder + runtime)
  - Stage 1: Install build dependencies (gcc for Python packages)
  - Stage 2: Install runtime dependencies from wheels
  - Stage 3: Copy app code, create non-root user, add health check
  - **Target**: Image size < 200 MB

- **Frontend Dockerfile**: Multi-stage build (deps + builder + runtime)
  - Stage 1: Install production dependencies (npm ci)
  - Stage 2: Build Next.js app (standalone output)
  - Stage 3: Copy built app, create non-root user
  - **Target**: Image size < 150 MB

- **Docker Compose Configuration**:
  - Services: backend (port 8000), frontend (port 3000)
  - Bridge network for inter-service communication
  - Health checks for backend (curl /api/health)
  - Dependency chain: frontend depends_on backend (service_healthy)
  - Environment variable support via .env files
  - Resource limits: backend (512MB), frontend (256MB)
  - Restart policy: unless-stopped

**2. End-to-End Testing Strategy**:
- **Tool**: Playwright (cross-browser testing)
- **Test Scenarios** (6 critical paths):
  - E2E-1: Complete search flow (P0)
  - E2E-2: Error handling - backend down (P0)
  - E2E-3: Error handling - invalid query (P1)
  - E2E-4: Empty results handling (P1)
  - E2E-5: Performance - 50 papers < 10s (P0)
  - E2E-6: Performance - graph rendering > 30 FPS (P1)
- **Coverage**: 100% of critical user paths
- **Execution Time**: < 3 minutes (CI-friendly)

**3. Production Configuration**:
- **Environment Management**:
  - .env.development (debug logging, relaxed CORS)
  - .env.staging (info logging, restricted CORS)
  - .env.production (warn logging, strict CORS, optimized)
- **Structured Logging**: JSON format for production
- **Resource Limits**: CPU/memory constraints per service
- **Security Hardening**:
  - Docker image vulnerability scanning
  - Non-root users in all containers
  - No hardcoded secrets (environment variables only)
  - CORS whitelist for production domains

**4. Task Breakdown** (4 sub-phases, 6 days total):
- **Phase 3.1**: Docker Configuration (2 days)
