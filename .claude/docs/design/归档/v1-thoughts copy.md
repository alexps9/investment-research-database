## 完整问题定义

## 问题定义与拆解

### 核心问题
1. 需要从论文中提取什么东西，以及如何将论文映射为标准化的技术因子/向量，来 carry 论文的信息？
2. 如何构建论文之间的技术关联网？通过引用关系还是社区检测？怎么识别突破瓶颈/有开创性的技术（比如 Mamba）？
3. 如何将高维复杂技术网络压缩为简洁可解释的技术演化树？什么样的演化树是好的（可解释性，人能看懂；连续性，路径不跳跃；覆盖核心论文，不丢关键信号）？

### 暂时不需要解决但比较重��的问题
- **上游信号**：社媒信息扩散比 arxiv 快，Arxiv 比传统期刊快。如何对高噪声社媒信号进行可信度加权，并与学术信号融合？
- **下游信号**：如何衡量一个技术是否在变热（根据技术扩散模型）？定义扩散量指标，衡量新技术对后续研究方向的影响范围与强度。以及有没有商业潜力判据。

## 第一步：提取论文特征，并分析得到向量

### 核心问题
如何将论文映射为标准化的技术因子/向量，来 carry 论文的信息？是以引用强度为准，还是以实际内容为准？

**目标**：在信息爆发初期，通过"语义坐标"和"方法论基因"锁定变异信号。

### 向量化的意义
- **社交媒体快但信噪比低**：主要是情绪驱动
- **引用量准但有滞后性**：一篇论文从 Arxiv 到正式发表并被他人引用，通常有 6-12 个月的延迟。等看到引用量爆发时，最好时机已经过去了
- **向量因为分析的是它和谁像**：它在解决什么瓶颈，它的学术基因，更加接近本质；且一发出来就能分析，所以容易在技术演进初期就看到它和老论文和瓶颈之间的距离

### 方案：SPECTER2 相似度边（语义）+ 文献耦合边（方法论）构建向量

#### SPECTER2 嵌入（语义向量，768维向量）
- 仓库：`https://github.com/allenai/SPECTER2`
- 由 Allen Institute for AI 训练的学术理解模型，Semantic Scholar 对每篇论文预计算好了，不用本地跑；直接调 API 拿
- 两篇论文的余弦相似度就是语义距离
- **优点**：
  - 不仅看字面意思，还通过引用关系校准字面意思

#### 文献耦合（方法论/技术基因向量）
- 根据引用关系识别文章是怎么做的
  - 两篇���文共同引用了 K 篇相同的参考文献 → 耦合强度 = K
- **优点**：
  - 不需要任何模型，纯图计算，NetworkX 几行就能算
  - 这个能抓到没互相引用，但方法论趋同的隐藏关联：比如两个独立团队同时从 SSM 理论出发

### 验证目标
- **语义验证**：能发现 Mamba 集群有很大偏离（Mamba 由于其底层数学基础（SSM）和解决的问题（线性扩展）与前者截然不同，它在向量空间中会与 Transformer 集群保持一定距离，但在序列建模这一大类中又很近）
- **方法论验证**：能发现隐性路径

## 第二步：构建论文之间的技术关联网

### 核心问题
如何构建论文之间的技术关联网？除了引用之外还有没有可能有别的关联？怎么识别突破瓶颈的技术？

### 形态：力导向布局，像 Obsidian 一样。采用分层叠加的形式。

#### 第一层：因果骨架（有向引用边）
- 基于 OpenAlex 的引用列表建立有向边
  - 调用 api，`Graph.add_edge(论文A, 参考文献B)` 建立有向边
  - 参考：`https://github.com/metaknowledge/metaknowledge`

#### 第二层：语义/方法论关联（无向加权边）
- 结合第一步的 SPECTER2 余弦相似度 和 文献耦合强度 K，计算加权。组合权重 = α×相似度 + (1-α)×耦合强度
- **为什么不能只依据引用关系？**
  - 引用可能是巧合（比如都引用了某篇经典），但语义相似是内容层面的真正关联

#### 力导向布局渲染
权重数值（相似度）决定距离，因果连线决定方向（矢量）。
- 赋值给边：NetworkX 中，`G[A][B]['weight'] = calculated_weight`
- 后端将 NetworkX 生成的 JSON 发给前端，包含 nodes（点）和 links（包含 weight 的线）
- 参考：`https://github.com/m-andrei/citation-network-visualization`

#### 社区检测
- Louvain 或 Leiden 算法进行聚类/社区检测，为不同的技术流派打标上色
- **意义**：直观地看到一个技术是如何从核心原点出发，在什么时间点由于瓶颈突破（权重突变）而分叉出新的社区，从而识别出真正的技术拐点
- 热度着色（根据引文增速）

### 验证目标
- **引用关系骨架验证**：能看到脱离主干（Mamba 节点不再产生向主流 Transformer 变体（如 Llama/GPT-4）的连线，因果箭头直接指向早期 SSM 论文）
- **力导向图验证**：能体现技术距离（由于权重计算（相似度+耦合度），Flash Attention 会被紧紧拉在 Transformer 原点旁边，而 Jamba 作为混合架构，会悬浮在 Transformer 与 Mamba 社区之间。）
- **Leiden 社区检测验证**：能自动分出技术流派（不同颜色，Attention 优化、MoE 架构和 Mamba/SSM 等独立社区）

## 第三步：从拓扑图到技术演化路径

### 好的技术演化树的标准
- **可解释性**：节点必须具备明确的技术增量标签（例如：从 Attention 到 Flash Attention 的效率提升）。
- **连续性**：路径上的向量位移应该是平滑的。如果出现巨大的跳跃，说明中间存在漏掉的关键信号（加速节点）。

### 生成演化树方法
1. **定义起点/根节点**（绕不开的老论文）
2. **用主路径分析 (MPA)**：呈现为 2-3 条平行演进的骨干线（如：纯 Attention 优化线、SSM 挑战线、混合架构融合线）
   - 在 NetworkX 基础上，计算网络中每条边被知识流经过的频率（SPLC 算法），提取出流量最高的 Top 5 路径。
   - 参考：`https://github.com/wlin12/MainPathAnalysis`
   - 参考：jdvelasq/techminer

### 节点包含信息
论文名，技术增量标签

#### 技术增量标签生产方法
对于树上的两个相邻节点 A（先辈）和 B（后辈），计算它们的差分向量 ΔV（不可解释）
- **差分触发**：对于树上的每一条边 A→B，自动触发一个 LLM 比较任务。
- **构建 Skill**：让 LLM 对比两篇论文的正文，回答："相对于 A，B 到底多做了什么？"

### 演化路径示例
- **短期**：Attention 效率优化（Flash / sparse）
- **中期**：Hybrid 架构（Transformer + SSM）
- **长期**：潜在替代范式（State Space Models，代表如 Mamba）

---

## 其他：如何衡量一个技术是否在变热

根据技术扩散模型：
- **引文增速**：监测核心论文被引用的增长斜率。
- **扩散层次**：观察信号是否从顶级源传导至大量跟随者，最后渗透到二三线期刊（证明范式已公认并准备产业化）

```python
# 新增 services/semantic_network.py
class SemanticNetworkBuilder:
    def __init__(self):
        self.specter2_client = Specter2Client()  # 调用 HuggingFace API
    
    def build_weighted_network(self, papers):
        # 1. 原有引用边（有向）
        citation_edges = self.build_citation_edges(papers)
        
        # 2. 新增语义边（无向加权）
        semantic_edges = self.build_semantic_edges(papers)
        
        # 3. 返回双层图
        return {
            "citation_edges": citation_edges,  # 实线
            "semantic_edges": semantic_edges,  # 虚线
        }
```
