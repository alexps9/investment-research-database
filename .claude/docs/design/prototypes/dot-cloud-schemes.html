Dot Cloud 方案对比

6 种不同的"重要节点 + 背景论文"混合呈现方式 · 点击 canvas 可 hover 查看效果

实施计划

1. 数据架构变更

paper 新增 role 字段区分重要节点和背景论文：

// world-model-data.json paper schema
{
  "id": "sora",
  "role": "foundation",  // "foundation" | "adaptation" | "background"
  "lane": "video_gen",
  "row": "diffusion_video",
  "year": 2024, "quarter": 1,
  "impact_score": 92,
  // background papers 可省略 title 以外字段
}
审查重点：role 的阈值——impact_score > 50 自动为 foundation？还是纯手动标注？建议：手动标注 foundation（~30篇），其余自动归为 background。

2. 渲染管线（推荐方案 A+B 混合）

基于"均匀散点 + 靠近对应 Foundation"的混合策略：

// page.tsx 核心渲染逻辑
function renderQuarterCell(papers, cellX, cellY, cellW, cellH) {
  const foundations = papers.filter(p => p.role === 'foundation');
  const backgrounds = papers.filter(p => p.role === 'background');

  // 1. 先放 Foundation 节点（大圆，带 label）
  foundations.forEach((p, i) => {
    const x = cellX + layoutFoundation(i, foundations.length, cellW);
    const y = cellY + cellH * 0.4;
    renderNode(p, x, y);  // 8-16px radius
  });

  // 2. 散布 Background 小点
  backgrounds.forEach(p => {
    const seed = hashStr(p.id);
    const rng = seededRandom(seed);  // 稳定位置
    const x = cellX + rng() * cellW;
    const y = cellY + rng() * cellH;
    renderDot(p, x, y);  // 2px, opacity 0.3
  });
}
  • 小点用 paper.id 的 hash 做 seed → 位置稳定，刷新不跳
  • 大节点优先占据 cell 中心区域，小点填充剩余空间
  • 论文越多 → 小点越密 → 自然传达热度

3. 数据流向

日报论文的入库路径：

  • 日报爬取 → 原始 paper list
  • LLM 分类 → 自动打 lane + row 标签
  • 写入 JSON → role="background"，无需手动
  • 人工 review → 发现重要论文 → 升级 role="foundation"

4. 交互设计

  • Hover 小点 → tooltip 显示论文标题 + 日期
  • Hover 空白区域 → 若附近有密集点云，显示 "该季度 N 篇论文"
  • Click 小点 → 侧边栏展示论文详情 + ArXiv 链接
  • 区域选择 → 框选多个点 → 列表展示
审查重点：500+ 个 SVG circle 的性能——超过 1000 个点考虑用 Canvas 层渲染背景点，SVG 只画 Foundation 节点。

5. 关键决策清单

  • □ 选择哪个方案（或混合）？
  • □ Foundation 阈值：手动标注 vs impact > X 自动？
  • □ 小点大小：固定 2px 还是按 impact 微调(2-4px)？
  • □ 性能策略：纯 SVG 还是背景点用 Canvas？
  • □ 日报入库：全自动 LLM 分类还是需要人工确认？
