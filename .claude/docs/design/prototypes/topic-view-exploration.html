Topic View v3 — 跨Lane贝塞尔分叉（6种风格变体）

共同方向：三泳道 + 主干/分叉/融合 + 跨Lane因果弧 · 差异：曲线风格 / 密度 / 融合处理 / 标注方式

#1 紧凑弧线 + 因果聚类
曲线紧密贴合主干再分离 · 节点按因果聚类而非等距 · 融合=双线汇入菱形
风格：曲线紧凑——分叉弧先贴合主干水平延伸再垂直剥离。节点按因果聚类（Mamba附近聚集了竞争者）。跨Lane弧用虚线+箭头。融合=菱形。
#2 有机河流 + 线宽编码优势
曲线宽松如河流 · 线宽随时间变化编码路线主导力 · 融合=汇流三角 · 标注沿线
风格：河流式宽弧——曲线大弧度缓慢偏离。线宽随时间增减编码"该路线当前是否dominant"。SSM在2024最粗→2025变细；Reasoning在2024后爆粗。融合=三角汇流。
#3 棱角地铁 + 锐利分叉
45°斜线分叉而非曲线 · 地铁图风格 · 融合=交叉圆环 · 标注用竖排标签
风格：地铁图棱角——所有分叉用45°直线（无曲线），强调"决策时刻"的尖锐感。融合=同心圆环交汇。整体风格更工程/几何。
#4 密集生态 + 多层递归分叉
信息密度最大 · 每条路线有2-3个子分叉 · 融合=多色汇入点 · 标注用微缩annotation
风格：最大密度——展示完整的递归分叉树（SSM→SSM-only/Hybrid/LinearAttn, Reasoning→Search/Verifier/Latent）。信息量最大但可能显得拥挤。适合"全景鸟瞰"场景。
#5 极简骨架 + 强跨Lane弧
每Lane只保留主干+1条fork · 跨Lane因果弧加粗突出 · 融合=双色大圆 · 留白最多
风格：极简——每Lane仅主干+1个最重要fork。大量留白让跨Lane因果弧成为视觉重心（加粗+大标注）。适合"讲故事"场景：先看因果关系，再看细节。
#6 时间透明度 + 历史淡出
节点/线透明度编码时间远近 · 越旧越淡 · 分叉用S形柔和曲线 · 融合=渐变色过渡点
风格：时间透明度——左侧(早期)节点和线条透明度低，右侧(近期)鲜明。S形柔曲线分叉。视觉聚焦在"当下最重要的是什么"，历史自然淡出但仍可追溯。
